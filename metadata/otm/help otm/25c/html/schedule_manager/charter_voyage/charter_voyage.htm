Charter Voyage

This page is accessed via Operational Planning > Schedule Management > Charter Voyage.

A charter voyage represents an ocean transport movement by a carrier from a loading port to a discharge port. Within a charter voyage, there are several stowage modes which represent, at a conceptual level, separate "compartments" within the charter voyage. There is also capacity associated with each stowage mode as defined on a consol that you create for each stowage mode defined on the charter voyage. This capacity controls the orders that can be booked on the charter voyage.

Note: You can manually create a charter voyage or import it via integration using the Charter Voyage interface.

For charter voyage and each of its defined stowage modes, you can create a consol that has a single empty shipment attached. There are two ways a consol can be created.

  • When voyage data is sent in to Oracle Transportation Management through integration. For each voyage, one consol is automatically created for each stowage mode defined on the voyage. A shipment is also created for each consol at the same time.
  • You can also create a consol manually from a charter voyage using the Create Consol action.

Voyage Definition

Use the following fields to define voyage attributes. Then define the stowage modes for the voyage on the Charter Voyage Stowage tab.

Attribute

Description

Charter Voyage ID

A unique reference given by the carrier to a certain journey or departure of a means of transport. This refers specifically to a voyage of the vessel between the loading port and the discharging port. This is not a required field and an ID is automatically generated by the Business Number Generator if you leave this field blank.

Service Provider ID

The carrier responsible for the charter voyage or the party who receives and handles the booking.

Voyage Name

Any text that further describes the voyage (informational purposes only).

Source/Destination Location ID

Locations that define the port of load and port of discharge for the voyage. Oracle Transportation Management uses these locations as the Origin and Destination locations on the consol shipment that is created for each voyage stowage mode on the voyage.

Estimated Arrival/Departure Date

The planned time period when the voyage leaves the port of load location and when it arrives at the port of discharge location. Oracle Transportation Management uses these dates to determine the Start and End time for the consol shipment that is created for each stowage mode on the voyage.

Actual Arrival/Departure Date

Defines the actual time period when the voyage leaves the Port of Load location and when it arrives at the Port of Discharge location.

Adjusting the Departure Date field can affect the capacity commit time, and booking freeze time fields on the Charter Voyage tab. Changes can also affect the Terminal Start Time, Terminal Close Time, and the Terminal Close Time Unitized fields on the Charter Voyage Stowage page.

Adjusting the Arrival Date field can affect the Capacity Commit Time and Booking Freeze Time fields on the Charter Voyage tab. Changes can also affect the Material Available Time and the Material Available Time Unitized fields on the Charter Voyage Stowage page.

The Terminal Close Time Unitized and Material Available Time Unitized fields are only used for unitized voyages and are otherwise ignored.

These adjustments only occur when the you change the departure date or the arrival date, and if that change is within the tolerance defined on the CHARTER_VOYAGE_DEPART_DATE_CHANGE_THRESHOLD or the CHARTER_VOYAGE_ARRIVE_DATE_CHANGE_THRESHOLD parameters. These fields are not adjusted if the change is outside of the tolerance define in the parameters.

If the change to the departure date is outside of the tolerance then the previous order movement will be adjusted instead of this voyage. After the order movement is adjusted the system checks for pre-existing shipments for that order movement in case the shipments are no longer feasible. If it is infeasible, then the change will be rejected. The change will also be rejected if the shipment for the previous order movement is on two vessels and one part of the shipment will no longer be able to arrive at the correct time for one of the vessels.

Vessel ID

The actual vessel that will make the voyage (information only). Vessels can be pre-defined in the Vessel Manager.

Voyage Project Name

A voyage project is a group of voyages that are linked together for performance reporting and calculation purposes. Use this field to assign this charter voyage to user-defined project (informational purposes only).

Project Sequence Number

Identifies the sequence of the voyage on the project (information only).

Feeder Vessel

Text that describes another vessel used to transport goods to/from the charter voyage vessel (information only).

Capacity Commit Time

A number of days, hours and minutes before the charter voyage Departure Time by which all the capacity needs must be committed to the voyage carrier (information only).

You can configure a milestone template that could monitor this duration and take specific action. For example, you could configure milestones to send notification to a user when the charter voyage Capacity Commitment Time is reached and perform the Change Consol Status - Committed action to set the status of all the consols on this charter voyage to COMMITTING_READY-TO-COMMIT.

This field is recalculated if you change the departure date or the arrival date, and if that change is within the tolerance defined on the CHARTER_VOYAGE_DEPART_DATE_CHANGE_THRESHOLD or the CHARTER_VOYAGE_ARRIVE_DATE_CHANGE_THRESHOLD parameters. This time cannot be reduced to a negative number by changing the departure date. This field is not adjusted if the change is outside of the tolerance define in the parameters.

Booking Freeze Time

Booking Freeze Time identifies a dead-line when no more bookings or amendments can be allocated on a single voyage (i.e. now the booking lines are fixed, shipment planning on a voyage is now completed). Enter a number of days, hours, minutes before the charter voyage Departure Time by which all the orders and container groups must be booked with the charter voyage carrier. Â Beyond this time, no additional orders or container groups may be added to the voyage (although Oracle Transportation Management does not automatically prevent this).

You can configure a Milestone Template that could monitor this duration and take specific action that would prevent an order from being booked on the consol shipments that are associated with the charter voyage. For example, you could configure milestones to set all the consols of this voyage to a frozen state when voyage booking freeze time is reached. All the shipments for the consols of this voyage will be in a REVIEWED-EXECUTED state. Once the consol's status becomes frozen, no further changes can be made to the booking lines on this consol and no additions for the consol are allowed.

Once all the consols of a voyage become frozen, a milestone could launch a process to summarize an aggregated list of booking lines on the voyage and send it to the carrier, loading port operator, and discharge port operator.

This field is recalculated if you change the departure date or the arrival date, and if that change is within the tolerance defined on the CHARTER_VOYAGE_DEPART_DATE_CHANGE_THRESHOLD or the CHARTER_VOYAGE_ARRIVE_DATE_CHANGE_THRESHOLD parameters. This time cannot be reduced to a negative number by changing the departure date. This field is not adjusted if the change is outside of the tolerance define in the parameters.

Canceling a Voyage

Use the Cancelled checkbox to cancel a voyage. Canceling a voyage does not delete the charter voyage from the database but flags the voyage as cancelled and prevents an order from being booked on the charter voyage.

If there are orders booked on any of the consol shipments already associated with a charter voyage, it cannot be cancelled.

Note: You can also delete the voyage provided there are no attached consol shipments.
