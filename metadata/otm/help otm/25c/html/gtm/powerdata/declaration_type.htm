Declaration Types

This page is accessed via Master Data > Power Data > Declarations > Declaration Types.

Declaration types are used to identify the kind of declaration you are filing and drives how trade transactions are consolidated, split, and aggregated into declarations. Declaration type is also used to link various information together. Data such as declaration sub type, procedures, produced document types, and additional information are a few examples of data that is linked to declaration type through a declaration type profile. It is recommended that declaration types be set up as a combination of a country/regime and the type of filing. For example, BE â IMPORT represents a Belgian Import Declaration and EU â Transit Arrival represents a European Union Transit Arrival declaration.

Use this page to create, edit, and/or remove declaration types.

Adding a Declaration Type

  1. Enter a unique identifier in the Declaration Type ID field.
  2. Enter an ID in the Logic Configuration field to associate a declaration logic configuration with the declaration type.
  3. Select the domain in which this declaration type will be active from the Domain Name drop-down list.
  4. Enter the declaration type name in the Name field.
  5. Enter the declaration type code in the Code field.
  6. If desired, enter additional information about the declaration type in the Description field.
  7. Enter the regime associated with the declaration type in the Regime ID field.
  8. Enter an ID in the Request Status Group Profile field to associate a status group profile with the declaration type. This will be used when a declaration message is sent to the customs.
  9. Enter an ID in the Response Status Group Profile field to associate a status group profile with the declaration type. This will be used when a declaration message is received from the customs.
