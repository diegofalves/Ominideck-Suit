Policies

This page is accessed via Master Data > Power Data > Compliance Rules > Policies.

Use this page to view the existing compliance policies available to you, or create new ones.

A compliance policy consists of one or more GTM services that are used in compliance rule screening, product classification, license assignment, restricted party list screening, and landed cost calculation. A compliance policy includes the sequence in which the services are executed as well as defined parameters for each service included. Compliance policies are typically used for trade transaction screening.

You can configure different compliance policies according to the jurisdictions under which transactions fall. For example, a ship-from transaction from the Netherlands to a US company must comply with EAR (Export Administration regulations) and the Dual Use Items Regulations for the European Community. The compliance policy would include the Compliance Rule Set Group for EAR, a rule set group covering the European rules as well as restricted party list screening and product classification lookup for both ECCN-US and ECCN-NL in order to make sure that items and parties in the transaction are classified completely before running the compliance rule screening.

Adding a Compliance Policy

  1. Enter an ID for the new policy in the Policy ID field.
  2. Enter a description for the policy in the Description field.
  3. Enter the policy's name in the compliance Name field.
  4. Select a domain where this policy will exist from the Domain Name drop-down list.

Compliance Policy Details

Use the Compliance Policy Details section to define the new policy.
