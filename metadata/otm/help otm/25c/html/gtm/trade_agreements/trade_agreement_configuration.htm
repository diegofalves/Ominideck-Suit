Trade Agreement Configuration

The trade agreement is content that you can download from a third-party content provider. Certain data is populated on the trade agreement including information in the header and the Partners section. You can then add additional information including:

  • Rollup Allowed: Select this check box to indicate if the trade agreement supports the rollup principle, also known as absorption. When selected, GTM will first try to qualify the finished good using the originating and the non-originating status of the components. Â  If the qualification does not pass, GTM will then apply the rollup principle to qualify. As part of the rollup, the value of the non-originating inputs contained in the intermediate materials which have acquired originating status is counted as originating content in the calculation of the Value Add criteria.Â 
  • Allow Partial Credit: Select this check box to indicate if the trade agreement supports accumulating originating content from non-originating components (components which are originating but not qualified).

    Note    : Only if the property gtm.roo.leafComponent.originating.considerQualification is set to true and the Allow Partial Credit checkbox is selected, then non-originating components will be considered during qualification.
  • Â Allow Prorate on Multi-Sourcing: When this checkbox is selected, GTM will use the cost and the regional value content Â  Â percentage of the component to determine the originating content. The originating content Â of each component is used by the qualification engine to determine if a manufactured good qualifies for a trade agreement.

    When the Allow Prorate on Multi-Sourcing checkbox (on the Trade Agreement page) is selected and a value is specified in the Percentage Sourced field (on the Item Origins page), the value calculation of originating /non-originating material will be prorated for components that are sourced from multiple partners.
  • Regional Value Content Methods: You can list the regional value content methods that apply for the trade agreement. Values include NET COST, TRANSACTION VALUE, BUILD UP and BUILD DOWN.
  • Rules of Origin Formulae: You can list the Formula IDs for the rules of origin that are valid for this trade agreement. If you do not specify the rules of origin formulas on the trade agreement, GTM will use the formulas that are associated with the Rules of Origin Rule Types associated with the Regional Value Content Methods on the trade agreement.Â 
  • Dependent Formulae: You need to specify any formulas that will be used by the qualification engine. This would include formulas for:
    • EXTENDED VALUE
    • COMPONENT EXTENDED VALUEÂ 
  • Remarks: Specify any additional remarks that pertain to the trade agreement.
  • Reference Numbers: You need to specify 2 reference numbers which tell GTM the formulas used to calculate extended cost.Â 
    • RVC METHOD1-EXTENDED VALUE FORMULA: Identifies the formula that is used to calculate the extended cost of the finished good. This is the same formula as defined in the Dependent Formulas section. For example, create a formula with an ID of EXTENDED VALUE. In the Reference Numbers section, specify the Reference Number Qualifier ID = RVC METHOD1-EXTENDED VALUE FORMULA and the Reference Number = EXTENDED VALUE.Â 
    • RVC METHOD1-COMPONENT EXTENDED VALUE FORMULA: Identifies the formula that is used to calculate the extended cost of each component. This is the same formula as defined in the Dependent Formulas section. For example, create a formula with an ID of COMPONENT EXTENDED VALUE. In the Reference Numbers section, specify the Reference Number Qualifier ID = RVC METHOD1-COMPONENT EXTENDED VALUE FORMULA and the Reference Number = COMPONENT EXTENDED VALUE.

Logic ConfigurationÂ 

You can set a logic configuration with the Logic Configuration Type of "GTM TRANSACTION CONFIGURATION", which helps you to drive the logic of the qualification engine based on your business scenarios. GTM ships with an out-of-the-box logic configuration that you can use, or you can copy it and make changes based on your business needs.

The following parameters in the Logic Configuration Type of "GTM TRANSACTION CONFIGURATION" are used by the qualification engine:

  • Constraint sets for identifying related objects:Â 
    • ITEM ORIGIN FOR ITEM STRUCTURE COMPONENT: This constraint set is used to match item origins on the item with item structure components.
    • ITEM ORIGIN FOR TRANSACTION LINE: This constraint set is used to match trade transaction lines with the item origins associated with the item on the line.Â 
    • ITEM STRUCTURE FOR TRANSACTION LINE: This constraint set is used to match trade transaction lines with the item structures.
    • ITEM VALUE FOR ITEM STRUCTURE: This constraint set is used to match item values on the item with the item structure.
    • ITEM VALUE FOR ITEM STRUCTURE COMPONENT: This constraint set is used to match item values on the item with item structure components.
  • General: tells GTM which qualifiers to use when running qualification and other actions.Â 

Once you have configured your logic configuration:

  • For item qualification, assign the logic configuration to the parameter set. The parameter set can then be defined in user preferences or domain settings.
  • For trade transaction line qualification, assign the logic configuration to a Trade Transaction Type. This enables you to have specific configurations for different types of transactions such as export transactions and import transactions.

Preference Criteria Configuration

You can optionally configure GTM to automatically assign the preference criteria to the item qualification as part of the trade agreement qualification process. Preference criteria provides details about how your finished good qualifies for a trade agreement and can be used on documents generated by GTM. For example, is it wholly obtained, does a product specific rule apply, is de minimis met and so on.Â 

You can create the preference criteria records as defined in the trade agreement. When you define the preference criteria, you can specify:

  • Trade Agreement: Enter the trade agreement that the preference criteria is related to.
  • External Reference Code: Identify the external code which can be used on documents.
  • Priority: Identify the order given in the trade agreement document for preference criteria. If more than one preference criteria apply, this defines the order in which they are assigned.
  • Related Success Codes: Identify the Justification Code(s) associated with the preference criteria. For example, you can identify if the preference criteria are related to a regional value content method or another rules of origin rule type.

When you trigger trade agreement qualification on your item-based objects, an item qualification record is created and/or updated. If the preference criteria can be determined, GTM assigns it and displays the Justification Code and Justification Description with more details. If you trigger trade agreement qualification for a trade transaction line, the preference criteria from the item qualification will be copied to the transaction lien qualification.
