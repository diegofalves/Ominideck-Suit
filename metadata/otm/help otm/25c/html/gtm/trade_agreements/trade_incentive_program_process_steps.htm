Trade Incentive Program Process Steps

The high-level steps followed for the Trade Incentive Program feature are:

Perform Entry/Exit Recording: Run this action on declarations or trade transactions. Based on the data setup and logic configuration, the application does the following:

  1. Identifies all the applicable trade incentive programs based on the entry and exit profiles.
    • Declaration type profile, declaration type, sub-type, transaction type profile, transaction type of the entry exit profile are matched with the details of the entry line.
    • Additionally, if profile type is "Procedure", declaration procedure is matched to the list of procedures on the profile.

      OR

      If profile type is "Other", reference numbers and remarks of declaration are matched to the details present on the profile.
  2. If there are any entry and exit saved conditions defined on the applicable program, execute the saved conditions with line ID as input to further filter the program that can be applied.Â 
  3. If there is any program fetched after applying exit profile and exit saved condition,Â 
    • Identifies all the trade incentive program rules and corresponding product specific rules applicable by product classification code. Iterates over those in the order of priority.Â 
    • Identifies the open inventories matching by the constraint set criteria ordered as per the "Accounting Method" defined on the trade incentive program.Â 
    • Balance quantities are deducted from one or more inventories as needed. Inventory movement of type "Issue" is created against those inventories. The inventory status is updated.
    • Performs the value calculations to calculate the value balances.
  4. If there is no program fetched after applying exit profile and exit saved condition,Â 
    • Checks whether there is any Bill of Material defined on the declaration line or its corresponding trade transaction line. If so, iterates over the components of the Bill of Material to run step # 3.
  5. If there is any program fetched after applying entry profile and entry saved condition,Â 
    • Checks if authorization is required and available. If so, captures a posting against the allowed goods and deducts the quantity and value balances of allowed goods.
    • Derives the maximum discharge period from all the trade incentive program rules and corresponding product specific rules applicable by product classification code.Â 
    • Calculates the maximum discharge date using the entry date and maximum discharge period.
    • Creates an inventory record and an inventory movement record of type "Receipt".
    • Copies data from declaration line to the inventory as configured in the data configuration.
