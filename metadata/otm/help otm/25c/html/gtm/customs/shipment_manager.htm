Customs

Declarations

Note: Customs Shipment and Customs Shipment Line have been renamed as Declaration and Declaration Line, respectively. However, this change will not impact backend processes in which the object types will continue to be named as GTM SHIPMENT and GTM SHIPMENT LINE.

This page is accessed via Customs > Declarations.

Use this page to manage the declarations you work with in GTM.

Declarations usually consist of one to many trade transactions. You can consolidate, split, or aggregate shipment information on your trade transactions into a declaration that can be used for customs filing. The consolidating, splitting, or aggregation can be at the trade transaction or trade transaction line level. Declarations can be then sent from GTM to regulatory bodies or trading partners for filing purposes.

Header

Adding a Declaration:

  1. Enter an ID in the Declaration ID field.
  2. Enter IDs in the Declaration Type ID and Declaration Sub Type ID fields to add the type and sub type of declaration.
  3. If you are looking at an existing record, click Status to view the internal statuses assigned to the record. Please note the following:
    • Declaration status will be calculated based on the top level status only.
    • Any line status (which includes its own status and the status of its immediate child lines) will be considered when the status is calculated.
    • Whenever a line's status is calculated, the status of it's parent line along with the declaration's status will be recalculated.
  4. Enter IDs in the Procedure ID and Prior Procedure ID fields to add information related to procedures to the declaration.
  5. Enter an ID in the Procedure Detail ID field to add procedure details to the declaration.
  6. Enter an ID in the TradeÂ Preference ID field to apply a tariff preference type to the declaration.
  7. Enter an ID in the Bond ID field to assign a bond to the declaration.
  8. Enter an ID in the Valuation Method field to assign a valuation method to the declaration.
  9. Enter an Exchange Rate ID and an Exchange Rate Date to add information related to exchange rates to the declaration.
  10. Enter a currency in the Currency ID field.
  11. Enter an Incoterm ID and an Incoterm Location.
  12. If the declaration includes hazardous cargo, select the Hazardous Cargo option.
  13. Enter Transport Mode, Conveyance Names, and Conveyance Flags in the following fields to define details of transportationÂ used at different stages of a shipment. Use the Conveyance Name and Conveyance Flag fields to add information such as vessel name or flight number. The fields are:
    1. Departure Transport Mode
    2. Departure Conveyance Name
    3. Departure Conveyance Flag
    4. Border Transport Mode
    5. Border Conveyance Name
    6. Border Conveyance Flag
    7. Arrival Transport Mode
    8. Arrival Conveyance Name
    9. Arrival Conveyance Flag
  14. Select the domain in which this declaration will be active from the Domain Name drop-down list.
  15. Sellect the Template checkbox if you want to use this declaration as a template when performing the actionÂ Create Declaration.
  16. In View and Edit modes, you will see the customs filing request and response status groups and codes associated with the declaration. In addition you can see Created By, Created Date, Last Modified By, and Last Modified Date information. Also, a Logic Configuration ID will be displayed if the declaration is associated with a corresponding logic configuration. If applicable to the declaration, you can view a Consolidation Criteria ID.

Declaration Policy

Use this section to define the policies you wish to be applied to this declaration .

Adding a Declaration Policy:

  1. Select an ID from the Policy Qualifier ID drop-down list.
  2. Enter a Policy ID.
  3. Click Save.
  4. In View and Edit modes you will see any compliance rules and compliance rule set group ID read-only information that are associated with the policies you select.

Declaration Lines

Use this section to add information about declaration line associated with this declaration.

Click New Declaration Line to move to the Declaration Lines window where you can enter the declaration line information.

If you are viewing/editing an existing declaration, declaration lines will be displayed in a grid. If you wish to add a new child line, click New Child Line to open a declaration line window.

Date

Use this section to add date information to the declaration .

Adding a Date Qualifier:

  1. Select a qualifier from the Date Qualifier ID drop-down list.
  2. Enter a date in the Date field.
  3. Click Save.

User-defined Code

Use this section to add user-defined codes to the declaration.

Adding a User-defined Code:

  1. Select a user-defined code category from the Category drop-down list.
  2. Select a user-defined code type from the Type ID drop-down list.

    Note: The available options are determined by your choice of category in the previous step.

  3. Select a user-defined code value from the Type Code drop-down list.

    Note: The available options are determined by your choice of type in the previous step. For easy understanding of the user, this drop-down list displays description of the user-defined code values instead of only the codes.

  4. Click Save.

Involved Parties

Use this section to add involved party information for the declaration.

The Indicator column corresponding to each involved party displays the restricted party list screening (RPLS) status of the party. RPLS status is indicated as follows:

  • White: indicates status as RPLS_NOT STARTED
  • Green: indicates status as RPLS_PASSED
  • Red: indicates status as RPLS_FAILED
  • Yellow: indicates status as RPLS_REQUIRES REVIEW or RPLS_ESCALATED

Adding an Involved Party:

  1. Select a qualifier from the Involved Party Qualifier ID drop-down list.

    Note: You can specify a Transmitter ID or Filer ID as reference number on the party.
    GTM will first check if a transmitter and/or filer is specified as an involved party. If so, GTM will use the reference number on the associated party to populate the Transmitter ID and Filer ID. If this data is not present on the party, then GTM will use the data in the properties.

  2. Enter a party in the Involved Party field.
  3. Click Save.

Quantity

Use this section to add quantity details to the declaration .

Adding a Quantity Qualifier:

  1. Select an ID from the Quantity Qualifier ID drop-down list
  2. Enter a value in the Quantity field.

    Note: If you will be doing AES filing with this declaration , you must include the quantity qualifier GROSS WEIGHT.

  3. Select a unit of measurement from the UOM drop-down list.
  4. Click Save.

Values

Use this section to add information about values to the declaration . You can add as many value qualifiers as you need.

Note: Value qualifiers can also be added to a declaration by running the Execute Compliance Rules action.

Adding a Value Qualifier:

  1. Select a value qualifier from the Value Qualifier ID drop-down list.
  2. Enter a currency value in the Value field.
  3. Select a currency from the drop-down list.
  4. Enter a formula expression in the Formula Expression ID field. The value of the value qualifier is calculated based on this formula expression.
  5. Select an option from the Method of Calculation drop-down list. Your list of options include the following:
    • Entered Manually: Select this option if you want to specify the value of the value qualifier.
    • System Calculated: Select this option if the value of the value qualifier is calculated using a formula assigned.
    • Calculated Externally: Select this option if the value of the value qualifier is calculated using the Â duty and tax values obtained from a third party data provider by executing the Estimate Duty and Taxes action.
  6. Select the Fixed checkbox if Â you DO NOT want to recalculate and override the value of the value qualifier using the modified formula on the Compliance Rule page, when the compliance rule is re-executed. Selecting this checkbox will also ensure that the method of calculation specified in the previous step will not be overridden. Clear this checkbox if you want to override the method of calculation and override the value of the value qualifier using the modified formula.
  7. Enter any additional note, if desired, in the Value Note filed.
  8. Select an option from the Payment Method drop-down list.
  9. Click Save.

Conveyances

Use this section to record the details of equipment used for this declaration.

Text Templates

Use this section to specify templates that will be used with this declaration.

Adding a Text Template:

  1. Enter an ID in the Text Template ID field.
  2. Enter a type in the Document Type field.
  3. Enter any related text in the Text field.
  4. Click Save to record the information or click Override All if you want your information to overwrite the template information already in the system.
