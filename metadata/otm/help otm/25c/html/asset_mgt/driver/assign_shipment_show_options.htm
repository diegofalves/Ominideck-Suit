Fleet Management

Assign Shipment (to Driver)

This page is accessed via Fleet Management > Driver Management > Driver Manager > Actions > Fleet Management > Assign > Shipment.

Use this action to assign a shipment to a driver. You can search for all options, select the best option or assign a shipment manually.

When you assign a shipment to a driver, the service provider of the shipment is changed to the one found in the rate offering that is tied to the driver or driver type. If there is no service provider associated with the driver or driver type, the DEFAULT SERVICE PROVIDER FOR FLEET ASSIGNMENT parameter will be used.

When you assign a driver, the associated power unit and the equipment associated with the power unit will be assigned.

The assignment of a shipment to a driver automatically updates the HOS Daily Summary data to reflect the work performed on the shipment.

You can configure how to get the driver home by setting the MARK DRIVER HOME STOP FIXED and GET THE DRIVER HOME FOR FLEET OPTIMIZATION parameters when using this action.

Note: This action will fail if the shipment has a SECURE_RESOURCES status of TENDERED or ACCEPTED.

Assigning a Shipment to a Driver

  1. Select the Assign Shipment action. The Assign Shipment to Driver - Override Criteria page opens. See the Override Criteria for Fleet Planning topic for more information on choosing objects to ignore and constraints to select.
  2. Choose the assignment options. These are the same as described in the Override Criteria for Fleet Planning topic, but are explained in more detail here:
    • Manual assignment: To assign a shipment manually, select the Shipment ID option button (default) and enter a Shipment ID. A grid may appear "Select an assignment to insert after". This shows the most recent assignments to the driver. Their next available time (NAT) must be later than the current time. The number of assignments displayed is also limited by the fleet management parameter "Driver Assignment Maximum Insertion Depth".
    • Best option: After clicking More Options, clicking OK, automatically selects the best option.
    • All options: Use Show Options (after clicking More Options) to show all valid shipments for the selected driver.
  3. If you chose to Show Options, a new page opens that displays valid options along with Costs, Source and Destination Location and Start and End Time. On the new page you can manually select a shipment to assign to the driver. Select the option button for the preferred Shipment ID.
  4. If you leave the Get the Driver Home drop-down at "Use Parameter Value", then the Get the Driver Home parameter is used to determine if the driver's home is added as a stop. Selecting "No" ignores the parameter and does not add the driver's home as a stop. Selecting "Yes" ignores the parameter, brings the driver home, and displays the Mark Home Stop Fixed field.
  5. If you set the Get the Driver Home value to "Yes", then the Mark Home Stop Fixed field appears. This setting lets you override the settings available in the MARK DRIVER HOME STOP FIXED parameter, but for this specific situation. The possible values are:

    If set to No, the return home stop is not marked as permanent for the driver's current assignment and also the "Stop_Not_Removable" special service is not set on the home stop. So the next assignment for the driver will start from the last freight stop of the current assignment.

    If set to Yes, the return home stop is marked as permanent for the driver's current assignment and also the "Stop_Not_Removable" special service is set on the home stop. So the next assignment for the driver will start from the home location.

    If set to "Based on Driver Shift", the "Stop_Not_Removable" special service is set on the home stop, but the permanent flag is not set on the stop. The state of the special service and permanent flag is used as an indicator for the logic to decide whether to remove the home stop based on the driver shifts or not during the next assignment. If the next assignment for the driver falls on the same shift as the current assignment, then the home stop will be removed. For the next assignment, the driver will start from the last freight stop of the current assignment. Â But, if the next assignment for the driver falls on the next shift or outside the current shift, then the home stop is not removed from the current assignment. The driver will go home after the current assignment and start from the home location for the next assignment.

Get the Driver Home Without Assigning a Shipment

  1. Optionally, if when you select "Yes" in the Get the Driver Home field, another table may appear. This table displays the shipment that is already assigned to the driver. If you choose that option instead of one of the new shipments in the first table, a new shipment will NOT be assigned to the driver. The shipment will stay the same as the one currently assigned to the driver. However, a non-freight-related stop will be added to the shipment for the driver's home. The option to not assign a shipment, but to still send the driver home, does not appear unless the driver is already assigned a shipment. Likewise, if the driver's final stop is already his home, then this option will not appear.
  2. Click OK.
