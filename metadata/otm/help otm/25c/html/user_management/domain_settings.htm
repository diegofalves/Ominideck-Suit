Configuration and Administration

Domain Settings

This page is accessed via Configuration and Administration > Domain Management > Domain Settings.

Use this option to configure the following settings for a domain:

  • Parameter Set
  • Postal Code Validate Type
  • Functional Currency

Configuring Domain Settings

  1. Select a Parameter Set ID from the corresponding drop-down list.
  2. The Use Parameter Set on Shipment checkbox indicates that the parameter set on the shipment should be used for all shipment actions. If this is not selected, the parameter set specified in user or domain will be used. Using a different parameter set could have results that are inconsistent. For example, one parameter set could specify "Hold as Late as Possible" is true while another is set to false. There is a related business property: glog.business.shipment.useShipmentPlanningParameterSet.

    Note: The elements of a parameter set can change over time. Just because the same parameter set is specified, the results can be different. If you change the configuration of a parameter set between shipment creation and shipment modification, you could get different results.

  3. Select a postal code validate type from the Postal Code Validate Type drop-down list.
  4. Select a conditional booking profile from the Conditional Booking Profile ID drop-down list.
  5. Select a compare engine threshold profile ID from the Compare Engine Threshold Profile ID drop-down list.
  6. Select a conditional booking threshold profile ID from the Conditional Booking Threshold Profile ID drop-down list.
  7. Enter a From Address Contact ID. This is contact that can appear in messages generated by the system. For example, if an email is generated by the system, this would be who the email is from.
  8. Select a functional currency from the Functional Currency drop-down list.
  9. Select the Purge Only (No Archive) checkbox to have a domain's data not be archived when it is cyclically purged. In order to use this checkbox, you must work with Oracle Support.Â Oracle Support can set the purge capabilities two ways after you contact them.

    The first setting Support can make is to enable non-archiving purging by domain. When set that way, when you select the Purge Only (No Archive) checkbox, the purge will not archive data.

    The second settingÂ enables the non-archiving purge for ALL domains. In that scenario, the Purge Only (No Archive) checkbox is NOT functional on the Domain Settings page, since the Support setting applies to all domains. Data in all domains will be purged without archiving.

    If you have not contacted Oracle Support about using Purge Only (No Archive) checkbox and select it, you will be prompted to contact Oracle Support.

    Note: If you select this checkbox,Â domain data will be completely lost when the two-year purge starts. See the"Purge Only Option" section of the Administration Guide for details.

  10. Click Finished.
