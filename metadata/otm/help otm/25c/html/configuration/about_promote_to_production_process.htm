About the Promote to Production (P2P) Process

The Promote to Production (P2P) is standard way to define and manage one or more data sets for the purpose of migrating data from one OTM/GTM instance to another. Promote to Production is designed to facilitate thhe initial as well as subsequent incremental migration of data. Countries, locations, regions, lane, rate, itineraries, configurations, status, event status, workflow, etc. are some of the migration data set examples.

Although Oracle Transportation and Global Trade Management application is fully functional out of the box, an operational system will typically require some custom configuration or master data setup. Best practices is for such a configuration to be developed and tested in a pre-production environment, accepted by product and business/operational experts, and then promoted to a production environment.

At a high level, an "Export" migration project is defined on the source system. This migration project contains data set represents object groups and objects are that needs to be migrated. The following ways import this data set onto the target system:

  • Migration using Zip File
  • Migration using Web Service
