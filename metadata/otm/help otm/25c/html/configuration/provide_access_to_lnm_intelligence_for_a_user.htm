How to Provide Access to Logistics Network Modeling Intelligence for a User

By default, an Oracle Transportation Management (OTM) user does not have access to view Logistics Network Modeling Intelligence. If you log on to OTM and click Transportation Intelligence > Dashboard or any of the other Transportation Intelligence menu options and you see the âNot authorized. User should configure BI App in the User Manager to access TI.â message, you do not have access to Logistics Network Modeling Intelligence via a Business Intelligence Role. This topic will explain the process required to grant an existing OTM user access to Logistics Network Modeling Intelligence.

To access Logistics Network Modeling Intelligence, you will need to be an OTM user with the appropriate Business Intelligence Role and correct VPD profile. At a high level, the process required to provide access to Logistics Network Modeling Intelligence includes the following steps:

  1. Create an OTM user role that has a VPD Profile of FTI_DEFAULT.
  2. Edit an existing OTM user and complete the following:
    1. Assign the user role created previously.
    2. Specify the Business Intelligence Application of LNM.
    3. Specify the Business Intelligence Roles or BICONSUMER and BIADMINISTRATOR.
  3. Finally, you load the data from OTM in to Logistics Network Modeling Intelligence.

Now letâs review each step in more detail.

Creating a New User Role

To create a new user role in OTM, complete the following:

  1. Go to Configuration and Administration > User Management > User Role.
  2. Click New at the bottom of the page.
  3. Enter a User Role ID.
  4. Enter a Level.
  5. Select a VPD Profile ID of FTI_DEFAULT.

    OTM ships with two VPD profiles that should be used for Logistics Network Modeling Intelligence: FTI_DEFAULT and SERVPROV. It is recommended that you only use these two VPD profiles with Logistics Network Modeling Intelligence. The Profile ID of FTI_DEFAULT should be used for the majority of users who will access Logistics Network Modeling Intelligence. Do not use the VPD profile of DEFAULT to access Logistics Network Modeling Intelligence. You can use the VPD profile of SERVPROV for OTM service provider (SERVPROV) users.
    • FTI_DEFAULT: This VPD Profile is applicable for all Logistics Network Modeling Intelligence users who are not service providers in OTM. This includes all the external predicates available in the existing DEFAULT OTM VPD profile and the new external predicates specific to the Logistics Network Modeling Intelligence solutionâs Historical Database tables.

      You can create your own VPD Profile ID for Logistics Network Modeling Intelligence as long as it includes the appropriate Logistics Network Modeling Intelligence tables.
    • SERVPROV: This VPD Profile is applicable for all Logistics Network Modeling Intelligence users who are also service providers in OTM. This includes all the external predicates available in the existing SERVPROV OTM VPD profile and the new external predicates specific to the Logistics Network Modeling Intelligence solutionâs Historical Database tables.
  6. And finally, make sure you have an Access Control, User Role Grants, and User Grants set for the user role as appropriate.

    For more details on setting access control, see the Access Control ListÂ help topic.
  7. Click Finished.

Also see User Role for more details.

Editing Existing OTM User

Now that you have created the user role, you can edit an existing OTM user to provide access to Logistics Network Modeling Intelligence dashboard.

To allow a user to access Logistics Network Modeling Intelligence, complete the following:

  1. Go to Configuration and Administration > User Management > User Manager.
  2. Click Search and edit an existing user.
  3. Add a User Role ID with the appropriate VPD Profile ID and Access Control.
  4. In the Business Intelligence Applications grid, enter an Application ID of LNM and click Save.
  5. In the Business Intelligence Roles grid, remove the default Business Intelligence Role of NO ACCESS.
  6. Enter a Business Intelligence Role that provides access to Logistics Network Modeling Intelligence and click Save.
  7. Click Finished.

For more details, see Manage User.

Loading Data

To load and unload scenario data to Logistics Network Modeling, use the modeling scenario actions of Send Data to Analytics and Unload Data from Analytics.
