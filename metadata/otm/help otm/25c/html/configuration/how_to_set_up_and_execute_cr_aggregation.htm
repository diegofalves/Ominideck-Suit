How to Set Up and Run Cooperative Routing Aggregation

Following is a process flow of navigating the cooperative routing (CR) process to run the aggregation and the CR solver. This covers several different pages.

  1. Bring in the Shipment Data:
    1. Bring shipment data in via integration or import.
    2. Open the CR shipment page to view/modify/delete shipment data.
  2. Create Shipment Sets:
    1. Using the Shipment Set Criteria page, create shipment set queries.
    2. Launch the shipment set assignment process as in the existing Oracle Transportation Sourcing business process.
  3. Run the Aggregation:
    1. Open the Project page to define the aggregation parameters.
    2. In the Project page, select the action Launch Aggregation.
    3. View the aggregation results. You can also select the SmartLinks View Aggregation and View Forecasts to view the lane volume aggregates and the lane volume forecasts.
  4. Prepare to Run the CR Solver:
    1. Open the Lane Volume Forecast page and modify or create forecasts as needed.
    2. Open the CR Fleet page and modify or create fleets as needed.
    3. Open the CR Scenario page to define the scenario.
  5. Run the CR Solver and Manage the Results:
    1. Open the CR Scenario page and run the CR scenario action Launch CR Solver.
    2. Open the CR Solution page to view the solver results.
    3. From the results page, click the SmartLink to view related routes.
    4. In the CR Route page, view the routes. After making any necessary edits, commit those you want to keep.
    5. Open the Scenario Lane Volume Forecast Commitments page to review the scenario commitments.
    6. When finished modifying and committing the routes, open the Scenario Summary page to view the overall details of the solution.
    7. From the Scenario Summary, you can choose the scenario as the chosen solution for the project.
  6. Create route templates from the CR Route:
    1. Select the desired CR Route.
    2. Click the action Convert to Route Template.
