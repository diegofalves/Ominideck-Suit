How to Define a Default Dashboard in Transportation Intelligence and Logistics Network Modeling Intelligence

Once your new Oracle Transportation Management (OTM) user has the appropriate access to Transportation Intelligence (TI) or LNM, Â you can assign them a new default dashboard. Refer to How to Provide Access to Transportation Intelligence for a User or How to Provide Access to Logistics Network Modeling Intelligence for a User.

The following steps explain how to define a default dashboard in user preferences.

Selecting a Default Dashboard

To assign a dashboard to a user, first you create or edit a user preference.

  1. To access user preferences, click Configuration and Administration > Preferences > User Preferences.
  2. Click New to create a new user preference.
  3. Enter a User Preference ID.
  4. Select a Domain Name in which you want to save this user preference.
  5. In the Preference section, select Dashboard Preference from the Name drop-down list. Refer to the help topic Business Intelligence Dashboard Preference to know how to create new dashboard preferences.
  6. From the Value drop-down list, select the dashboard that you want to display as your default dashboard when you click Transportation Intelligence > Dashboard.

    Values include the following dashboards which you might want to set as your default dashboard:
    • CPG Main Dashboard
    • LSP Customer Dashboard (should only be selected along with a Transportation Intelligence Role of LSP CUSTOMER). LSP stands for logistics service provider.
    • LSP Host Dashboard (should only be selected along with a Transportation Intelligence Role of LSP HOST)
    • Main Dashboard
    • Network Dashboard
    • Servprov Main Dashboard (should only be selected along with a Transportation Intelligence Role of SERVPROV)
    • Logistics Network Modeling Dashboard (only for users with a Business Intelligence application ID of LNM)

    The following additional dashboards are all used in other places within TI and will not typically be set as a default dashboard:

    • Carrier Scorecard Dashboard (Not typically used as a default dashboard. This dashboard is accessed via the OTM Shipment actions.)
    • System Dashboard (This cannot be set as the default dashboard. It must be viewed when logged in as DBA.ADMIN.)
    • Tender Dashboard (Not typically used as a default dashboard. This dashboard is accessed via the OTM Shipment actions.)
    • Transit Time Variability (Not typically used as a default dashboard. This dashboard is accessed via the OTM Shipment actions.)
  7. Click Save.
  8. Click Finished.

Assigning the User Preference

Now that you have created the new user preference, you need to assign it to your user.

  1. Go to Configuration and Administration > User Configuration > Manage User Access.
  2. Select a User Access Type of User Preference.
  3. Select the User ID of the user to which you want to apply this user preference.
  4. Click the Edit User Access button.
  5. Click the Replace Existing User Preference With The Following checkbox.
  6. Enter the name of the User Preference that you just created.
  7. Click Finished.
  8. The user for which you set this user preference will need to log out and back into OTM.

Now, when the user clicks Transportation Intelligence > Dashboard, they will see whatever dashboard you selected as the dashboard.

Several of the TI pre-defined dashboards require specific business intelligence roles or OTM user access to be viewed. Those dashboards requiring special setup are LSP Customer and Host Dashboards, Servprov Main Dashboard, and System Dashboard. For more details on these dashboards, refer to the topic TI/LNM/GTI Pre-defined Dashboards.
