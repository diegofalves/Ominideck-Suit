Add Shipment Tracking Event

This page is accessed via:

  • Shipment Management > Shipment Management > Buy Shipments > Actions > Shipment Management > Events > Add Shipment Tracking Event

Shipment tracking events describe activity on shipments. Enter shipment tracking events to inform planners, shippers, order owners, and other involved parties about actions related to a shipment or shipment group.

A faster way to add a shipment tracking event to a shipment is by entering a Quick Event. Quick Events add a status and reason for a particular stop on a shipment. Enter a Quick Event through the shipment's SmartLink.

Depending on what event type you enter, Oracle Transportation Management changes shipment status, shipment dates, and raises events for the shipment.

Note: You can determine if the shipment must have SECURE RESOURCES and be ENROUTE before shipment events can be recorded against shipments using the property glog.shipment.event.checkstatus.

Note: Tracking events must be matched to their related shipments. This is true even when you run the Add Shipment Tracking Event action on a selected shipment. To automatically match the entered tracking event to the shipment you have two options:

1. Set up a parameter set with Auto Match to Object ID(s) set to TRUE and assign it to a domain via domain settings. This will match the shipment to the tracking event based on ID.

2. Configure an automation agent listening for the event TRACKING EVENT - PROCESSING REQUEST and use the Match Shipment agent action configured based on your desired matching method e.g., By Shipment ID, By Matched Equipment, By Saved Query.

Equipment

If your event refers to a specific piece of Shipment Equipment, click New Equipment to enter it.

Reference Numbers

You can specify a Qualifier and Reference Number, if desired. The Qualifier identifies the type of reference number that will be used. Valid qualifiers generally follow some kind of standard, such as EDI standards, and are defined in Power Data. The Reference number is a user defined number that can be used to reference this piece of shipment equipment.

Event Details

  1. The Quick Code field provides an alternate short cut for entering status and reason codes. By merely entering a configured Quick Code, the Responsible Party, Status and Reason are automatically populated. Quick Codes are also configured in Status Messages Power Data.
  2. The Responsible Party describes the person that has entered the event. The values that appear in this field are determined by the Responsible Party Profile that is assigned to this action using the Audit Action page.
  3. By default, Oracle Transportation Management provides a Responsible Party Profile called Carrier with a Responsible Party of Carrier that is pre-configured with all available public status and reason codes. Once you select a Responsible Party, the Status and Reason fields will have specific values to choose from.
  4. Select a Status to describe the event. An event status is a grouping for event status codes. After you choose a status, various status codes appear.
  5. An event Reason is a grouping for event reason codes. Choose a reason and various Reason Codes appear.

    Note: The values you can choose for Status and Reason are pre-configured in Status Messages Power Data and are based on the Responsible Party.

  6. Click Notify to send anyone an email containing the currently displayed event reason information. You will be prompted for an email address. The person who you are notifying does not need to be defined as a Contact. Separate multiple email addresses with a comma.

Event Location

  1. Shipment tracking events can take place at actual locations or interim positions along the shipment. You can either enter a location or a position, but not both. If you are entering an event for a position, enter the City, Province Code, and Country Code for where the event took place.
  2. You must enter an Event Date/Time for the event.
  3. If the event is associated with a stop, select a stop location from the Event Location list. The list displays each stop location on the shipments.

    Note: If you access this screen using the Add Event action from the Visibility menu option, you cannot associate an event with a stop location.

  4. If you are entering an event (Event Type) that is not stop-related, you can use the Location Name field to record the name of any location for the event. This is for information only and does not drive any business logic or shipment status changes. However, the Location Name that you enter appears in the Event History log in Visibility.

Communication

The Contact Name and Contact Function Code fields serve as the signature of the person who provided the shipment event. These are part of standard 214, 314, 414 EDI form.

Remarks

Use the Event Comments field to record any text that you want.

Existing Shipment Reference Numbers

Any previously recorded shipment reference number and qualifiers appear in this section.

Note: The list of the reference number qualifiers in the drop-down list is those visible to the ADMIN user of the shipment domain, regardless of the current user.

Order Data

This section displays the Order Base ID and Shipment Ship Unit ID that was used to create the shipment (if available).

Ship Unit Received Data

  1. You can record received values for any ship unit, including Weight Received or Volume Received. These values can automatically update the Received Fields on the Ship Unit pages in the Shipment Manager.
