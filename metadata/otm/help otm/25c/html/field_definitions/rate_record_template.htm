Rate Record Template

An Oracle Transportation Sourcing rate record template is an Oracle Transportation Management rate that allows you to utilize a sub-set of Oracle Transportation Management's extensive rating capabilities. A rate is considered an Oracle Transportation Sourcing rate if you select the Template checkbox on the associated rate offering.

A rate record template is used in Oracle Transportation Sourcing to populate the CRT Configuration Details page. First, you specify a rate record template ID on the CRT Configuration Header page.

Note: Rate records based on rate offerings marked as templates or read only will ONLY be used for Oracle Transportation Sourcing and Oracle Transportation Rate Maintenance. They will not be used by Oracle Transportation Management for shipment building or execution.

Creating and Using a Rate Record Template

To create an Oracle Transportation Sourcing rate record template, complete the following:

  1. Create an Oracle Transportation Management rate offering.
  2. Select the Template checkbox on the rate offering.
  3. Create a rate record based on the rate offering you just created.

    Note: You should only associate a single rate record with a rate offering.

    Note: To allow Oracle Transportation Sourcing to support the use of multiple transport modes, the transport mode specified on the rate record template is ignored. Instead, the transport modes listed in the mode profile ID on the constraint set are used. If different constraint sets are specified on the project and bid round, the constraint set ID on the bid round is used.

  4. Specify costs on this rate record using the sub-set of rate basis items and costs types supported by Oracle Transportation Sourcing.
  5. In the Rate Record Template field on the CRT Configuration: Header page, enter the rate record created above.
  6. Then, when you view the CRT Configuration Details page, the Rate Attributes section is populated based on information pulled from the rate record template and the Lane Attributes section is populated with all lane attributes regardless of what rate template is used.

    Note: Once a rate record (rate record template) is attached to a CRT Configuration, you cannot edit either the rate offering or rate record. The rate information is pulled from the rate record template upon creation of the CRT configuration and this information will not be updated if the rate record is changed. To avoid the loss of data, the rate record offering and rate record are both locked when the CRT configuration is created.

  7. You must also specify the rate record template on the project.
