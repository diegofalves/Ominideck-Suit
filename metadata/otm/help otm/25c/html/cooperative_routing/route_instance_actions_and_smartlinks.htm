Click Actions on the Route Instance page to launch the following actions. This page is accessed via Operational Planning > Cooperative Route Management > Route Instance.

Change Route Instance

Activate Route Instance

This action marks an instance as active enabling it to be used. One or more actions may be selected.

Change Service Provider

This action changes the service provider for the route instance and each of the shipments on the route instance. The shipments are re-rated with the appropriate route execution rate for the new service provider and then redriven.

Note: This action uses the domain's default parameter set, which is not necessarily the one used when the bulk plan created the route instance shipment assignments. Therefore any difference in the values of the ROUTE EXECUTION DEADHEAD RATE SERVICE ID or ROUTE EXECUTION DEADHEAD RATE DISTANCE ID parameters might make the action infeasible.

Note: This action may not be effective if the different service providers are using different rate services or rate distances.

Note: Any route instances that changed service providers will still be associated with the original route template. The original route template is still associated with the original service provider.

Deactivate Route Instance

This allows you to deactivate instances. Instances can only be deactivated when no shipments are associated with any of the legs.

Re-Calculate Planned Statistics

Planned statistics can be calculated via this user interface action or an agent action if agent actions are defined.

Re-Calculate Actual Statistics

Actual statistics can be calculated via this user interface action or an agent action if agent actions are defined.

Review Instance Details

This action opens the Review Instance Details page and allows you to view details for the instance. You can view planned and actual data along with leg and shipment details.

SmartLinks

SmartLink

Summary

View Similar Route Instances

Finds route instances similar to this route instance, if exists. Two instances are similar if their legs have the same region geography and if they have the same cyclic order (though not necessarily the same start leg). A route instance can represent a "loop", for which a single truck drives each leg in turn and then is ready to drive the same loop during the next week (which would be the next instance). If another truck is driving the same loop, but starts at a different point on the loop, its route instance is similar. Inactive instances are not returned; nor are completed instances. Find Similar Instances returns only similar instances whose start/end time on the route instance overlap with the original instance's start/end times.
