Change Route Instance

Assign Shipment

With this action, you can assign a shipment to a route instance leg. The shipment can be manually entered or chosen from a list of potential shipments. Certain route execution constraints such as region and order/shipment compatibility can be ignored when finding potential shipments or during the actual assignment.

Batch Unassign Shipments

Mark Leg as Missed

If set, this means that the leg will not have any shipments assigned to it. The leg assignment status will be set to ASSIGNMENT_MISSED.

Mark Leg as Not Assigned

This action changes the leg status from ROUTE INSTANCE LEG ASSIGNMENT_MISSED back to ROUTE INSTANCE LEG ASSIGNMENT_NOT ASSIGNED.

Validate Assignment

This action validates the assignment on the route instance and sets the deviation reason code.

This is only applicable for route instance legs that are already assigned with a shipment.
