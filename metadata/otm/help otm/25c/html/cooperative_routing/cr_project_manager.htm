Cooperative Routing

Project

This page is accessed via Cooperative Routing > Route Analysis > Project.

The Cooperative Routing Project manager defines the parameter and shipment sets to be used for an aggregation. After aggregation, it also defines the set of forecasts to be used for a cooperative routing (CR) solution.

The parameters defined by the project for aggregation include the definition of the time buckets, the rule to be used for determining the forecast count and the rules to be used to define outliers. The time bucket definition and the outlier rules are themselves distinct entities.

  1. Enter a unique ID for the project in the Project ID field.
  2. Enter a Description.
  3. Enter a Forecast Rule for generating forecasts during aggregation.
  4. Enter the Domain Name for the project.
  5. Enter the Start Date to define the start of the date range for historical shipments.
  6. Enter the End Date to define the end date for historical shipments.
  7. Enter a Cycle Definition ID which defines the periodic time cycle for the project.
  8. Enter a Distance Method ID to identify the rate distance entity which defines the distance engine. This is needed at the very least, to determine inter-region deadhead (a leg without cargo) distances. The Rate Distance is the normal way of specifying how distances are computed. By itself, it does not depend on any rate data.
  9. Enter the Forecast Adjustment (%) to indicate the percentage to adjust the forecast rule. This can be positive or negative.
  10. Optionally, enter the Forecast Adjustment (Std Dev). This indicates the multiples of standard deviation by which to adjust the forecast rule. This can be positive or negative. Note that if both adjustments are specified, the aggregator chooses the adjustment that results in the smaller forecast.
  11. If you want, select a Parameter Set ID.
  12. Optionally, enter a Scenario Solution ID to define the scenario with the solution selected for this project.

    Note: This is for information only.

  13. If you want, enter a Confidence Method to determine the method of computing the confidence factor. The choices are Historical Data or Standard Normal Distribution.
  14. Enter a Region Group ID.
  15. Enter a Service Method ID. This is the rate service used for determining distances and times.
  16. Optionally, in the Outlier Rule grid, enter the Outlier Rules associated with this project for identifying outliers during aggregation.
  17. Optionally, in the Shipment Set Criteria grid, enter the shipments sets defining the shipment data for this project.
  18. Click Finished.
