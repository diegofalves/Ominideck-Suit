Mobile Layout

This page is accessed via Configuration and Administration > User Configuration > Mobile Layout.

A mobile layout containsÂ the default look and feel of the Oracle Transportation Mobile Web Application. By copying and editing a mobile layout, you can change the number of boxes shown, the color and label of the boxes, the saved query used to fill the boxes, etc. Use the Copy Mobile Layout action to duplicate the public or other custom mobile layout. The mobile layout is configured using JSON and REST API integration. See the REST API for Business Object Resources Guide for instructions on how to make API calls. SeeÂ Mobile Metadata Expression SyntaxÂ for details on how to structure the JSON.Â 

  • If you are creating a mobile interface that your driver will use to make his deliveries, copy and then modify the copy of DEFAULT_MOBILE_LAYOUT.
  • If you are creating a mobile interface that a carrier or service provider will use to accept bids, copy and then modify the copy of SERVPROV_MOBILE_LAYOUT.Â 

Creating a Mobile Layout

Note: While you can create a new mobile layout by clicking New, it is recommended that you select a sample mobile layout and click Copy Mobile Layout.Â If you are creating a mobile interface that your driver will use to make his deliveries, copy and then modify the copy of DEFAULT. If you are creating a mobile interface that a service provider will use to accept bids, copy and then modify the copy of SERVPROV.

  1. Enter a Mobile Layout ID.
  2. Enter a NameÂ to describe the layout.
  3. Select a Domain Name.
  4. Edit the JSON text area to customize how your browser-based mobile app will appear.
  5. Click Finished.

After creating a mobile layout, select it in your user preferences in order to use it.
