Note: Only users with ADMIN status can mark data for purging. Only Database administrators can schedule and run the purge process. Detailed configuration of the purge process takes place when Oracle Transportation Management is installed. For more information, see your System Administrator.

Use this action/process to purge business objects from the database. Some objects, such as Driver Work Invoices and Devices, have separate actions, in addition to recurring process management tasks, that will purge the data from your system. Other objects are purged in process management.

Note: Templates are not deleted when you purge data.

Purging the documents will also purge the content associated with the document. The content will only be purged as long as it is not associated with any other document. Purging a document, which has link documents associated with it, will cause the link documents to be set to individual documents.

As a database administrator, you can perform purges based on the age of the data, status information, or domain. You can set up your data purge to happen immediately, during the next scheduled process, or on a specific day and time.

Purging through the user interface uses the planning logic. If you cannot delete a shipment in the user interface, you cannot purge it in the user interface either. Purging through the application server deletes the shipment and all related tables. Application server purging deletes the shipment regardless of if it is tendered, completed, etc. It is recommended that you use application server purging for old (greater than 1 year) shipments.

The property glog.purge.numberOfRecords allows you to decide the max number of records to purge.

Note: Purging permanently deletes the shipment/order data.

Note: When a database administrator perform schedule purge, OTM permanently deletes tracking events that are marked for purge and also removes mapping between tracking events and its associated objects.

When you perform the shipment purge process on the following business objects:

  • Shipment: OTM deletes the marked shipment and its associated child objects such as shipment stop, shipment ship unit, and demurrage transactions, etc. OTM also deletes tracking events on a shipment.
  • Order Release: OTM deletesÂ the marked order release and its associated child objects such as order movement, order release line, and order release stop, etc. OTM also deletes tracking events on an order release.
  • Order Base: OTM deletes the marked order base and its associated child objects such as order release, order release line, etc. OTM also deletes tracking events on an order base.

Purging Data through Process Management

  1. Go to Configuration and Administration > Process Management > Configuration and Administration > Mark for Purge.
  2. Select a data type from the Data Query Type list.
  3. When the screen refreshes, select a saved query in the Based on Saved Query field or click Define to open a window where you can develop a new query. If you do not select a saved query, the system will process the purge request with predefined qualfiers.
  4. If you selected "ORDER BASE" or "ORDER RELEASE" as the data query type, the Include Shipment checkbox is displayed. Select this checkbox if you want the shipment information associated with the order base or order release to be purged when the order base or order release is purged.
  5. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  6. Select a Log ProfileÂ to generate focused logging when reproducing an error for the specific action. You cannot select a log profile when you choose to schedule a process, as automatic generation of action logs is not supported.
  7. Select Execute, Publish, or Schedule to determine when the process will be executed.
  8. Click Submit.

Modifying the Shipment and Order Purge Process

When you submit a shipment/order purge process, the data is stored in the Oracle Transportation Management database. Oracle Transportation Management periodically reviews the database to determine which processes should run based on the data that was submitted, specifically the schedule data. You can display the data parameters for any process that you submit and change the settings or delete it (this is true for scheduled processes only).
