Generate All Auto-Pay Invoices

This page is accessed via Financials > Process Management > Generate All Auto-Pay Invoices.

The Generate All Auto-Pay Invoices process creates payment invoices directly from buy shipments in the current domain, now or at a scheduled time. This process does not generate consolidated invoices.

Auto-pay invoices can be generated only from buy shipments that have a payment status value of NEW. Also, before auto-pay invoices can be generated, the Autopay checkbox in the Service Provider Manager must be checked for one or more service providers.

Although auto-pay invoices are generated from shipments, they still need to be matched to their shipments to start the buy settlement process (approving the invoice, issuing a payment voucher, allocating the voucher, and so on). Before matching an auto-pay invoice to its shipment, you may want to send the invoice to the service provider for review and approval.

Note: If you plan to run multiple versions of this process simultaneously, make sure that none of the batches are based on the same shipments. If the batches use the same shipments, the processes will not work correctly.

From the Based on Buy Side Shipments drop-down list, select a query. The queries define the type and range (for example, by Domain Name, Service Provider ID, Location ID) of buy-side shipments from which auto pay invoices are to be generated. To create a new query, click Define, enter your search criteria and click Save. Then enter a name for the query and click Select.

Note: If you define a query based on a service provider that has not been selected for auto payment, no shipments will be found for invoicing. You may find it useful to define a query based on shipment status (for example, CONFIRMED or ISSUED).

You can select the Split Rule ID to generate single or multiple auto-pay invoices. OTM uses the split rule criteria to split the invoices. For more information, see Split Rule Criteria. You can either select the split rule or leave it blank:

  • If you provide a split rule, OTM generates multiple invoices based on the split rule.
  • If you do not provide a split rule, OTM checks the Invoice Split Rule parameter and then generates single or multiple invoices.

    Note: If you do not want to provide a split rule every time you create an invoice, you can configure a split rule by defining the 'Invoice Split Rule' parameter which enables OTM to generate single or multiple invoices.

  • If you do not provide a split rule and the parameter is not defined, OTM generates only one invoice with all the cost lines.
