Shipment Management

Create Shipment Group Process

This page is accessed via Shipment Management > Process Management > Create Shipment Group.

This process allows you to create shipment groups based on a certain rule.

Differences between Bulk Planning and this Process

Regarding the shipment group rule, some of the fields that are used in bulk plan will be ignored. In bulk plan, when shipment groups are created, OTM uses Effective and Expiration Date, Mode, Lane, and Itinerary Profile to filter out the valid shipment group rules. In this process, all these fields to filter the rules will be ignored since the input screen accepts one specific rule (there is no need to validate the rule like in the bulk plan).

The bulk plan also uses Ship Group Content Def to create groups on only a subset of the legs based on the value selected. This seems valid for bulk plan because the logic works on one order bundle or shipment graph at a time and is for one itinerary. So, there is an option to select a subset of legs from the itinerary for the shipment graph. However, in this process, the shipments from the Bulk Shipment Query could have come from various bulk plans and so there may not be one itinerary for all those shipments. Hence, selecting a subset of legs based on the Ship Group Content Def is not possible for this process. So, this is another field that is ignored. Also, OTM just uses this rule for the non-trailer build process irrespective of what is set on the shipment group type.

Like in bulk plan, this process will consider the full shipment fields as well as partial shipment checkboxes in the process of shipment group creation.

In bulk plan, some of the fields cannot be used. For example, the shipment reference number is not set on the shipment until after the bulk plan runs. Hence the bulk plan cannot use that field to create new shipment groups. However, for this process, since it is after the shipments are created and saved to the database, you have the ability to create shipment groups based on all the fields and checkboxes on the shipment group rule.

Creating a Shipment Group Process

  1. Select a Perspective of Buy or Sell.
  2. Click Go. The page will redraw with additional fields.
  3. From the Based on Shipments drop-down list, select a query. It is important to create the shipment query for the proper shipment set for this process since the shipments can be formed from various bulk plans and the bulk process logic does not assume that they are formed in a particular way or at a particular time.
  4. In the Shipment Group Rule ID, select the shipment group rule to use.
  5. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  6. Select a Log ProfileÂ to generate focused logging when reproducing an error for the specific action. You cannot select a log profile when you choose to schedule a process, as automatic generation of action logs is not supported.
  7. Select Execute, Publish, or Schedule to determine when the process will be executed.
  8. Click Submit.

The Create Shipment Group process always creates new shipment groups (based on the specified rule) for the set of shipments that were selected.

Note: The Create Shipment Group process always creates new shipment groups (based on the specified rule) for the set of shipments that were selected. In case you want to check (for each of the selected shipments) if there is an existing shipment group which conforms to the specified rule and then add the shipment to that shipment group, then follow these steps. If no such shipment group exists, then a new shipment group will be created. This is applicable to secondary charge and handling shipments.
Creating a new Agent Event and an Automation Agent:
1. Create a new agent event via the Business Process Automation > Power Data > Event Management > Agent Events page.
2. Configure an automation agent with the Agent Type as Shipment and agent event as the one created in the previous step.
a. Configure an action and select the ADD SHIPMENT TO GROUP option from the drop-down list.
b. Select the Shipment Group Type ID which is specified on the desired shipment group rule.
Configuring the Data Query:
1. On the Data Query page, Â select a Data Query Type of 'Shipment' and click Go.
2. In the Based On drop-down list, provide the saved query of shipments the same way it would be done for the Create Shipment Group process.
3. Select the agent event that was created earlier.
4. Provide other details as necessary and click Submit.
