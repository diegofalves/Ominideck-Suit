Bulk Rating

This page is accessed via Operational Planning > Process Management > Bulk Rating.

This bulk rating process provides a simple approach to selecting a set of shipments for re-rate and monitoring their progress.

A bulk rating process is requested similarly to an order bulk plan. An integration user uploads a transmission of shipments to rate and ends the transmission with a request to publish a bulk rating workflow topic. This applies a saved query to retrieve a set of shipments to rate. It then divides this set into batches for synchronous rating and routes each batch into a workflow queue. A batch may consist of one or more shipments, configurable for performance.

You can view summary results for the bulk rating during execution. Each bulk rating request is represented with a Bulk Rating ID. When a bulk rating request completes rating, the record is updated to reflect the number of shipments processed, rated, and distanced. When all batches are complete, the record is updated to show total clock time taken to process the bulk rating request.

Notification events (along with the appropriate notification stylesheets) are available for Bulk Rating. This provides a mechanism to notify a contact upon completion of the rating. The notification includes summary information from the bulk rating record.

Use the following fields to submit a bulk rating request.

  1. You can assign a Bulk Rating ID to each batch of shipments that needs rating. This ID is used to tag summary results for quick lookup and notification. If unspecified, a Bulk Rating ID is generated using the Business Number Generator.
  2. The processing of shipments within a bulk re-rate is spread across multiple threads of a batch workflow queue. If a workflow topic is assigned for each shipment, the overhead of publish/subscribe competes with any performance gain. If, on the other hand, shipments are divided in batches, evenly across the available threads, queue preemption is lost. For configuration, you may set the Batch Size per workflow topic. If omitted, it defaults to n/m where n is the number of shipments and m the number of available threads.
  3. A Shipment (both buy and sell) Query can be used to determine the set of shipments to bulk rate by choosing a Saved Query from the Based on Shipments list or click define to create a new query.
  4. If Oracle Transportation Management cannot assign a rate to the shipment, you can opt to calculate distance for it by marking the Calculate Distance checkbox.
  5. Events generated by assigning a rate to a shipment add approximately 10% to the overall processing time. If you are not interested in these events during a bulk rating request, you can suppress their generation with the Suppress SCEM checkbox. No auditing, notification, or agent triggers would occur as a result of the rating. Note, however, that the SCEM events resulting from the shipment integration are not suppressed.
  6. As each shipment is rated, distanced, or if there is a calculation failure, Oracle Transportation Management can send an INS (shipment transmission) to an external system with the result. If omitted, no outbound integrations are sent during the re-rate process. Use the Notify Method to choose the method for sending outbound integration. Also use the following settings to control the information that is sent:
    • On Rating Success: If shipment rating is successful, the process updates the status of the shipment to the provided external status value. If a remark (and external system) are supplied, an INS is sent out to the external system with the given remark.
    • On Rating Failure: External system and status specification in case rating fails and Oracle Transportation Management is not calculating distance on the shipment.
    • On Distance Success: External system and status specification in case rating fails, Oracle Transportation Management is calculating distance and the distance calculation succeeds.
    • On Distance Failure: External system and status specification in case rating fails, Oracle Transportation Management is calculating distance and the distance calculation fails.
  7. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  8. Select Execute, Publish, or Schedule to determine when the process will be executed.
  9. Click Submit.

Bulk Rating Action

For each bulk rating record, you can use the following action to display additional details about the order releases and their corresponding planned shipments.

Send Interface displays the Shipment Results page listing the shipments that were created as a result of the bulk plan.
