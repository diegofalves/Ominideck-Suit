Launch Integration

This page is accessed via:

  • Business Process Automation > Integration > Integration Manager > Upload an XML/CSV Transmission. On the Upload an XML/CSV Transmission page, click Upload and select a file. This page opens after you select the appropriate XML or CSV file.
  • Transportation Intelligence > Power Data > Transportation Intelligence > Upload Target CSV. On the Upload Target CSV page, click Upload and select a CSV file. This page opens after you select the appropriate CSV file.
  • Global Trade Intelligence > Power Data > Transportation Intelligence > Upload Target CSV. On the Upload Target CSV page, click Upload and select a CSV file. This page opens after you select the appropriate CSV file.

Note: The layout of the page varies depending on the type of file you uploaded.

CSV

Note: For Upload Target CSV page, CSV is the only option supported.

  1. In the Command list, select the appropriate transaction code.

    Note: There is no validation for any of the columns in the CSV during the Upload Target CSV process. So you must enter the correct value in the target_name column. If you misspell a target name, then a new target is created with the misspelling. It is very important that the targets uploaded are correct!

    You cannot use the D or DD transaction codes to delete existing targets. A transaction code of D or DD will only remove the target records in the OTM database, but it will not remove the records from the HD. It is not possible to remove the target records from the Transportation Intelligence (TI) database via either this page or by running the Run Targets process.

  2. The Data Directory field will default to a temporary folder on the server where your CSV file is uploaded prior to updating the database. You should leave this field as is.
  3. The Data File Name field contains the name of the file that you recently selected to upload in the Upload XML/CSV Transmission page. Unless you selected the wrong file name during that step, you should leave this field as is.
  4. The Run SQL Loader field is a drop-down list Â that contains two values. One value is - runsqlloader, the second is an option to leave this field blank. The default value is to leave this field blank, and it should be left as is.
  5. The XValidate drop-down list allows you to turn on or off verbose diagnostic messaging. To leave messaging on, set the value in the drop-down list to Y, which is the default. When you set XValidate to Y, it also prevents values in the database from being wiped out if there are null values in the CSV. Setting the XValidate to N may wipe out values in the database or create an error if the CSV file is missing the value.
  6. In the Encoding list, select the appropriate encoding type for your CSV file. If your file contains standard ASCII characters, then it can be encoded as ISO-8859-1. If it contains non-standard and international characters, then it should be encoded as UTF-8.

  7. Enter a From Domain and a To Domain when you are copying data from one domain to another. When you copy a table between domains, make sure you copy all the dependant tables as well.

XML

After an XML file is uploaded, the content of the XML is displayed. No further action is required.
