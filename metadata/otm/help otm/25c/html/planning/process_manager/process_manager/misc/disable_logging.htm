Disable Logging

This page is accessed via Configuration and Administration > Process Management > Disable Logging.

Excessive logging causes performance issues, so this lets you suppress logging for all Log IDs at or above a level of frequency (from Rare onward). Executing the Disable Logging process does not suppress Exception logging. It also does nothing unless the AUTOMATIC LOG SUPPRESSION optional feature is enabled. By default, there is a staged Disable Logging process scheduled to run every 30 minutes, for logs with a frequency of 100 per Second. This process runs at startup to avoid excessive logging at startup. The LogSuppression log ID tracks when logs are disabled. This can suppress any system logging turned on for diagnostics.

Logging suppressed via Disable Logging can be enabled by:

  • Editing a Log File record. When AUTOMATIC LOG SUPPRESSION is enabled, modifying a log file record automatically removes the log suppression for all IDs in that log file. Opening a log file and clicking Finish will enable the IDs listed in the log. When you edit a log, the log is enabled until the value in the Enable Until field on the log, which allows you to temporarily override the scheduled process which disables logging.
  • Unsuppressing Log IDs in the Logging Overview screen. These IDs may be disabled when the next Disable Logging recurring process runs.
  • Opting out of the AUTOMATIC LOG SUPPRESSION optional feature. This effectively enables all logging defined in Log Files.

Disabling Logs

  1. Select the Detail Level. Any log IDs with at least that level of detail will be disabled. For example, if you select "Ten per Second" then that includes the log IDs with a detail of "Hundred per Second" and "Every Millisecond". See the Log Suppression ID Frequencies section for a list of which logs fall under each category.
  2. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  3. Select a Log ProfileÂ to generate focused logging when reproducing an error for the specific action. You cannot select a log profile when you choose to schedule a process, as automatic generation of action logs is not supported.
  4. Select Execute, Publish, or Schedule to determine when the process will be executed.
  5. Click Submit.
