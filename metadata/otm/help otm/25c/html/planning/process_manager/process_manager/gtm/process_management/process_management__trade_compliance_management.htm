Process Management: Transactions

This page is accessed via Transactions > Process Management.

The Process Manager provides several options for controlling restricted party screening and transaction creation. Within each option, you can determine exactly what gets processed based on specific queries that you define and when it gets processed based on specific time intervals.

Declaration

  • Create Declarations: Use this process to create declarations from multiple trade transaction lines based on the selected declaration type.

Screening

  • Restricted Party Screening: Use this process to screen parties against the restricted party list and the restricted party exclusion list.
  • Sanctioned Territory Screening: Use this process to screen transactions and shipments against the list of sanctioned territories.

Transaction

  • Create Transactions from OTM Order Releases: Use this process to create screenable transactions from order releases and shipments.
  • Create Transaction from OTM Shipments: Use this process to create screenable transactions from order releases and shipments.

Logs

The following types of logs exist:

  • Agent: a log of each agent execution. Each record represents an agent initiation, completion, error or user-specified log instruction.
  • Agent Actions: a log of each agent action execution. Each record represents an action initiation, completion or error.
  • Integration: record events related to the integration of data, via XML schemas, between the database and external systems. These logs display the time, transmission number, code, and other statistics about integration events.
  • System: is a default log file that records the time, severity, and other data about system events such as status changes, planning actions, and rate engine actions.

Recurring Processes

Use this process to modify the schedule of any process found on the Process Manager menu. To modify a schedule, a process must have been previously executed.
