Process Management: Logistics Machine Learning

The Process Manager provides several options for automating the Logistic Machine Learning transit time estimation process. This page is accessed via Logistics Machine Learning > Process Management.

The Process Manager provides several options for controlling the batch processing of historical shipment data and sending to analytics, machine learning services, and performing training on that data. Within each option, you can determine exactly what gets processed and when it gets processed based on specific time intervals.

  • Data Extraction: Use this process to combine the functionality of the Load Data into Analytics and Export Data to Machine Learning Services actions into a single process. Or, run each process separately.
  • Load Post Training Data to into Analytics:Â Use Load Post Training Data into Analytics action to load training shipment data into analytics based on a machine learning scenario. Run the Perform Training action before you run this action.
  • Perform Prediction: Use this process to send buy shipment attributes along with the necessary scenario data to the machine learning service. Based on the request, the machine learning service provides the Predicted Transit Time for each shipment and populates the Predicted Arrival information on the last Dropoff stop of the shipment.
  • Perform Training: Use this process to tell the machine learning service to start a machine learning training run based on the data from the scenario saved query ID.
  • Retrieve Machine Learning Logs:Â Use this process to copy of the machine learning logs from Embedded Machine Learning (EML) server to OTM logs folder when a machine learning training has been completed.Â 

Logs

The following types of logs exist:

  • Agent: a log of each agent execution. Each record represents an agent initiation, completion, error or user-specified log instruction.
  • Agent Actions: a log of each agent action execution. Each record represents an action initiation, completion or error.
  • Integration: record events related to the integration of data, via XML schemas, between the database and external systems. These logs display the time, transmission number, code, and other statistics about integration events.
  • System: is a default log file that records the time, severity, and other data about system events such as status changes, planning actions, and rate engine actions.

Recurring Processes

Use this process to modify the schedule of any process found on the Process Manager menu. To modify a schedule, a process must have been previously executed.
