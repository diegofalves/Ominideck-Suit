Bulk Plan Order Movements Process

This page is accessed via:

  • Operational Planning > Process Management > Bulk Plan Order Movements
  • Operational Planning > Planning Results > Shipment Planning

Use this process to have Oracle Transportation Management automatically build buy-side or sell-side shipments from order movements in your domain. If some of the selected order movements are already planned on shipments which are not executed, those order movements will be unassigned from the shipments and routed together with the rest of the order movements.

Note: You can also initiate a bulk plan as an action.

  1. Enter a Bulk Plan ID to uniquely identify the bulk plan or leave the field blank to have Oracle Transportation Management automatically generate an ID based on default format defined in the Business Number Generator. If a bulk plan is scheduled as a recurring job with a user-entered Bulk Plan ID, a timestamp is appended to the Bulk Plan ID each time it runs. This timestamp is the standard: YYYYMMDDHHmmss. This ensures the bulk plan ID is unique each time it runs. It also allows you to query for all runs of a particular requested bulk plan ID. By sorting on the Bulk Plan XID, descending, you can quickly see all runs, from latest to oldest.
  2. Choose an order movements saved query in the Based On Order Movements field. Alternatively, click Define to create a new one (if you create a new saved query, you must exit the page and re-select the menu option to have the new saved query appear as a choice.)
  3. Choose a Parameter Set.
  4. Enter a Scenario Group for the bulk plan. You may query on bulk plans by scenario group to limit the results returned by the query.
  5. Select the What-If Scenario checkbox if you do not want shipments to be generated.
  6. Select the Use Partition checkbox to process the order set (as determined by the saved query entered in the Based on Order Movements field) based on their current partition assignments.
  7. If you do not want to use the current partition assignments, you can optionally specify a Partition Profile to plan only order movements that result from Based on Order Movements saved query AND only those order movements with a Partition ID that is in the Partition Profile, or null. The saved query defined on each partition is used to determine the partition to which an order movement belongs.
    Only the orders that result from Based on Order Releases saved query will be bulk planned. However, only the intersection from both sets will have the partition ID assigned.

    Note: The Based on Order Movements query and Partition Profile are different queries. If you have a Based on Order Movements query selected and a partition profile, OTM will look for order movements that are in both. This can affect performance as OTM will search through both sets of data.

  8. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  9. Select a Log ProfileÂ to generate focused logging when reproducing an error for the specific action. You cannot select a log profile when you choose to schedule a process, as automatic generation of action logs is not supported.
  10. Select Execute, Publish, or Schedule to determine when the process will be executed.
  11. Click Submit.Â Oracle Transportation Management displays a Bulk Plan ID that you can use to review the results.
