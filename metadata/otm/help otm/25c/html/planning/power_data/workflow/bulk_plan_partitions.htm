Partitions

This page is accessed via Business Process Automation > Power Data > Event Management > Partitions.

Partitions group orders based on saved query order criteria for bulk plan processing. For example, you can group orders for bulk planning based on criteria such as source location or region, destination, or any other attributes that can be defined in a saved query. Orders with the same Partition ID can then be processed in parallel bulk planning runs that result in efficient processing time.

Use this page to create and manage Partition IDs that you can assign to order releases and identify when you initiate a bulk plan. Partition ID assignments (to order releases) can be performed in the following ways:

  • Manually record a Partition ID on the Other Attributes tab or an order release.
  • Assign Partition IDs by Partition Profile using the Assign Partition option in Process Management.

Creating Partition IDs

Partition ID is a unique identifier for the partition. You can use this ID to make manual partition assignments on order releases. You can also use this ID to include a partition in a Partition Profile.

Saved Query ID defines a saved query that you associate with a Partition ID.

The saved query acts as a filter and determines whether an order belongs in a partition. For example, you can define a saved query that looks for orders with a specific source location, etc. If an order passes the saved query criteria, then it belongs to that partition.
