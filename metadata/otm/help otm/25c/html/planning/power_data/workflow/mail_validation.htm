Mail Validation

This page is accessed via Business Process Automation > Power Data > Mail Management > Mail Validation.

This page allows you validate that an email can be sent from the system. Various technologies and rules are checked, including SPF, DKIM, DMARC, Mail Delivery service requirements and Mail Quotas. In particular, failure to register a valid Mail Sender will block outbound mail.

If mail is not blocked as a result of the validation checks, you can test the sending of an actual SMTP mail request. Select the Send Email field on this page to generate a test email from the specified sender to the specified recipient. If the SMTP check fails (or the email is not received), note the log ID on the Mail Validation results. This can be used to send detailed diagnostics to Support and verify whether any error occurred routing the email to Mail Delivery. See Mail Validation Troubleshooting for more information.

Note:Â The mail sender or domain cannot be from a public domain such as gmail.com, hotmail.com, or yahoo.com. Use a domain local to your organization for mail senders.

  1. Enter a From email address.
  2. Enter a To email address.
  3. Select the Send Email field to send a test email.
  4. Enter an Attachment Size, simulating an attachment. The test email will create an attachment of the size you indicate to any test emails created. If you create a large attachment, the email may fail to send if it exceeds the attachment size allotted by the system.
  5. Click Validate.

Mail Validation

The system validates sending the email and creates a Mail Validation table illustrating the groups of errors and their explanations.Â For an explanation of the validation results, see Mail Validation Troubleshooting.

Note: Make a note of the Mail Log ID, and the From and To emails, and be prepared to share that information with Support. Log IDs may be reused after 2 days.

The Appending checkbox indicates that you have made multiple attempts to validate this email. Instead of a new log, the results are appended to the existing log.

The mail validation table displays the following information:

  • Group: The general grouping of the technology used for the validation.
  • Validation: The test being run.
  • Passed: Indicates that this test requires further attention, not that sending the email is going to fail.
  • Error: Additional information about the failed check.
  • Explanation: An explanation of the error with a link to a detailed cause and resolution.
