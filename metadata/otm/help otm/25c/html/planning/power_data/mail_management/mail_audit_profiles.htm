Mail Audit Profiles

This page is accessed via Business Process Automation > Power Data > Mail Management > Mail Audit Profiles.

A mail audit profile controls what email is audited and which of those emails content is persisted (saved for resend). ItÂ associates a mail audit group with a set of email statuses for persistence. Depending on how you use this page, the following scenarios are available:
Â 

  • Define just a mail audit group, and do not define any persist states. Then the system audits the mails that are associated with that group but does not persist their content.
  • Define without a mail audit group, but add at least one persist state. It then audit and persists all mail in the persist states.
  • The combination of the first two: add a mail audit group and at least one persist state. It will then audit and persist all the mails defined by that group that are in those persist states.
  • Define without a group or any persist states. In this scenario, the system audits all emails without persisting the content.

Several audit profiles exist for you to use: Â 

  • AUDIT ALL: Audits all emails. Inactive by default.
  • AUDIT FAILURES FOR RESEND: Audits and persists all emails that failed due to Quota, Sender, Suppression or Failure. Active by default.
  • AUDIT TENDERS: Audits all tender emails. Active by default.
  • AUDIT TENDERS FOR RESEND: Audits and persists all tender emails. This includes emails in a Sent status. Active by default.
  • AUDIT REPORTS: Audits all report emails. Active by default.
  • AUDIT REPORTS FOR RESEND: Audits and persists all report emails that failed due to Quota, Sender, Suppression or Failure. Active by default.

Adding a Mail Audit Profile

  1. Enter a Mail Audit Profile ID.
  2. Enter a Mail Audit Group ID.
  3. Enter a Rank. The rank is used to determine which mail audit profile is the main audit profile displayed on the Mail Audit and Saved Mail pages. This helps to identify the email. The lowest rank, closest to 0, is the one that takes precedence.
  4. Select Active.
  5. Enter a Domain.
  6. Select a Persist value and click Save for each persist value entered.
  7. Click Finished.
