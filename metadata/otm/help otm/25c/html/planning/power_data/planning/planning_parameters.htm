Parameter Sets

This page is accessed via:

  • Shipment Management > Power Data > General > Parameter Sets
  • Master Data > Power Data > General > Parameter Sets

Parameters determine how various processes behave within Oracle Transportation Management. You can use parameters to model your business processes so that Oracle Transportation Management builds shipments to your specifications. You can set various parameters to specify how to rank itineraries for shipment processing, perform order bundling, estimate rate distance, make decisions about cross-dock and pool planning, and so on.

Each parameter has specific values that can be configured to control the processing behavior mentioned above. Parameters are contained within a parameter set that is assigned to domains. A public parameter set is provided (called DEFAULT) and automatically assigned to each domain that you create. However, you can create a new parameter set to override some/all of the settings in the DEFAULT Parameter Set. The default parameter settings in the DEFAULT Parameter Set are described in the List of Parameters topic. Note that the parameter value column is nulled if you copy a default parameter set.

If you have a parameter defined in both the Default parameter set and in a personal parameter set, the value of the parameter in the personal parameter set will be used.

You can create a Parameter Set, name it what you want, and change any of the default values. The Parameter Set page displays the parameters by group in the first column and the defaults in the second. The third column shows a blank field so you can override the default or after a value is overridden the new value will display in that column. You can view just overridden values using the parameter action View Overrides. There is a Collapse All button to minimize all groups, if you only want to work with one or two groups.

After creating a parameter set, use the Domain Settings page to activate it. A parameter set can be defined at the domain. You can also define a parameter set in user preferences which would be more specific than at the domain.

Note: The elements of a parameter set can change over time. Just because the same parameter set is specified, the results can be different. If you change the configuration of a parameter set between shipment creation and shipment modification, you could get different results.

Creating Parameter Sets

You can also create a new parameter set using the Copy Parameter Set action.

  1. On the Parameter Set page, enter a unique ID for the set.
  2. Optionally, enter a Description for the set.
  3. Enter a Value for the parameter for which you want to override the default. The default displays in the second column.
  4. Click Finished when all new default values are entered.

Note: If you created a parameter set, you must assign it to a domain for it to become effective.
