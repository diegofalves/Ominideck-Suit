Shipment Management

Logic Configuration - Solution Improvement

Logic configuration solution improvement parameters are accessed via the Logic Configuration page if you select a logic configuration type of Solution Improvement.

LTL Split

Setting LTL Split parameters allows you to define constraints for the LTL splitting process.

LTL splitting is affected by the LTL SPLIT PACKING METHOD and LTL SPLIT ONLY USE EQUIPMENT ON SHIPMENTS parameters.

Solution Improvement

Use this Logic Configuration Type to configure a solution improvement that evaluates each shipment for potential improvement in cost at the end of a bulk plan run. This is done by running the shipments through some pre-defined iterations. For example, Oracle Transportation Management evaluates the cost impact of moving one order bundle from one shipment to another. If this results in cost savings while still maintaining the feasibility of both affected shipments, the change is committed. Applying several such iterations repetitively results in better quality solutions.

At the end of bulk plan, the planned shipments are evaluated for opportunities to improve solution quality, specifically, to reduce cost. Using the output of bulk plan, Oracle Transportation Management first tries to pair each shipment with all other shipments.

Note: After you create a Solution Improvement Configuration, you must assign it to the SOLUTION IMPROVEMENT CONFIG ID parameter for the parameter set that you are using for bulk planning.

Note: Currently the Solution Improvement logic does not run for Network Routing (that is, when the planning parameter ORDER ROUTING METHOD is set to Network Routing).

Parameters

The parameters are in alphabetical order and list the group they are in.

Parameter

Group

New in Version

Functional Description

LTL Split

LTL Split

Determines if splitting is allowed.

MAX PERCENTAGE OF SPLITS

LTL Split

This controls the number of splits that a bulk plan may create. It is specified as a percentage of the total number of customer pickups or deliveries.

MAX SPLITS TO AN ORDER

LTL Split

This sets the limit on how many times each order on a shipment may be split. The value you set for this parameter applies globally to all orders.

MAX UTILIZATION PERCENTAGE TO SPLIT

LTL Split

If you define this parameter, a shipment is split only if its capacity utilization is less than or equal to the maximum value defined in this field.

If this value is blank, it is treated as 100%. This allows shipments to be evaluated in LTL Split.

MAXIMUM EVALUATION DISTANCE

Solution Improvement

Calculates the distance between each order bundle's source/destination locations of one shipment and each order bundle's source/destination locations of the other shipment. If any one order bundle pair has both source to source and destination to destination distances equal to or below this value, the two shipments will be paired together and evaluated. Otherwise, they will not be considered.

MAXIMUM EVALUATION DURATION

Solution Improvement

If the difference between the start times of the two shipments is equal to or below this value, they will be paired together and evaluated. Otherwise, they will not be considered.

MIN SPLIT VOLUME

LTL Split

This prohibits splitting any order which results in any split shipment that is below the minimum volume specified.

MIN SPLIT WEIGHT

LTL Split

This prohibits splitting any order which results in any split shipment that is below the minimum weight specified.

MOVE ORDERS TO ANOTHER SHIPMENT

Solution Improvement

Removes (unassigns) an order bundle from the first shipment and adds it to the second shipment. If it is successful but the total cost of these two shipments is not less after the move, then Oracle Transportation Management tries to move the bundle on the second shipment to the first.

NUMBER OF IMPROVEMENT ITERATIONS

Solution Improvement

You can use this parameter to control the number of iterations of these steps that should be performed. After going through these steps, the shipments are changed. If you go through them again, more improvement opportunities may be found thus creating more savings. The default value is set to 1.

SEND ALL ORDERS ON A SHIPMENT DIRECT

Solution Improvement

Remove all the order bundles from the shipment and send each of them in a separate direct shipment.

Note this is processing order bundles, not individual orders. An order bundle is order releases that have the same source and destination and can be bundled together (with compatible commodities, etc.).

SEND ORDERS ON A SHIPMENT DIRECT ONE AT A TIME

Solution Improvement

Gets an order bundle, removes it from the shipment, and sends it in a direct shipment.

If it is successful but the total cost of the two shipments is not less than before, the next bundle on the shipment is tried.

STOP AT FIRST MATCH

LTL Split

If set to true, Oracle Transportation Management ends the splitting selection process upon making the first successful match. When this is not set, Oracle Transportation Management continues to search for the best (e.g., the most economical) split solution.

SWAP ORDERS ON TWO SHIPMENTS

Solution Improvement

Removes an order bundle from the first shipment and adds it to the second shipment, and removes an order bundle from the second shipment and adds it to the first shipment.
