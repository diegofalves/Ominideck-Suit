Logic Configuration - GTM Campaign Configuration

You can access logic configuration parameters for creating, approving campaigns, sending notifications, etc. via the Logic Configuration page if you select a logic configuration type of GTM Campaign Configuration.

Parameters

General

Parameter

New in Version

Description and Parameter Value

DISPLAY PRODUCT CLASSIFICATION DETAILS

20C

Specifies whether to display product classification details on the Campaign and Campaign Line pages.

Default: Yes

DISPLAY QUALIFICATION DETAILS

19.2

Indicates whether the campaign to be created is to collect the qualification details from partners. If yes, then the qualification fields will be displayed on the Campaign and Campaign Line pages.

Default: Yes

OWNER CONTACT QUALIFIER

19.2

The campaign administrator entered while performing the Create Campaign action will be copied against this party qualifier on the campaign.

Default: CAMPAIGN OWNER CONTACT

RECEIVING CONTACT QUALIFIER

19.2

You need to setup the contact details for every partner to whom notification needs to be sent. This contact detail needs to be copied to the campaign lines on campaign line creation via an automation agent and direct sequel update.

Default: SUPPLIER CONTACT

RECEIVING PARTY QUALIFIER

19.2

The partner (Party) will be copied to this party qualifier on the campaign lines when the campaign is created.

Default: SUPPLIER

SENDING LEGAL ENTITY

19.2

You can define the legal entity (Party ID) against this parameter. This will be sent to the partner receiving the notification as the legal entity who has initiated this campaign.

19.2

The Campaign ID will be created using this Business Number Type. You can identify a different BN Type for different cases or different campaign types.

Default: GTM_CAMPAIGN_XID

CREATE DATA CONFIGURATION

19.2

Identifies the data configuration containing the details to be copied Â from Trading Partner Item or Item while creating campaign.

PERSPECTIVE

19.2

Identifies the perspective of the campaign. Currently only SUPPLIER is supported.

Default: SUPPLIER

PURPOSE

21A

Identifies the purpose of the campaign. Following are the values :

ORIGIN: Creates a campaign line for a) each partner site associated and b) for each related partner item for which partner sites of the partner are not associated to the item. The trading partner should provide origin details for each campaign line.Â Note that this value is deprecated and is not supported. It is recommended that you use the value "ITEM" as the alternative.

QUALIFICATION: Creates a campaign line for each eligible trade programs applicable for the trade agreement and import classification type. The trading partner can qualify the campaign lines.

ITEM: Creates campaign lines one for each trading partner to collect general details on item. The trading partner can provide details requested against the campaign lines.

Note: Existing campaigns will work without defining any purpose. This parameter will be required to have a value in future releases.
