Logic Configuration - Resource Scheduler

Logic configuration resource scheduler parameters are accessed via the Logic Configuration page at Shipment Management > Power Data > General > Logic Configuration if you select a logic configuration type of Resource Scheduler.

Logic configurations of this type are used to specify parameters to control the behavior of the Resource Scheduling logic. Resource scheduling logic is used to create work assignments for resource schedules and their associated Resource Schedule Instances.

GENERATE RESOURCE SCHEDULE INSTANCES

General

6.4

When set to True, bulk planning will automatically generate resource schedule instances for the resource schedules associated with the bulk plan.

Even if the parameter is False, resource schedule instances that already exist (generated prior to this bulk plan) will be loaded into the bulk plan and considered for work assignment creation. When True, existing resource schedule instances will be loaded and any missing resource schedule instances for the resource schedules and time frame of the bulk plan will be generated.

Note that the parameter FLEET AWARE BULK PLAN must also be set to True for resource schedule instances loading/generation to occur. The appropriate resource schedules for the bulk plan are determined using the RESOURCE SCHEDULE PROFILE ID parameter.

RS COST TOLERANCE IN MERGE ALGORITHM

General

6.4

This allows you to enter the cost tolerance to be considered in savings calculation during a work assignment formation. When set to the default (0), work assignments are not formed if there is no cost benefit. The combined cost of all shipments in a tour should always be less than the uncombined costs of the shipments. If you set the cost tolerance value for this parameter to a positive value, it will encourage adding more shipments to the work assignment even if there is no cost benefit. This only applies when using Distance based savings option on the RS SAVINGS CALCULATION TYPE parameter.

RS DISCOUNT PERCENTAGE

Resource Scheduler Consolidation

6.4

This allows you to enter the discount percentage to be considered for distance based savings. The discount percentage is applied to the common distance value during the savings calculation of the merge algorithms when the RS SAVINGS CALCULATION TYPE parameter is defined as "Distance based Savings." The higher the discount percentage value, the greater the distance savings. Also, the probability of placing more shipments into a work assignment is greater with a higher discount percentage.

Range of values = 0 to 100%.

RS IGNORE RATE OFFERING RULE CHECKS

General

6.4

When set to true, the system skips a quick rule check before evaluating options, and evaluates the work assignment with all feasible rate options.

When set to false, adds a check before executing the time consuming drive and rating inside algorithm so as to avoid the evaluation of unwanted work assignment formations, thus enhancing the overall performance.

RS MODE

RS PAIRWISE CONSOLIDATION

Resource Scheduler Consolidation

21A

Within the work assignment sequential savings merge algorithm, this provides the option of adding shipment pairs to the work assignment string, instead of individual shipments. Scenarios that typically perform out and back loops where each loop is two shipments can benefit form this. It can prevent the need for an empty leg back to the resource destination location at the end of the work assignment.

When this is set to true, a two pass merge approach will be used when putting shipments together onto a work assignment. The first pass will only look to create strings of two shipments. This first pass will use all the other logic configuration settings to guide the algorithm, but will stop each string at two shipments and move on to the next string.

The second pass will take in the strings created in the first pass and go through the merge algorithm to create larger strings of shipments for work assignments. Note that coming out of the first pass there may be some strings that contain only one shipment while others will have successfully combined into two shipment strings. All of these shipments will be sent into the second pass for further merge consideration.

Fleet operations that typically perform out and back loops where each loop is two shipments would benefit from this enhancement. It can minimize the need for an empty leg back to the resource destination location at the end of the work assignment as well as further optimize the pairing of outbound shipments with inbound shipments.

If set to false,the normal one pass merge algorithm will be used.

RS PAIRWISE RESOURCE MULTIPLIER

Resource Scheduler Consolidation

21C

This parameter is only considered when the parameter RS PAIRWISE CONSOLIDATION is set to true.

Increasing the value of the parameter RS PAIRWISE RESOURCE MULTIPLIER will artificially increase the resource availability for the first pass (pairing pass) in the two pass process used to create work assignments when using pairwise consolidation. The default value is 1.0.

For example, setting it to two will double the number of resources available. The value of the parameter can also be lowered if there is a desire to decrease the resource availability for the pairing pass. Â 

Note: Setting this parameter to zero will turn off the pairwise consolidation.

If out and back pairs and also some unpaired shipments are desired on work assignments, then the parameter value should be set to the average number of out and back pairs that are expected across all work assignments.

OTM recommends to set this parameter value to the largest number of out and back shipment pairs that can be assigned to any one work assignment if only out and back pairs are desired on any work assignment.

RS SEQUENTIAL SAVINGS SORT METHOD

Resource Scheduler Consolidation

21A

This allows you to choose which method to use in determining the first shipment from which to build a work assignment. You can guide the algorithm to build work assignments that are more likely to contain the types of shipments desired by the resources associated with the work assignments. The valid values are:

"0. None" is the default. With this option, the first shipment to merge is chosen based on an internally generated ID, essentially random.

"1. Earliest start time ascending" will choose the shipment with the earliest possible start time,Â  not their current start time as the shipment around which to build a work assignment. This setting replaces the propertyÂ glog.business.workAssignment.sortBasedOnShipmentStartTime.

"2. Weight descending" will choose the shipment with the heaviest weight as the shipment around which to build a work assignment. This setting replaces the property glog.business.workAssignment.sortBasedOnShipmentWeight.

"3. Duration descending" will choose the shipment with the longest duration as the shipment around which to build a work assignment.

"4. Cost descending" will choose the shipment with the highest cost as the shipment around which to build a work assignment.

"5. Savings descending" will choose the shipment pair which offers the most savings as the shipments around which to build a work assignment.

"6. Current start time ascending" will sort shipments by their start times to determine which shipment to first consider for merges with other shipments, and assignment to resource schedules .

The following glog.business properties will be in effect when this parameter is set to the default of "0. None."

glog.business.workAssignment.sortBasedOnShipmentCost

glog.business.workAssignment.sortBasedOnShipmentStartTime,

glog.business.workAssignment.sortBasedOnShipmentWeight.
RS USE PROJECTED TOUR Resource Scheduler Consolidation 22C

When set to true, the work assignment building process will start by stringing together any shipment which has a common projected tour name into one string. The initial sequence of these shipments in the string will be determined by the projected tour sequence number associated with each shipment.

By default this parameter is set to false.

RS WAIT TIME PENALTY

Resource Scheduler Consolidation

6.4

Allows you to enter the wait time penalty (range: 0 to 100) to be considered during the work assignment formation. The default value is 0, which means no penalty. The purpose of this penalty is to avoid building work assignment with large wait times in between shipments.
WA BETWEEN SHIPMENT DISTANCE COST MULTIPLIER Resource Scheduler Consolidation 24A

This allows you to control the cost impact of the distance traveled between shipments on a work assignment. This cost is used only to influence the work assignment optimization algorithm. This is not an actual cost on any shipment.Â The value of this parameter is multipliedÂ by the distance between shipments and used as a cost. For example, setting of 1 results in a cost of 20 USD for 20 miles of distance between shipments. Similarly, setting of 2 results in a cost of 40 USD for 20 miles of distance between shipments.

By default, this parameter is set to 1.

WA MAX DURATION OF TOUR

General

6.4

This allows you to enter the maximum time duration allowed for a work assignment.

The WA MAX DURATION OF TOUR parameter is used as a quick check inside the work assignment optimizer when using the sequential savings algorithm. It is used to rule out shipment combinations that have no chance of being feasible with any resource schedule instance. A string of shipments exceeding the parameter value will not be evaluated for feasibility with any resource schedule instance.

WA USE NFR STOPS BETWEEN SHIPMENTS Resource Scheduler Consolidation 24B

This parameter is used to insert an Non Freight Related (NFR) stops between shipments on a Work Assignment (WA). When the parameter is set to true, the application will add the NFR stop which connects the two shipmentsÂ within the optimization process such that the rating of shipments within the WA includes the NFR stops.

When the parameter is set to false, the application will add theÂ NFR stop at the start of a WA shipment if the first stop location for the shipment is not the same as the last stop of the previous shipment.Â The NFR stop location will be the same as that of the last stop of the previous shipment. If you remove the shipment from a WA, then the NFR stop will automatically be removed.

By default, the parameter is set to false.

Resource Scheduler Consolidation

Note: You can change this parameter to negative values, and setting a negative valueÂ will cause the optimization algorithm to minimize the number of resources used when creating work assignments.
