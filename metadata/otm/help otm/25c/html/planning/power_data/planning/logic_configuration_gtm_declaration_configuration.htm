Logic Configuration - GTM Declaration Configuration

You can access logic configuration parameters for creating declarations via the Logic Configuration page if you select a logic configuration type of GTM Declaration Configuration.

Use this page to configure logic to group and then aggregate similar transaction lines into declarations based on specific consolidation and aggregation constraint sets.

Note: If no grouping criteria are specified, all transaction lines are grouped together into one declaration.
If no aggregation criteria are specified, each transaction line will create one declaration line.

You can specify data copy configurations in this page. Depending on this configuration, data will be copied from the source object to the target object when you perform the Create/Add to Existing Declarations action.

This page also allows you to configure parameters related with the generation of declaration messages.

These parameters can also be used for performing the actionÂ Perform Entry/Exit Recording.

Parameters

General

The parameters in this section are used to set rules around how GTM groups, splits, and aggregates corresponding lines into declarations and declaration lines. In addition, these parameters set the base data such as qualifiers to be used in the logic.

Message Configuration

The parameters in this section are used to configure certain data prior to a declaration being sent to a regulatory agency or partner for filing. This data includes which qualifier should be used to identify the filer, transmitter, sender and receiver.

Parameter

New in Version

Description and Parameter Value

ACTION - DECLARATION REMARK QUALIFIER

6.4.2

This specifies the remark qualifier to be used to populate the action that created the declaration message.

Default: DECLARATION ACTION

FILER - INVOLVED PARTY QUALIFIER

6.4.2

This specifies the filer party.

Default: FILER

OUT XML PROFILE
