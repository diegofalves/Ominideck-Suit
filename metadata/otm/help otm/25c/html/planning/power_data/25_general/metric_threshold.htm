Metric Thresholds

This page is accessed via Configuration and Administration > Metric Alert Management > Metric Thresholds.

Note: Your user must haveÂ the DBA.ADMIN user role to use this functionality.

Create metric thresholds to establish when a metric notification will be triggered. You can create thresholds based on Metric Cumulative Criterion and Metric Current Criterion.

  • In a Cumulative threshold, you compare one historical value to another (or to a fixed value). Historical values are updated hourly by the Persist Metrics recurring process and all cumulative thresholds are checked.
  • In a Current threshold, you compare current values to a historical or fixed value. They hold the current hour's average, maximum, and count along with any new value to accumulate into statistics. When a current threshold compares to a historical value, it uses the historical calculations last calculated by Persist Metrics.

An audit of all triggered thresholds is available in Metric Threshold Results.

Provided Metric Thresholds

Oracle stages the following records:

  • OBJECT LOCK DEADLOCK: This threshold triggers when the system detects a deadlock between agents. This automatically collects the OPEN PROCESSES diagnostic collector to identify the source of the deadlock. These data can be retrieved via Metric Threshold Results.

Adding a Metric Threshold

  1. Enter a Threshold ID.
  2. Select a Domain Name.
  3. Select Active.
  4. Enter a Metric Type ID.
  5. Enter a Current Criterion ID.
  6. Enter a Notification Profile ID.
  7. Enter a Metric Subcomponent. Use the Search icon to locate valid subcomponents for this metric.
  8. Enter a Cumulative Criterion ID.
  9. Click Finished.
