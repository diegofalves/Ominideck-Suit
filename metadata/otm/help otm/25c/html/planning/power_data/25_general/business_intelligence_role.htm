Business Intelligence Roles for Transportation Intelligence and Global Trade Intelligence

You cannot access this page from the standard Oracle Transportation Management menus.

Note: You should never modify the existing business intelligence roles.

Business intelligence roles determine user access for Transportation Intelligence (TI) and Global Trade Intelligence (GTI). Roles are then associated with users via the User page. By default, users have no access (the business intelligence role is NO ACCESS), so users must be given rights to access Transportation Intelligence and Global Trade Intelligence.

In addition to the Business Intelligence Roles, you must select the appropriate user role. The user should have the appropriate VPD profile as follows:

  • For TI: FTI_DEFAULT or SERVPROV
  • For LNM: FTI_DEFAULT
  • For GTI: GTI_DEFAULT

Note: TI and GTI only use the default user role associated to the user logged into OTM as detailed above. TI and GTI do not support dynamically changing the user role via the User Role link in the Settings and Actions page.

Pre-defined Business Intelligence Roles

The following business intelligence roles are pre-defined:
  • BIAdministrator: This role mimics the OAS role of the same name. This role can be used for TI, GTI. This new role replaces the legacy TI BI role of PRESENTATION SERVER ADMINISTRATOR.
  • BIAuthor: This role mimics the OAS role of the same name. This role can be used for TI, GTI. This new role replaces the legacy TI BI role of DEVELOPER.
  • BIConsumer: This role mimics the OAS role of the same name. This role can be used for TI, GTI. This new role replaces the legacy TI BI roles of EXECUTIVE and OPERATIONAL.
  • KPIMANAGER: This the only role that can be used to edit target values of metrics from dashboard. This role must be assigned together with one of the other roles. For example, use the BIAuthor or BIConsumer role to give the user access view and/or edit the default dashboard while the KPIMANAGER role provides the ability to edit targets.
  • SERVPROV: This role should only be used to access the Servprov Main Dashboard. This role is only used for TI.
  • LSP HOST: This is the only role that can be used with the LSP Host dashboard. Also this role is incompatible with any of the other dashboards listed in the table below. This role is only used for TI.
  • LSP CUSTOMER: This is the only role that can be used with the LSP Customer dashboard. Also this role is incompatible with any of the other dashboards listed in the table below. This role is only used for TI.
  • NO ACCESS: A user with a business intelligence role of No Access will not be able to access any dashboard.

Full Control, Read, No Access, and Access are OAS terms.

  • Full control: allows you to edit the dashboard and save the changes.
  • Read: equivalent to read-only.
  • No access: you cannot even view the dashboard. You will receive an error.
  • Access: Only used for Ad-hoc Query and provides user full access to use the Transportation Intelligence > Ad Hoc Query menu item.

Note 1: While these dashboards can be assigned to specific users, they are also available from Oracle Transportation Management actions.

  • Carrier Scorecard Dashboard is available from the shipment action Change Service Provider > Analyze Carrier Scorecard. Click Carrier Analysis.
  • Tender Dashboard is available from the shipment action Tender > Analyze Tender Performance.
  • Transit Time Variability Dashboard is available from the shipment action Shipment Management > Change Shipment Details > Change Schedule.

Note 2: The System Dashboard cannot be assigned to a user. You will see an access denied error. To access the System Dashboard, log in to Oracle Transportation Management as DBA.ADMIN. Then, go to Transportation Intelligence > Data Management. The Load Log and Load Rejects are all tabs on the System Dashboard.

Note 3: For the LSP Customer Dashboard, only users with the Business Intelligence Role of LSP CUSTOMER will be able to access it.

Note 4: For the LSP Host Dashboard, only users with the Business Intelligence Role of LSP HOST will be able to access it.

Note 5: If you log in as SERVPROV.admin or any other service provider, you will only be able to access the Servprov dashboard regardless of user preferences and permissions in OAS.

Note 6: This is Ad Hoc Query in the Transportation Intelligence menu.

Note 7: This is Alerts in the Transportation Intelligence menu.

Note 8: This is only available in the Transportation Intelligence menu when logged into Oracle Transportation Management as DBA.ADMIN.

Note 9: This is the only GTI pre-defined dashboard. It is not an TI dashboard. A single OTM user cannot view both the TI and GTI dashboards. To view both the TI and GTI dashboards, create one user with access to TI and set their dashboard preference to point to the TI dashboard. Then, create a second user with access to GTI and set their dashboard preference to point to the GTI dashboard.

Note 10: The write-back dashboards consist of reports that allow you to change values, insert new, and delete existing targets, forecasts and industry benchmarks as needed. They can only be accessed directly from the pre-defined dashboards and are not available on any OTM or TI menus. See the write-back dashboards for complete details.

Note 11: To insert targets, forecasts or benchmarks, you must have the BI Role of KPIMANAGER in addition to existing BI role(s). Without the KPIMANAGER role you can see the appropriate insert dashboard, but you cannot actually enter any data into the Target Value, Target Description, and Target Final Remarks fields. If you do not have the KPIMANAGER role, you can view the KPI but Update is not available.
