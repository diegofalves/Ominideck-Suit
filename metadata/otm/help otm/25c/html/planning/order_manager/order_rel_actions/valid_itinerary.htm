Show Routing Options

Note: If you selected the action Show Network Routing Options - Ocean, see the help topic Show Network Routing Options - Ocean.

This action is accessed via Order Release actions.

Use this action when you want to create a shipment (buy or sell), and want to review and assign an itinerary from a valid list generated by Oracle Transportation Management (OTM).

There are several stages and options associated with using this action, all of which are outlined below. The process begins by selecting an Order Release and using the action Show Routing Options under Create Buy Shipment or Create Sell Shipment.

If the orders have different priorities, OTM will not let the bundling go through.

The parameter NUM OF DAYS ADDED TO LD FOR OR PLANNED ON MULTILEG can be used to have OTM add this number of days to an order release's late delivery date to search for a late, infeasible rate service solution. This parameter controls just how late will be allowed. This parameter is only used for legacy order release planning (prior to 6.4.2); for Show Routing Options to return an infeasible result. The default is 0.

Order Movements

You can also configure Oracle Transportation Management to automatically create order movements when planning order releases using the Auto Create Order Movements parameter.

If you use the parameter and want to unassign an order from a shipment, you can also use the Unassign and Delete Order Movements action so the appropriate order movements are removed as well.

Ignore the Following Constraints

First you will need to set constraints to ignore.

  1. Select the constraints to ignore.
  2. Click OK.

Routing Constraints

  1. Select routing constraints. A page appears that allows you to limit the valid itineraries returned by OTM to just those that have matching values to what you enter. Constraint options include:
    • Transport Mode and Mode Profile
    • Service Provider and Service Provider Profile
    • Equipment Group and Equipment Group Profile
    • Primary Leg Source Location and Primary Leg Source Location Profile
    • Primary Leg Dest Location and Primary Leg Dest Location Profile
    • Port of Load and Port of Load Profile
    • Port of Discharge and Port of Discharge Profile
  2. If you want OTM to return all valid itineraries except certain ones, then enter that value in the appropriate field and check the corresponding exclude checkbox. If you do not enter any constraints, all valid itineraries are returned.
  3. When complete, click the show options button.

Valid Itineraries

Note: Itineraries which are time infeasible are also returned. They are indicated as such by an hour glass icon with a red "x".

A Valid Itineraries page appears that displays all the valid itineraries based on standard search criteria and any constraints you identified in the previous step. This is a list of possible candidates for building a shipment for the order release. To view details about each itinerary, click the appropriate itinerary ID.

Some of the itineraries on the list still may not build a feasible shipment. Since it is not optimal to put non-valid itineraries through the planning logic, it only shows itineraries that may work, but still may not pass through the planning logic criteria.

If there are more than four locations in a route, OTM displays "..." between the first two locations and the last two locations.

  1. If you know what itinerary you want, and you know that it is feasible, you can select it and click optimize to build the shipment with that itinerary.
  2. If you want to see if an itinerary can build a shipment, select the itineraries you are interested in, or leave them all unselected for them all to run, and click the Show Options button. This brings up the Show Routing Options page.
  3. You can also click the Refine Constraints button to go back to the previous Routing Constraints page in order to edit routing constraints.

Show Routing Options

All the valid itineraries from the previous page appear in the Valid Itineraries page. Any itineraries that could not be built into feasible shipments are labeled accordingly.

All itineraries that can be built into feasible shipments are listed in order of cost and show shipment start and end times. You can drill down to see shipment details.

  1. Select the itinerary that you want to use to create the shipment, and click Ok.

    Note: If the order release you are working with already belongs to a shipment, it is removed from that shipment and resent through the planning logic. Potentially, it can be put onto a new shipment.

  2. Click the Refine Options button to further specify the rate record, rate offering, and equipment groups combination for the shipment.
  3. Click Itineraries to go to the previous Valid Itineraries page to view valid itineraries.

Refine Options

Use the Refine Options page to specify the exact cost, rate offering, rate record, and equipment group used to build a shipment. You may want to use this page to select a carrier that is more expensive, but can handle your shipment on fewer pieces of equipment.

  1. Select an option to sort the data, such as by transit time instead of by cost.
  2. Select the combination that best suits the needs of your shipment.
  3. Click Save to build the shipment.
