Shipment Management

Manual Repack Equipment Detail

This page is accessed via Shipment Management > Shipment Management > Buy Shipments > Actions > Shipment Management > Change Shipment Details > Manual Repack Equipment and then click Repack.

Validations

  • The new ship unit count must be more than the existing shipment ship unit count.
  • The sum of ship unit count split in the grid must be less than or equal to the new ship unit count.
  • If you select, the Assign to Equipment option from the Action Type drop-down list, the equipment ID and compartment number are mandatory. In case of a single compartment shipment, this will default to 1.
  • If you enter a new ship unit count, the Action Type drop-down list will only show the "Assign to Equipment" option.
  • For repacked ship unit, if ship unit count is null, OTM will not allow you to increase ship unit count. Ship unit count should always be one.
  • For bulk ship unit, if you split the ship unit, OTM will not allow you to increase its count. Ship unit count should always be one.

Additional Validations

For reuse shipment equipment, there are on upstream and/or downstream shipment, moving from reuse shipment equipment to another new shipment equipment or a new shipment equipment, additional validation needs to be done:

  • When you move ship units from one existing shipment equipment to other shipment equipment, these two shipment equipment must have the same Travel Path Number.
  • If there are two shipment equipment, these two equipment must travel the same path in order to move ship units between them.
  • New shipment equipment has a travel path number zero, as soon as a ship unit is moved into it; the new shipment equipment inherits the travel path number from the ship unit. When a new shipment equipment becomes empty, its Travel Path Number becomes zero.
  • Existing shipment equipment retains its travel path number regardless it is empty or not.

On the Manual Repack Equipment Detail page:

You can update the ship unit count on the original shipment ship unit. You can only increase the total number of ship unit in New Ship Unit Count. If you want to decrease the ship unit count, select the Delete or Move to Order Movement action type.

Â Under the Repack Instructions section, for the selected ship unit, you can enter number of Ship Unit Count.

Select the following options from the Action Type drop-down list.

  • The Assign to Equipment option allows you to repack ship units to available equipment.
  • The Delete option allows you to delete ship units.
  • The Move to Order Movement option moves part of ship units to order movement.

If you want to split a certain ship unit count to equipment, the Shipment Equipment ID along with Compartment Number drop-down list appears. If you want to Delete or Move to Order Movement, the equipment columns will not be shown on the Repack Instructions section.

When you finish specifying the changes, OTM will redirect to the Manual Repack Equipment Input page, which will now be refreshed with the changes specified. The unassigned ship units will be displayed in another grid.

Note: Container optimization is not called for this action, the system uses a different criteria to determine whether you should generate shipment ship unit equipment reference unit records for merged shipment ship units. 1) For a shipment with shipment ship unit equipment reference units, when you perform the Manual Repack Equipment action, the system needs to make sure the shipment ship unit equipment reference unit records are copied to the split shipment ship units if the split to shipment ship units are still on the shipment. 2)Â For a shipment with shipment ship unit equipment reference units, the system makes sure the shipment ship unit equipment reference units records are removed if the shipment ship unit is on an on unplanned order movement. Unless the Fixed ERU Count is true, the system will keep the shipment ship unit equipment reference units for the unplanned order movements.
