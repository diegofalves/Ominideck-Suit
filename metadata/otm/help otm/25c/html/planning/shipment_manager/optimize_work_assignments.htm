Optimize Work Assignments Action

This page is accessed via Shipment Management > Shipment Management > Buy Shipments > Actions > Fleet Management > Manage Work Assignment > Optimize Work Assignments.

This action creates work assignments for multiple shipments. To execute this process, you must select multiple shipments and a parameter set. This process considers the resource scheduler instances parameters configured in the parameter set. Resource scheduling logic is used to create work assignments for resource schedules and their associated Resource Schedule Instances. The resource schedule instances are determined by the logic configuration parameter RESOURCE SCHEDULE PROFILE ID.

The following shipments are not considered in the Optimize Work Assignments action:

  • Sell Shipments
  • Shipments with different RS MODE
  • Shipments with driver assigned
  • Consol shipments
  • Continuous move shipments
  • Ground schedule shipments with theÂ RESERVATION_OPEN status
  • Ground schedule shipments with the RESERVATION_CLOSED status and has DYNAMIC TRIP or STATIC TRIP schedule type
  • Shipments with FLIGHT or VESSEL schedule type

Shipments that do not pass the above validations will not be part of work assignment and all the other valid shipments count are displayed under Shipments Assignable in the Work Assignment Planning screen.

Note: The optional feature âDRIVE WORK ASSIGNMENT IN COLUMN GENERATIONâ is used to allow all shipments on a Work Assignment (WA) to drive one gigantic shipment vs. existing logic of driving latest shipment. If the optional feature is enabled, the application will provide the optimum rest time as required while creating a feasible solution/column for all shipments on a WA. If the feature is disabled, then the application will not provide the optimum rest time as required for the all shipments on a WA.

Unassigning Shipments

If all the selected shipments are part of an existing work assignment, OTM will disband the work assignment and then include shipments in the Work Assignment Planning. In another scenario, if few (but not all) shipments are part of an existing work assignment, OTM removes those shipments from the work assignment and includes them in the Work Assignment Planning.

Note: If the number of remaining shipments in the work assignment is less than the WA MIN SHIPMENTS IN TOUR logic parameter, then the work assignment feasibility code is WORK_ASSIGNMENT_LOGIC_PARAMETER_INFEASIBLE.

For example, consider that the WA MIN SHIPMENTS IN TOUR logic parameter is set to two. In this example, a total of five shipments are selected. Three of them are part of the ABC work assignment and two of them are part of the XYZ work assignment. If you run the Optimize Work Assignments action on all these shipments, OTM will unassign shipments from the existing work assignments and then plan the shipments. During the unassignment process:

  • In case of the ABC work assignment, OTM will disband the ABC work assignment. Â 
  • In case of the XYZ work assignment, which contains three shipments, OTM will include the two selected shipments in the planning and remove them from the XYZ work assignment and updates the work assignment. The number of remaining shipments in the XYZ work assignment is one and is less than the WA MIN SHIPMENTS IN TOUR logic parameter which is set to two, then the work assignments feasibility code is changed to WORK_ASSIGNMENT_LOGIC_PARAMETER_INFEASIBLE.

Optimizing Work Assignments

  1. Select a Parameter Set ID.
  2. Enter a Description.
  3. Click OK. The Work Assignment Planning page appears.
