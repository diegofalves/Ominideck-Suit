Build Shipment Set

This page is accessed via:

  • Sourcing > Historical Shipments > Shipment Set Criteria > Actions > Build Shipment Set > Build Shipment Set
  • Cooperative Routing > Historical Shipments > Shipment Set Criteria > Actions > Build Shipment Set > Build Shipment Set

The action moves shipments from Oracle Transportation Management based on the constraints specified in the selected shipment set criteria. Â If you have a large set of shipments to bring over, it is better to run the action in the background. The action will email you when it is done and you can continue working in OTM until you see the email.

Note the following exceptions and exclusions:

  • The action only moves shipments which use rates defined in the domain in which the shipment set criteria is saved. It creates the Sourcing shipments (p_shipments) in the domain where the OTM shipments are located.
  • You can import shipments from a domain and sub-domain as follows:
    • Assuming the following:
      • the OTM shipments are in the ABC domain,
      • those OTM shipments use rates in the ABC/ASIA domain, and
      • the shipment set criteria is in the ABC/ASIA domain.
    • Then, the Build Shipment Set action imports shipments from both the ABC and ABC/ASIA domains provided the rate records those shipments use are in the ABC/ASIA domain (the same as the shipment set criteria).

    Note: Before you run this action in a child domain, ensure that the child domain has write grants (see domain grants) for its parent domain. This grant is required because this action creates the Sourcing shipments in the OTM shipment's domain. This includes the P_SHIPMENT table and all of its child tables.

  • The action excludes buy shipments missing an equipment group. This is true even if the equipment group profile is not specified on the shipment set criteria.
  • This action also excludes all sell shipments.
  • If the mode profile ID specified on the shipment set criteria does not contain any transport modes, then this action assumes that all modes match. The result is that all shipments are pulled into the shipment set. To avoid this, ensure that the transport modes in the your mode profile ID match the transport modes of the shipment that you are pulling into Oracle Transportation Sourcing.
  • If no lane is found when using the desired geo hierarchy profile ID specified on the shipment set criteria, the lane ID on the shipment is left blank. Shipments with null lanes cannot be used by Oracle Transportation Sourcing. If you see any shipments with null lanes, run the Fill Missing Lanes action.
  • This action creates the Sourcing shipments (p_shipments) in the domain where the OTM shipments are located. If you are running this action from a different domain, you must give the domain in which you are running the action write permissions to the domain in which the OTM shipments are located.

Building a Shipment Set

  1. Select a shipment set criteria and click Actions > Build Shipment Set > Build Shipment Set.
  2. If you want to run the action in the background, accept the default of Yes for the Run Job in Background field. The server can time out while waiting for a result if the action is run in the foreground for a large data set. Hence, for large data sets, you need to run the action in the background.
  3. If you selected Yes to Run Job in Background, specify a contact in the Contact ID field. When the action finishes, an email will be sent to the contact. Email notification is the only communication method even if fields are populated for other communication methods. There is no results page so the email notification is the only notification.
  4. Click Build Shipment Set to run the action. The result page appears.

The result page displays the success message and the list of Oracle Transportation Management shipments that were rejected because they had zero weight or the NMFC was not defined in them.

You can use the shipment search page to view the sourcing shipments that were added to the shipment set. To view sourcing shipments, go to Sourcing > Historical Shipments > Shipments. If you have more than 1,000 shipments, you can export the results.
