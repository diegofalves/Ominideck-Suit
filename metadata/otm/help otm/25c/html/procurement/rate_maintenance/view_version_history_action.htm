View Version History Action

Access the version history page via the Approval Summaries Detail workbench layout is accessed as follows:

  1. Navigate toÂ Sourcing > Rate Maintenance > Approval Summary. Enter search criteria and click Search.
  2. Select Always Open in New Window so that the approval summary page opens in a pop-up window.
  3. Right-click on a row in the search results and select View Approval Details. The APPROVAL_SUMMARY_DETAIL workbench layout open in a pop-up window.
  4. To view the version history for rate offerings, select a rate offeringÂ in the Rate Load Rate Offerings tab. Then, clickÂ Actions > View Version History.
  5. To view the version history for rate records,Â select a rate recordÂ in the Rate Load Rate Records tab. Then, clickÂ Actions > View Version History.
  6. Close the browser tab to return to the Approval Summaries Detail workbench layout.

View Version History allows you to link from a new rate to an older existing version of that rate (the parent). You can view the IDs of previous versions of the selected rate offering or rate record. To understand the changes made between versions of a specific rate offering or rate record, click theÂ View Differences icon. If the icon is grayed out then there is only one version of the rate offering or rate record.

Use Case

You can use version history as follows:

  1. On the rate offering and rate record templates, specify the Rate Offering Parent ID and Rate Record Parent ID. Adding the parent IDs on the templates ensures those columns are shown in the rate load definition. The parent rate cannot be the same ID as the template rate.
  2. Include the Rate Offering Parent ID and Rate Record Parent ID columns in the rate load definition. Do not select the read-only option.
  3. Download rates without any parents.
  4. The downloaded spreadsheet contains columns for both the Rate Offering Parent ID and any Rate Record Parent IDs. Those columns should be blank since the downloaded rates have no parents defined.
  5. Next in the downloaded spreadsheet, copy the IDs of the existing rates to the parent columns and define new IDs for the new rates.
  6. Upload the edited spreadsheet.
  7. When you review the newly uploaded rates, you can run the View Version History action and see the relationship to the downloaded rates.

The same steps work if your downloaded rates already have parents. In step 5, you can replace the (to be) grandparent IDs with the (to be) parent IDs.

For Rate Offering

In the Rate Load Rate Offerings tab, select a rate offering row and click Actions > View Version History. You see the Version History page which displays data including the parent rate offering ID, the rate load group, rate load definition ID, approval summary ID, and upload and approval details.

When the Parent Rate Offering ID is specified on the selected rate offering, you can see the older existing versions (parents and grandparents) of the selected rate offering.

For Rate Record

In the Rate Load Rate Records tab, select a rate record row and click Actions > View Version History. You see the Version History page which displays data including the parent rate record ID, the rate load group, rate load definition ID, approval summary ID, and upload and approval details.

When the Parent Rate Record ID is specified on the selected rate record, you can see the older existing versions (parents and grandparents) of the selected rate record.
