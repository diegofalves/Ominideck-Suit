Sourcing

Build Rates

This page is accessed via Sourcing > Bid Creation > Project > Actions > Sourcing > Rates > Build Rates.

The Build Rates action uses the template rate offering you previously created and inserts the rate offering type of your choice. Once the action completes, you will have one rate offering per winning carrier and matching rate records for the lanes. This includes the accessorials and special services for arbitraries you have defined on your project.

Prerequisites for the Build Rates Action

Prior to running the Build Rates action, ensure that the following are completed:

  • Bid publish status is set to BID_PUBLISH_YES
  • The selected solution ID is specified on the bid round

Setting Bid Publish Status

There are several actions that let you set or change the bid publish status.

During the normal sourcing flow, you would typically use the Set Bid Publish Status for All Bids action. This action selects or de-selects all bids in a given solution for publishing by changing the bid publish status.

If at a later time, you need to change the status on some bids, you can either use the Change Publish Status action from the solution or the Change Publish Status action from the bid.

Selecting Solution ID on the Bid Round

Before building rates, you must go to the Bid Round and select which solution ID to use.

Note: The Build Rates action automatically uses the bid round with the highest number in the Bid Round Sequence field. So if there are three bid rounds and they all have a solution ID specified, the action will use the bid round with the Bid Round Sequence of 3.

Building Rates

On the Build Rates page, complete the following:

  1. Enter a rate quality in the Backup Carrier Quality Rating ID field.

    The rate quality helps Oracle Transportation Sourcing identify a backup carrier and give preference to the actual awarded carrier. Only backup carriers get this rate quality attached to their rate records. Hence, the rate quality should penalize the backup carriers, e.g. positive percentage or positive dollar amount.
  2. Enter an Effective Date and an Expiration Date to set the rate version dates on the new rates.
  3. If you want to run the action in the background, accept the default of Yes in the Run Job in Background field. The server can time out while waiting for a result if the action is run in the foreground for a large data set. Hence, for large data sets, you need to run the action in the background.
  4. Designate a contact in the Contact ID field if you selected Yes for Run Job in Background. When the action finishes, an email will be sent to the contact. Email notification is the only communication method even if fields are populated for other communication methods. There is no results page so the email notification is the only notification.
  5. Click Build Rates to start the action.

When you run the action, only those bids that have a status of BID_PUBLISH_YES are converted to rate records. The Oracle Transportation Management rate tables within the database are populated.

New Oracle Transportation Management Rate Offering and Rate Records

The Build Rates action copies the Sourcing rate offering and rate records that were created for each awarded bid.

Note: All attributes specified as non-biddable on the CRT and their values are copied from the rate offering or rate record template.

Let's look at the newly created rate offering and rate records in more detail:

Rate Offering

The Build Rates action copies both the rate offering and rate record from each awarded bid. Neither the Template or Read Only checkboxes are selected which means that the new rate offering is an Oracle Transportation Management rate offering and can be used for shipment planning.

The rate offering ID is created as follows using the default BNG context:

  • [First 8 letters of service provider XID]-[First 9 letters of transport mode XID]-[rate version effective year]-[three-digit sequence]
  • An example is CARRIER-TL-2012-001

A new rate version is created based on the effective and expiration dates specified on the build rates action page. The version ID is SOURCING_EffectiveDate_ExpirationDate. An example is SOURCING_6/12/12_6/30/13.

Non-biddable Attributes for Rate Offerings

For all non-biddable fields in the carrier response template the following apply:

  • non-biddable data is copied from template rate offering to the new rate offering
  • other fields like remarks, NMFC, and comments which do not appear in carrier response template are also copied directly from template rate offering to the new rate offering

Note: For non-arbitrary special service costs, the template data is copied to the newly created rate offering whether the costs are biddable or non-biddable in the carrier response template.

Rate Records

The Build Rates action copies the rate record for each awarded bid. The rate record is associated to the single rate offering described above. These rate records are Oracle Transportation Management rate records and can be used for shipment planning.

By default, the rate record ID is created as follows: [Rate Offering ID]_[lane XID]. For example, YELLOW-LTL-2012-001_180623496.

You can create your own BNG rules, mark them as default, and they will be used for the naming of the rate records and rate offerings. Turn onÂ ALLOW RATE GEO BNG RULE IN SOURCING BUILD RATES option feature.

Effective date and expiration date are pulled from the build rate action input page. While the rate quality ID is pulled from the solution ID. The majority of the data on the Attributes, Rate Costs, Accessorials and Special Service tabs are pulled from the published bids.

Non-biddable Attributes for Rate Records

For all non-biddable fields in the carrier response template the following apply:

  • non-biddable data is copied from template rate record to the new rate record
  • other fields like remarks, NMFC, and comments which do not appear in carrier response template are also copied directly from template rate record to the new rate record

Using Business Number Generation

You can configure Oracle Transportation Sourcing to generate rate offering IDs as required by your business process. You can configure IDs using the business number generator (BNG). The rate offering business number type uses the RATE_OFFERING_CONTEXT. The business number type defaults for rate offering ID is detailed in the help.
