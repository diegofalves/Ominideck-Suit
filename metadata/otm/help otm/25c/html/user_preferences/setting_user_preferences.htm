Configuration and Administration

User Preferences

This page is accessed via:

  • Configuration and Administration > Preferences > User Preferences
  • User (Settings and Actions) icon in the Unified Global Header. Then, under Administration, select User Preferences.

A user preference set determines the interface language, currency type, date and time format, units of measure, and other settings that are used as the defaults in business objects during your sessions (see below for a complete list). After a user preference set is created, it can be assigned to an entire domain, a specific user, or a specific user level.

Note: Service provider users are only able to edit the user preference that is assigned to them. The service provider administrator is only able to edit user preferences that are in the SERVPROV domain.

The values in appropriate fields or the language of appropriate interface elements, are automatically converted to your new preferences in saved business objects. For example, if monetary amounts were entered in dollars and you change your currency preference to British pounds (GBP), the numbers in those fields will be converted to the correct amount in pounds.

Note: Number formatting; for example 1,000.00 versus 1.000,00; is set in Regional and Language Options in the Control Panel for Windows settings not Oracle Transportation Management.

Note: The Unit of Measure manager controls default UOMs. User preferences can override those defaults for the user.

  1. Enter a unique User Preference ID for the user preference.
  2. Select a Domain Name (ADMIN level users only) in which to store the user preference. The default is the domain you are logged into and the list displays the domains to which you have write access.

Preference

  1. Select a preference Name from the list and then select the Value that you want.
  2. Click Save for each preference setting that you want to configure. If you do not configure a preference setting, the system default setting is used.
  3. Click Finished.

Note: Whenever you change a preference in an "active" user preference, you must log out and log back in for that preference setting to take effect.

Language

The language of field names, buttons, icons, and other text elements in the user interface. The language preference does not change menu items and text outside the Oracle Transportation Management window or the language of text saved in entry fields.

In some instances, there is more than one variation of a language based on the country. For example, there is European French and Canadian French. In that case, you must create another user preference for Country, in addition to the one for Language.

Note: For invalid combinations of the country and language preferences (for example, Spain and Swedish), the language preference overrides the country preference. Invalid combinations may cause performance issues.

For reports, you also must specify the country user preference as well as the language user preference.

Note: Some supported languages require special computer configuration.

Note: Fax, Online Help, Training Materials, and Documentation are only supported in English.

Parameter Set

Allows you to specify a parameter set as a default. This is more specific than defining it at the domain. This parameter set defaults to the Ignore Constraints page where it can be overridden.

Note: For some actions, OTM uses the Parameter Set defined in a user's preference, if it exists. Otherwise, OTM uses the Parameter Set defined as the default for the domain (in Domain Settings). For all other actions, OTM does not check to see if a user has a Parameter Set defined in their preference. Only the Domain Settings parameter set are used. Here are the actions that use the parameter set from user preference first, then from domain:

  • All the fleet actions
  • Unassign order release
  • Move order to shipment
  • Resequence stops
  • Collapse order movement
  • Split order movement
  • Create relay order movement
  • Move to Shipment
  • Unassign Order Movement

The other actions do not look at the parameter set from the user preference.

This preference is used only for branding purposes and local themes.

The value in this preference represents the directory structure of the client XSL, with slashes between the directory names. If this user preference is blank or has the value of default, the standard Oracle Transportation Management XSL is used.

The directory represented by the user preference for client XSL files is checked when loading each XSL stylesheet.
