About Transportation Operational Intelligence

Transportation Operational Intelligence is based on Oracle Analytics Server (OAS). The Transportation Operational Intelligence subject area allows you to access transactional data present in OTM to create reports and data visualization projects.

Logistics Network Modeling Intelligence is available as a subject area under Transportation Intelligence > Ad Hoc Query > Analysis > Logistics Network Modeling Intelligence.

Subject Area

The Transportation Operational Intelligence subject area has three main sections, each catering to a different component:

  • Shipment Operational Analysis
  • Order Release Operational Analysis
  • Invoice Operational Analysis

Shipment Operational Analysis

The Shipment table in OLTP has been broken into Dimension shipment and Fact Shipment, and the other connections have been made to the Fact Shipment. The important directories that have been included are:

  • Shipment Facts (Shipment table)
  • Shipment Dimension (Shipment table)
  • Shipment Status Dimension (Shipment status, Status type, status value tables)
  • Shipment Source and Destination locations (Location + Country + Region tables)
  • Shipment fiscal and time dimensions (Time table)
  • Shipment Servprov (Service Provider table)
  • Shipment Transport mode (transport modeÂ  table)
  • Shipment Perspective(Perspective table)
  • Shipment domain (Domain table)

Loading Data

To load and unload scenario data to Logistics Network Modeling, use the modeling scenario actions of Send Data to Analytics and Unload Data from Analytics.

Dashboard

A Dashboard is a collection of performance metrics reflecting common strategic goals. The metrics are displayed as various key performance indicators formatted as charts, graphs, tables, cross-tabs, and status indicators. There are multiple tabs organizing the metrics. Dashboard comes pre-defined with standard metrics. If you have access to Logistics Network Modeling and you have the Logistics Network Modeling Dashboard set as a default dashboard in user preferences, this page can be accessed via Transportation Intelligence > Dashboard.

Note: A user can only have one default dashboard set at a time. As a result, a single user cannot have both an Transportation Intelligence (TI) default dashboard and a Logistics Network Modeling default dashboard. To view both the TI and Logistics Network Modeling dashboards, create one user with access to LNM and set their dashboard preference to point to the LNM dashboard. Then, create a second user with access to TI and set their dashboard preference to point to the TI dashboard.

Ad Hoc Query

This component allows creation of reports on the fly. For details on Ad Hoc Queries, click the OAS help icon. This page is accessed via Transportation Intelligence > Ad Hoc Query. Then under Analysis and Interactive Reporting select Analysis > Logistics Network Modeling Intelligence. You should have an OAS license to work in this page.

Catalog

Any queries that you have created and saved appear under Catalog > My Folders and are only be available to you.

Any queries that the TI administrator created and saved appear under Catalog > Shared Folders. Any TI user can access these queries.

Important! All user-defined reports and dashboards must be created inside the shared/Custom folder or a sub-folder. Defining reports and dashboards inside the shared/Custom folder will ensure your user-defined reports and dashboards are retained during upgrades. All out-of-the-box folders are read-only so you cannot save content in folders other than shared/Custom. If you try to save content outside of shared/Custom, you will see an error which may be one of the following: "Access denied for user to path..." or "Can't save in root folder.".
