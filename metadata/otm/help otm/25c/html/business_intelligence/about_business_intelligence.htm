Transportation Intelligence

About Transportation Intelligence

Transportation Intelligence (TI) provides an environment for managing business metrics by monitoring, analyzing, and reporting metrics at various levels of the organization. To provide access to Transportation Intelligence, you need to add a business intelligence application and a business intelligence role to each user as appropriate to access different functionality. Transportation Intelligence is based on Oracle Analytics Server (OAS). Read the documentation for Oracle Analytics ServerÂ found on the Oracle Help Center.

Overview

OAS is the tool that Oracle Transportation Management development used to build Transportation Intelligence. OAS enables Oracle Transportation Management to query, format, and display the information you see in Transportation Intelligence.

Transportation Intelligence is the set of rules that govern how operational data is moved from Oracle Transportation Management to the historical database, defines metrics, and displays the results. The historical database is optimized for read-only queries along with pre-determined dimensions.

The process of moving data between databases is often referred to as ETL for Extract, Transform, and Load. The HD Star Schema refers to the way data is organized in business intelligence applications.

The data that is available to you is of two types: facts and dimensions.

  • Facts are typically numbers and can be added together. Examples of facts include shipment weight and number of orders.
  • Dimensions are often a non-number type of data and allow you to group the facts. Examples of dimensions include time periods, transport modes, and destination locations.

Transportation Intelligence monitors the following values:

  • Actual: Obtained from the operational data.
  • Target: Expected performance, entered by the user or derived. You can define your own targets.
  • Industry Benchmark: Entered by the user or derived. Transportation Intelligence does not ship with any industry benchmarks, but you can enter your own. You can do this by creating Key Performance Indicators or KPIs and setting targets in Transportation Intelligence.
  • Forecast: Entered by the user or derived. Transportation Intelligence does not ship with any industry benchmarks, but you can enter your own. You can do this by creating Key Performance Indicators or KPIs and setting targets in Transportation Intelligence.

You can define your own targets, industry benchmarks, and forecasts by creating Key Performance Indicators or KPIs in Transportation Intelligence.

The final result is a metric, a measurement of which you want to keep track. You can visualize metrics in formatted reports. Or, you can display multiple metrics on a dashboard.

Transportation Intelligence contains the following components.

Dashboard

A Dashboard is a collection of performance metrics reflecting common strategic goals. The metrics are displayed as various performance indicators formatted as charts, graphs, tables, cross-tabs, and status indicators. There are multiple tabs organizing the metrics. Dashboard comes pre-defined with standard metrics. This page is accessed via Transportation Intelligence > Dashboard. In user preferences, you can define a default dashboard. See Business Intelligence Roles for a table displaying dashboards, roles, and access. Dashboards are user-configurable. See My Dashboard below for details.

My Dashboard

It is a configurable Dashboard so you can display the metrics you want in the format you want. This is in OAS. You should have an OAS license to work in this page. For details on My Dashboard, click the OAS help icon on the Dashboard Editor page. This page is accessed via Transportation Intelligence > My Dashboard. Then click the available link.

My Dashboard can only be seen by the OTM user who creates it. However, your system administrator can share the reports that are part of a My Dashboard.

Ad Hoc Query

This component allows creation of reports on the fly. For details on Ad Hoc Queries, click the OAS help icon. This page is accessed via Transportation Intelligence > Ad Hoc Query. Then under Analysis and Interactive Reporting select Analysis > Transportation Intelligence. The Transportation Intelligence subject area contains data for the 6.3 version and later. The subject area of Fusion Transportation Intelligence has been deprecated, is not supported, and is not available by default. Set the SHOW_FTI_SUBJECT_AREA report common property to see the Fusion Transportation Intelligence subject area. You should have an OAS license to work in this page.

Catalog

Any queries that you have created and saved appear under Catalog > My Folders and are only be available to you.

Any queries that the TI administrator created and saved appear under Catalog > Shared Folders. Any TI user can access these queries.

Important! All user-defined reports and dashboards must be created inside the shared/Custom folder or a sub-folder. Defining reports and dashboards inside the shared/Custom folder will ensure your user-defined reports and dashboards are retained during upgrades. All out-of-the-box folders are read-only so you cannot save content in folders other than shared/Custom. If you try to save content outside of shared/Custom, you will see an error which may be one of the following: "Access denied for user to path..." or "Can't save in root folder.".

Administration

Controls administrative functions such as managing sessions. This page is accessed via Transportation Intelligence > Administration. This is in OAS. You should have an OAS license to work in this page. For help with Administration, click the OAS help icon.

Since this an OAS page, you see the same screen that is shown when administering OAS. Just because a link or function is show on this page, does not indicate that it is supported by Oracle Transportation Intelligence. For example, Oracle Analytics Publisher is not supported as part of Oracle Transportation Intelligence, but it will appear on this page since it is installed by OAS.

Note: This functionality is only available if you are logged in as a DBA.

Alerts

This component allows creation of notification. You can determine the priority, recipients, the schedule, and conditional requests among other criteria. This is accessed via Transportation Intelligence > Alerts. This is in OAS. Since this an OAS page, you see the same screen that is shown when administering OAS. You should have an OAS license to work in this page. For help with Alerts which are called Agents in OAS, click the OAS help link.

Data Management

You can use Data Management to view information related to the data load including the logs and rejects.

Note: This functionality is only available if you are logged in as a DBA.

Process Management

Process Management allows you to manually push data and targets to the HD (historic database). See the topics Data Flow to Transportation Intelligence or Data Flow for Global Trade Intelligence.

Note: This functionality is only available if you are logged in as a DBA.

Power Data

Power Data allows you to view and manage setup data for Transportation Intelligence including targets. Additionally, you can define dashboards.
