Shipment Management

SKU

Stock keeping unit (SKU) information can be entered through integration, or manually. If entering a SKU manually, it is accessed via Shipment Management > Visibility > SKU. The Inventory tab contains more information about the SKU.

The following properties can be relevant when adding SKUs:

  • glog.asset.expiration_clear_days: Define within how many days before the real expiration date is counted as invalid.
  • glog.skuEvent.copyEventQuantitiesToSku: Determines if Â the table is automatically updated. i an SKU event includes entries of quantities. Then the SKU Quantity table can be updated accordingly.
  • glog.sku.totalsbucket.treelevel.max: This property determines number of levels of bucket hierarchy to be considered This property affects the calculation of bucket counts and the displaying of bucket hierarchies.

Adding a SKU

  1. Enter a SKU ID that uniquely identifies the SKU.
  2. Enter a Packaged Item ID, the ID of the packaged item tracked by the SKU.
  3. Enter a Supplier ID.
  4. Enter an Owner ID to identify the owner of the item.
  5. Enter a Warehouse ID to identify the current warehouse location of the item.
  6. Enter a Ship Unit Spec ID to identify the how the material will be shipped, such as on a pallet.
  7. Enter the average number of days that an item in the SKU has been in inventory in the Average Age field.
  8. Enter the most recent date on which the quantities of this SKU were changed in the Last Inventory Date field.
  9. Select a Domain Name.
  10. Enter a Description of the item.
  11. Enter an Involved Party. Involved parties can be added to a SKU record to specify who receives a notification when a change, known as an SKU event, occurs to that record. When an involved party has been added to an SKU record, that party also appears in the Individual Counts action.

    Note: At least one automation agent must be created for involved parties to be notified of changes to SKU records. The Agent Type for the agent must be SKU, and the action performed by the agent must be NOTIFY INVOLVED PARTY.

    One or more SKU events can be selected to trigger the action, such as SKU - QUANTITY ON HAND CHANGED or SKU - STATUS CHANGED.

  12. Select a Communication Method.
  13. Select an Involved Party Qualifier.
  14. Select the Permanent checkbox if you want the involved party to continue to receive notifications of subsequent SKU events related to this SKU record. If you want the involved party to be notified of the next SKU event only, the Permanent checkbox should be cleared. If the Permanent checkbox is cleared, the occurrence of the first SKU event results in the deletion of that involved party from the SKU record.
