Control Screening

This page is accessed via:

  • Compliance > Control Screening
  • Ask Oracle > Control Screening

Control ad hoc screening offers you a way to perform transactional screening of personal information, product and usage information, location information, and user-defined conditions. This service supports Export Administration Regulations algorithms and cumulative algorithms.

Based on the algorithm you select, the screening service uses an algorithm handler to feed the Compliance Rule Set Group ID or Compliance Rule Set ID to the screening engine where the conditions of the rules are matched to the transactions

Performing a Control Screen

Only one field is required for a control screen; however, the more detailed your criteria, the more accurate your results will be. Once you have entered your match criteria, click Match to perform the screening. A status will be assigned when the screening process is completed. For more information about internal status designations, see GTM Internal Status Designations.

Note: Product classification status will not be updated when you perform control screening. It will be updated only when you run the Verify Classification or Look Up Product Classification actions on a trade transaction or transaction line.

Screening Criteria

Select a compliance rule set group for the screening from the Compliance Rule Set Group ID drop-down list.

Transaction Information

If you want to designate a transaction for screening, use this section to define the matching transaction criteria.

  1. Enter a type of transaction in the Transaction Type field.
  2. Enter a type of transaction in the Declaration Type field.
  3. Enter a date in the Transaction Date field.
  4. Select a transport mode in the Transport Mode ID drop-down list.
  5. Select a country of origin ID from the Country of Origin ID drop-down list.
  6. Select a country of manufacture ID from the Country of Manufacture ID drop-down list.
  7. Enter an Incoterm in the Incoterm ID field.
  8. Enter a trade transaction item structure in the Structure ID field.

Item Information

If there is a specific commodity, you can specify it in this section.

  1. Enter an item ID in the Item ID field.
  2. Select a commodity ID from the Commodity ID drop-down list.
  3. Select a user-defined commodity ID from the User-defined Commodity ID drop-down list.

Product Classifications

You can choose specific products to be checked as part of the compliance screen.

  1. Enter an ID in the Product Classification Type ID field.
  2. Enter a code in the Product Classification Code field.
  3. Click Save.

Involved Parties

You can designate the screening be done based on involved parties.

  1. Select a qualifier from the Involved Party Qualifier drop-down list. Generally, the involved party qualifier defines the role of the involved party.
  2. Enter the Party ID of the involved party in the Involved Party field.
  3. Click Save.

Involved Locations

You can designate the screening be done based on involved locations. Define those locations in this section.

  1. Select a qualifier from the Involved Location Qualifier ID drop-down list. Generally, the involved location qualifier defines the role of the involved location.
  2. Enter the Location of the involved party in the Location field.
  3. Click Save.

Transaction Remarks

You can match specific transactions based on transaction remarks.

  1. Select a remark qualifier from the Remark Qualifier drop-down list.
  2. Enter the remark test in the Remark Text field.
  3. Click Save.

Transaction Line Remarks

You can match specific transaction lines based on transaction line remarks.

  1. Select a remark qualifier from the Remark Qualifier drop-down list.
  2. Enter the remark test in the Remark Text field.
  3. Click Save.

Transaction Reference Numbers

  1. Select a reference number qualifier from the Reference Number Qualifier ID drop-down list.
  2. Enter the Reference Number Value.
  3. Click Save.
  4. Continue to enter as much remark information as you require. Click Save after entering each line of data.

Transaction Line Reference Numbers

  1. Select a reference number qualifier from the Reference Number Qualifier ID drop-down list.
  2. Enter the Reference Number Value.
  3. Click Save.

User-defined Code Information

  1. Select a code category from the Category drop-down list.
  2. Select a type ID from the Type drop-down list.

    Note: The available options are determined by your choice of category in the previous step.

  3. Select a value of the code from the Value drop-down list.

    Note: The available options are determined by your choice of type in the previous step. This drop-down list displays description of the user-defined code values instead of only the codes.

  4. Click Save.

Compliance Rule Matches

To execute the screening, click Match. The Compliance Rule Matches window opens. This window displays the following:

  • the ID of the compliance rule you selected,
  • control category, type, and code
  • party qualifier
  • exception rule id, control type, and control code
  • documents associated with the compliance rule ID
  • screening results
