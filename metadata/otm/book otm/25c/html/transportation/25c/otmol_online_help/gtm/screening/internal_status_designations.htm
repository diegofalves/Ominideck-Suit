GTM Internal Status Designations

GTM assigns internal status designations to track whether screening tasks have been performed, completed, or failed.

Status buttons are located on the following managers:

  • Party
  • Trade Transaction
  • Trade Transaction Line
  • Declarations
  • Declaration Lines
  • Location
  • Registration
  • Product Classification
  • Campaigns
  • Campaign Lines

Click the Status button on any of these managers to see the statuses that have been assigned.

The tables below outline the statuses you may encounter.

Restricted Party List Statuses

Object

Status Value

Explanation

Party (Contact)

RPLS_NOT_STARTED

The restricted party list screening has not yet been initiated.

RPLS_PASSED

Restricted party screening has been executed and one of the following conditions has occurred:

  • No restricted party list records have been matched.
  • One or more restricted party list records have matched and all of them have been resolved as Not A Match.

RPLS_FAILED

Restricted party screening has been executed and at least one restricted party list record was resolved as a verified match.

RPLS_REQUIRES_REVIEW

Restricted party screening has been executed and one of the following conditions exists:

  • Restricted party records were matched but are not yet resolved (i.e. they are still designated as potential matches).
  • Restricted party records were matched, none have been resolved as verified matches, and there are still match records waiting to be resolved.

RPLS_ESCALATED

The status value has been set to escalated because either the Escalate button has been clicked or the Change Restricted Party Status action has been used. You can employ a saved query to pull up records with this status. Restricted party screening on a party will not change the status if the party is in Escalated status. The reviewer must manually review the Party-Restricted Party matches and reset the status to Verified Match/Not A Match.

TL_RPLS_PASSED

Restricted party screening has been executed and one of the following conditions has occurred:

  • One or more restricted party list records have matched and all of them have been resolved as Not A Match.

TL_RPLS_FAILED

Restricted party screening has been executed and at least one restricted party list record was resolved as a Verified Match.

TL_RPLS_REQUIRES_REVIEW

Restricted party screening has been executed and one of the following conditions exists:

  • Restricted party records were matched, none have been resolved as verified matches, and there are still match records waiting to be resolved.

TS_RPLS_FAILED

Restricted party screening has been executed on the trade transaction and at least one restricted party list record was resolved as a verified match.

TS_RPLS_REQUIRES_REVIEW

Control Screening Status

Object

Status Value

Explanation

Trade Transaction Line

TL_CONTROL_SCREENING_NOT_STARTED

Control screening has not been performed against the trade transaction line.

TL_CONTROL_SCREENING_ON_HOLD

Control Screening (i.e. Compliance rule screening) process has been executed and compliance rules with category=control have been hit and not all of them have a compliance rule with category=exception associated or have not been assigned one or moreÂ licenses for the total quantity indicated in the line.

TL_CONTROL_SCREENING_PASSED

Control screening (i.e. Compliance rule screening) process has been executed and one of the following conditions have occurred:

  • No compliance rule with category = control has been hit.
  • Compliance rules with category = control have been hit and all of them have a compliance rule with category = exception associated or have been assigned one or more license for the quantity indicated in the line.

You manually change the status from hold to failed when the decision is that the transaction line should be removed or it violates any regulation.

Trade Transaction

TS_CONTROL_SCREENING_NOT_STARTED

Control screening has not been performed against the trade transaction.

TS_CONTROL_SCREENING_ON_HOLD

Control Screening (i.e. Compliance rule screening) process has been executed and compliance rules with category=control have been hit at any trade transaction line and (not all of them have a compliance rule with category=exception associated or have not been assigned one or more license for the total quantity indicated in the line).

TS_CONTROL_SCREENING_INCOMPLETE

Control Screening (i.e. Compliance rule screening) process hasn't been executed for all the trade transaction lines.

TS_CONTROL_SCREENING_PASSED

Control screening (i.e. Compliance rule screening) process has been executed to all the trade transaction lines and one of the following conditions has occurred:

  • No compliance rule with category = control has been hit at any trade transaction line.
  • Compliance rules with category = control have been hit at any trade transactions line and all of them have a compliance rule with category = exception associated or have been assigned one or more licenses for the total quantity indicated in the line.

TS_CONTROL_SCREENING_FAILED

You manually change the status from hold to failed when there is at least one line in status Failed.

The required classifications on a trade transaction line/declaration line Â will be used to set product classification lookup statuses.
