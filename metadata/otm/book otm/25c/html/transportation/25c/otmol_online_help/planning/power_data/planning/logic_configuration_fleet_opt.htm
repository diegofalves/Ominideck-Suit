Logic Configuration - Fleet Optimization

Logic configuration fleet optimization parameters are accessed via the Logic Configuration page if you select a logic configuration type of Fleet Optimization.

The parameters in the Driver Optimization logical group are specific to drivers. Those in the Fleet Optimization logical group can be used by multiple processes.

For the Optimize Driver Assignments action, use the parameters under the 'General' and 'Savings Merge Algorithm' groups to define values used by the savings merge algorithm when Savings Merge is selected for the Fleet Assignment Algorithm parameter. If 'Column Generation' is selected instead, use the parameters under the 'General' and 'Column Generation Algorithm' groups.

DRIVER EVALUATION TYPE

Savings Merge Algorithm

Driver Optimization

6.4.2

This parameter determines when driver assignment is performed. This allows you to enter the type of driver evaluation performed inside the savings merge algorithm. The parameter guides the algorithm to do a thorough driver evaluation at different levels: 1: Perform driver evaluation at end of process (fastest performance), 2: Perform driver evaluation after each iteration, 3: Perform driver evaluation for all paired shipments (slowest performance).

The default is 1. Perform driver evaluation at end of the process

DRIVER WAIT TIME PENALTY COST PER HOUR

Savings Merge Algorithm

Driver Optimization

6.4.2

This allows you to enter the wait time penalty cost per hour to be considered during the driver assignment optimization. The default, '0', means no penalty cost. The purpose of this penalty is to avoid assigning drivers to shipments with large wait times in between them. This cost will not be part of the actual shipment cost. It is just used in the evaluation process to form tours with minimum wait times between shipments. Real rates can also be setup for the wait times between shipments (for example, you can use the rate basis item "Driver Idle time from Previous Assignment" and setup cost per hour/day of idle time) and this can be used during the driver assignment optimization to form tours with minimum wait times. In order for the real cost to be properly considered, select 'Cost Based Savings' for the parameter Fleet Savings Calculation Type. For distance based savings, only this parameter can be used during evaluation to charge a penalty for driver wait times between shipments. For cost based savings, either this parameter or real cost can be used.
The default is 0.

EQUIPMENT TYPE SEARCH DISTANCE BUFFER

General

Fleet Optimization

6.1

This defines the regions for which driver pools are located. OTM takes the area and adds an additional buffer. The larger this value, the larger the region covered, which provides more choices, but is also a longer run time.

Note: If FLEET ASSIGNMENT ALGORITHM is set to Savings Merge this parameter will not be honored.

EVALUATE EQUIPMENT TYPE USING COST

General

Driver Optimization

6.1

If this is true, the real cost of the equipment type to a shipment will be calculated through the rate engine during evaluation process of the optimization.

If false, the evaluation of the equipment type assignment will be based on distance. The run time will be quick if set to false.

Note: If FLEET ASSIGNMENT ALGORITHM is set to Savings Merge this parameter will not be honored.

FLEET ASSIGNMENT ALGORITHM

General

Driver Optimization

6.4.2

This allows you to select the algorithm used by the driver assignment optimization logic. The supported values are: 1.Column Generation, 2. Savings Merge.

If you select '2. Savings Merge', you will need to define values for the parameters (whichever are applicable for your scenario) in the "General' and Savings Merge Algorithm' groups.

If you select Savings Merge, OTM will not handle equipment type assignment during the optimize driver assignment action.

If you select '1. Column Generation', you can define values for the parameters in the 'General' and 'Column Generation Algorithm' groups.

The default is 1. Column Generation.

FLEET DISCOUNT PERCENTAGE

Savings Merge Algorithm

Driver Optimization

6.4.2

If you have two separate shipments, merging them will cost more since more miles will be covered but it may still make sense to merge them even if you have to incur empty miles. A higher percentage for this parameter will encourage combining shipments with empty miles in between. You can apply a discount to make this look and be more attractive.

For example, you have two shipments each 100 miles. You can combine them but then you'll have 20 miles in between with an empty shipment. Is that worth it?

This allows you to enter the discount percentage to be considered for distance based savings. The discount percentage is applied to the combined distance value during the savings calculation of the merge algorithms when the Fleet Savings Calculation Type parameter is defined as "Distance-based Savings." The higher the discount percentage value, the greater the distance savings. Also, the probability of assigning drivers to shipments is greater with a higher discount percentage.

Range of values = 0 to 100%.

Default value = 10%

FLEET FIXED COST PER RESOURCE

Savings Merge Algorithm

Driver Optimization

6.4.2

This allows you to enter the fixed cost per resource to be considered in the merge algorithm during the driver assignment optimization. This cost is just used in the evaluation process and is not part of the actual shipment cost. If you set a higher value for this parameter, OTM will encourage adding more shipments to the string even if there is no cost benefit.
The default is '0'.

For example, you have 2 shipments and 2 drivers available to be assigned to those shipments. If the shipments are not being strung, driver 1 will drive shipment 1 and driver 2 will drive shipment 2 (assumption).

There are 2 driver resources used here. So the total cost UNCOMBINED = cost of shipment 1+ cost of shipment 2 + 2 * FLEET FIXED COST PER RESOURCE.

If you string the shipments together and assign to one driver: Combined Total cost = cost of shipment 1+ cost of shipment 2 + cost of empty distance between shipment 1 and shipment 2 + 1* Fleet FIXED COST PER RESOURCE.

If COMBINED Total cost is less than the total cost UNCOMBINED, then OTM will string the shipments together.

Note: This parameter is used only during the evaluation process. It will not be part of the actual shipment cost.

You may not need to set values for this parameter for most cases. If your objective is to string as many shipments as possible and assign drivers to those shipments, then set a value for this. If you want a minimum cost solution, just use the default value for this parameter.

MARK DRIVER HOME STOP FIXED

General

Fleet Optimization

6.4.3

This logic configuration parameter is used by the Optimize driver assignment action (both Savings Algorithm and Column generation). The Assign Driver (to Shipment) and Assign Shipment (to Driver) actions use the MARK DRIVER HOME STOP FIXED parameter instead of this logic configuration parameter.

If set to No, the return home stop is not marked as permanent for the driver's current assignment and also the "Stop_Not_Removable" special service is not set on the home stop. So the next assignment for the driver will start from the last freight stop of the current assignment.

If set to Yes, the return home stop is marked as permanent for the driver's current assignment and also the "Stop_Not_Removable" special service is set on the home stop. So the next assignment for the driver will start from the home location.

If set to "Based on Driver Shift", the "Stop_Not_Removable" special service is set on the home stop, but the permanent flag is not set on the stop. The state of the special service and permanent flag is used as an indicator for the logic to decide whether to remove the home stop based on the driver shifts or not during the next assignment. If the next assignment for the driver falls on the same shift as the current assignment, then the home stop will be removed. For the next assignment, the driver will start from the last freight stop of the current assignment. Â But, if the next assignment for the driver falls on the next shift or outside the current shift, then the home stop is not removed from the current assignment. The driver will go home after the current assignment and start from the home location for the next assignment.

The values for the logic configuration parameter "MARK DRIVER HOME STOP FIXED" are effective only if the logic configuration parameter "GET THE DRIVER HOME FOR FLEET OPTIMIZATION" is set to true.

Otherwise it always use the default approach (default value is No).

MAXIMUM DISTANCE BETWEEN SHIPMENTS

General

Fleet Optimization

6.1

This parameter sets the maximum distance allowed between two shipments. This is based on one shipment's destination location and another shipments source location. If the distance exceeds this value, the shipments will not be strung together.

MAXIMUM NUM OF SHIPMENTS IN STRING

General

Fleet Optimization

6.1

This is the maximum number of shipments that can be strung together. The default is 4 so the number of shipments assigned to one driver during this process cannot exceed 4.

MAXIMUM TIME BETWEEN SHIPMENTS

General

Fleet Optimization

6.1

The drive time between one shipment's end time and the next shipments start time cannot exceed this value in order to string the shipments together.

NUMBER OF COLUMN GENERATION ITERATIONS

Column Generation Algorithm

Fleet Optimization

6.1

Controls how many times the assignment algorithm will run.

NUM OF ITERATIONS FOR FLEET OPTIMIZATION

Column Generation Algorithm

Fleet Optimization

6.1

This parameter controls the number of times the assignment algorithm is run. During the assignment process, the equipment types can be over-allocated and hence some of the assignments may become infeasible during the commitment stage. By invoking the assignment algorithm multiple times, the failed shipments may find other equipment types during subsequent runs.

PENALTY COST OF UNUSED DRIVER

Column Generation Algorithm

Fleet Optimization

6.3

This parameter defines a cost penalty that can affect how drivers are assigned to shipments. When set to a positive value, the system will try to avoid the penalty by assigning all possible drivers. When set to a negative value, the system will assign fewer drivers. Since this penalty is considered into the overall cost, choose a small value such as .01 if you do not want to force the system to choose a costlier solution.

Default = 0.

POSITION EQUIP TYPE DURING DRIVER ASSIGN

General

Driver Optimization

6.1

If this is true, the position equipment type logic will be performed during many to many driver optimization. Otherwise, many to many driver assignment will skip the equipment type positioning logic.

Note: If FLEET ASSIGNMENT ALGORITHM is set to Savings Merge this parameter will not be honored.

STRUNG SHIPMENTS MAXIMUM DURATION

General

Fleet Optimization

6.1

From the start time of the first shipment that is in the string, to the end time of the last shipment in the string is considered the duration. This parameter sets the maximum duration allowed. If a driver can only work 10 hours/day and the planner is planning for a day's work for the drivers, then this parameter can be set to 10 hours to reduce run time for unnecessary evaluation of drivers.

UNASSIGN EQUIPMENT WHILE UNASSIGN TYPE

General

Driver Optimization

6.1

During optimize driver assignment, the first step is unassign. This tells the system whether to unassign equipment while unassigning the equipment type.

Note: If FLEET ASSIGNMENT ALGORITHM is set to Savings Merge this parameter will not be honored.
