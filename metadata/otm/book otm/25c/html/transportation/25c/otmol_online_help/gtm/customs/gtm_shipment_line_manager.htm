Customs

Declaration Lines

Note: Customs Shipment and Customs Shipment Line have been renamed as Declaration and Declaration Line, respectively. However, this change will not impact backend processes in which the object types will continue to be named as GTM SHIPMENT and GTM SHIPMENT LINE.

This page is accessed via:

  • Customs > Declaration Lines.
  • Customs > Declarations. Then click New Declaration Line.

Use this page to manage the declaration lines you work with in GTM. Also, you can check the internal status of the shipment at any time by clicking Status to see the Statuses window.

Note: Flex fields are available on this page. See About Flex Fields for more information.

Identification

The information is read-only in the following fields:

  • Item Name
  • Item Description
  • Preceding Transaction Line

Adding a Declaration Line:

  1. Enter a unique ID in the Declaration Line ID field.
  2. Enter a description of the declaration line in the Line Description field. This description will be displayed when you generate a customs commercial invoice. If this field is blank, item description will be displayed in the customs commercial invoice.
  3. Select the domain in which this declaration line will be active from the Domain Name drop-down list.
  4. Enter an item in the Item IDÂ field.
  5. Enter the declaration ID in the Declaration ID field.
  6. Enter IDs in the Procedure ID and Prior Procedure ID fields to add information related to procedures to the declaration line.
  7. Enter an ID in the Procedure Detail ID field to add procedure details to the declaration line.
  8. Enter an ID in the Trade Preference ID field to apply a trade preference to the declaration line.
  9. Enter a trade program ID in the Trade Programs field. This will be sent to external system to get the duty and tax values related to the trade program for the selected declaration line, and used for landed cost calculation.
  10. Enter an ID in the Bond ID field to assign a bond to the declaration line.
  11. Enter an ID in the Valuation Method field to apply a valuation method to the declaration line.
  12. Enter an Incoterm ID and an Incoterm Location.
  13. Enter an origin country in the Country of Origin field.
  14. Enter the manufacturing country in the Country of Manufacture field.
  15. If this declaration line is a child of another declaration line, enter the ID of the parent line in the Parent Declaration Line field.
  16. Select an option from theÂ Trade Incentive Program Zone Status drop-down list. GTM copies the zone status from the declaration line to the trade incentive program inventoryÂ when it is created.
  17. If you are looking at an existing record, click Status to view the internal statuses assigned to the record. Please note the following:
    • Declaration status will be calculated based on the top level status only.
    • Any line status (which includes its own status and the status of its immediate child lines) will be considered when the status is calculated.
    • Whenever a line's status is calculated, the status of it's parent line along with the declaration's status will be recalculated.

Dates

Use this section to add date information to the declaration .

Adding a Date Qualifier:

  1. Select a qualifier from the Date Qualifier ID drop-down list.
  2. Enter a date in the Date field.
  3. Click Save.

Required Classification

In this section you can view the classifications required on the declaration line. When you run the Look Up Product Classification and Verify Classification actions, required product classification types will be populated in this section depending on the service preference you configure.

These required classifications will be used to set product classification lookup statuses.

Product Classification

Use this section to record product classification information for the declaration line.

Adding a Product Classification:

  1. Enter a type in the Product Classification Type ID field.
  2. Enter a code in the Classification Code field.
  3. Enter a number in the Binding Ruling Number field. Binding rulings could be used to justify a decision to use a particular product classification code for a particular item.
  4. Enter the date from when the binding ruling is effective in the Binding Ruling Effective Date field.
  5. Enter the date till when the binding ruling is effective in the Binding Ruling Expiration Date field.
  6. Select Fixed if you do not want the product classification data to be overwritten by the Product Classification action of the Execute Compliance Rule action.
  7. Select the purpose of the classification code from the Purpose drop-down list.
  8. Click Save.

User-defined Code

Use this section to add user-defined codes to the declaration line.

Adding a User-defined Code:

  1. Select a user-defined code category from the Category drop-down list.
  2. Select a user-defined code type from the Type ID drop-down list.

    Note: The available options are determined by your choice of category in the previous step.

  3. Select a user-defined code value from the Type Code drop-down list.

    Note: The available options are determined by your choice of type in the previous step. For easy understanding of the user, this drop-down list displays description of the user-defined code values instead of only the codes.

  4. Click Save.

Involved Parties

Use this section to add involved party information for the declaration.

The Indicator column corresponding to each involved party displays the restricted party list screening (RPLS) status of the party. RPLS status is indicated as follows:

  • White: indicates status as RPLS_NOT STARTED
  • Green: indicates status as RPLS_PASSED
  • Red: indicates status as RPLS_FAILED
  • Yellow: indicates status as RPLS_REQUIRES REVIEW or RPLS_ESCALATED

Adding an Involved Party:

  1. Select a qualifier from the Involved Party Qualifier ID drop-down list.

    Note: You can specify a Transmitter ID or Filer ID as reference numbers on the party.
    GTM will first check if a transmitter and/or filer is specified as an involved party. If so, GTM will use the reference number on the associated party to populate the Transmitter ID and Filer ID. If this data is not present on the party, then GTM will use the data in theproperties.

  2. Enter a party in the Involved Party field.
  3. Click Save.

Quantity

Use this section to add quantity details to the declaration .

Adding a Quantity Qualifier:

  1. Select an ID from the Quantity Qualifier ID drop-down list
  2. Enter a value in the Quantity field.
  3. Select a unit of measure from the UOM drop-down list.
  4. Click Save.

Values

Use this section to add information about values to the declaration line. You can add as many value qualifiers as you need.

Note: Value qualifiers can also be added to a declaration line by running the Execute Compliance Rules action.

Adding a Value Qualifier:

  1. Select a value qualifier from the Value Qualifier ID drop-down list.
  2. Enter a currency value in the Value field.
  3. Select a currency from the drop-down list.
  4. Enter a formula expression in the Formula Expression ID field. The value of the value qualifier is calculated based on this formula expression.
  5. Select an option from the Method of Calculation drop-down list. Your list of options include the following:
    • Entered Manually: Select this option if you want to specify the value of the value qualifier.
    • System Calculated: Select this option if the value of the value qualifier is calculated using a formula assigned.
    • Calculated Externally: Select this option if the value of the value qualifier is calculated using theÂ duty and tax values obtained from a third party data provider by executing the Estimate Duty and Taxes action.
  6. Select the Fixed checkbox if Â you DO NOT want to recalculate and override the value of the value qualifier using the modified formula on the Compliance Rule page, when the compliance rule is re-executed. Selecting this checkbox will also ensure that the method of calculation specified in the previous step will not be overridden. Clear this checkbox if you want to override the method of calculation and override the value of the value qualifier using the modified formula.
  7. Enter any additional note, if desired, in the Value Note filed.
  8. Select an option from the Payment Method drop-down list.
  9. Click Save.

Note: If the optional feature "ENHANCED DUTY TAX CALCULATION" is enabled andÂ you perform the action Estimate Duty and TaxesÂ on a declaration line that has "DUTY" as a value qualifier, a Details link is displayed against this value qualifier.Â Click this link to view a breakup of the duty and taxes (base duty and details of duty types like penalty, exemption, etc.) for the product classification codes associated with the declaration line.

Percentage Values

Adding a Percentage Value:

  1. Select a percentage qualifier from the Percentage Value Qualifier ID drop-down list.
  2. Enter a percentage in the Value field.
  3. Click Save. You can enter multiple values. However, they should not total more than 100.

Sanction Control Results

This section displays the following data associated with the selected declaration line:

  • Control Type
  • Control Code
  • Compliance Rule Set Group ID
  • Compliance Rule ID

Additionally, if you have an authorization value, you can record the information in the Authorization field. Click Save after entering a value. Please note that if the declaration line has an exception code as a result of screening, you will not be able to record an authorization.

Control Screening Results

This section provides read-only information regarding the control screening results for the declaration line.

Additionally, if you have an authorization value, you can record the information in the Authorization Type and Authorization fields. This will be used to release a hold on the declaration line. Click Save after entering a value. This will release any hold on the declaration line resulting from the control code and control type provided in this section.

Assigned Licenses

In this section you will see information about licenses which have been assigned to this declaration line.

Other Control Screening Results

This section displays screening results based on the control category 'OTHER'. You can see the following data associated with the selected transaction line:

  • Control Type
  • Control Code
  • Compliance Rule Set Group ID
  • Compliance Rule ID

Reporting Results

This section displays data associated with the results of control screening.

Reporting Quantity

This section allows you to record reporting quantity details like product classification type, quantity qualifier, quantity, UOM, and attribute name for the declaration line.

Adding a Reporting Quantity:

  1. Enter a product classification type in the Product Classification Type ID field.
  2. Enter a product classification code in the Classification Code field.
  3. Select a qualifier from the Quantity Qualifier ID drop-down list.
  4. Enter a value in the Quantity field.
  5. Enter a unit of measurement in the UOM field.
  6. Enter a name in the Attribute Name field. This attribute refers to which unit of measurement you select in the previous step. For example, the attribute name could be REPORTING UOM1 or REPORTING UOM2.
  7. Click Save.

Value Formula Results

Note: This section is deprecated and is not supported.

This section will display information about the value formula, compliance rule set group, and compliance rule associated with this declaration line.

Remarks

Use this section to record remarks associated with the declaration .

Adding a Remark:

  1. Select a remark qualifier from the Remark Qualifier ID drop-down list.
  2. Enter your remarks to the Remark Text field.
  3. Click Save.

Reference Numbers

Use this section to record reference numbers that apply to this declaration. You can add as many reference numbers as you require.

Adding a Reference Number:

  1. Select a reference number qualifier from the Reference Number Qualifier ID drop-down list
  2. Enter the reference number in the Reference Number field.
  3. Click Save.

Required Documents

This section will display information about documents which are required for this declaration line.

Produced Documents

Use this section to add information about documents that might have to be added with the declaration line.

Adding Document Information:

  1. Enter and ID in the Produced Document Type ID field.
  2. Enter a number in the Document Number field.
  3. Enter a date in the Document Date field.
  4. Enter locations in the Document Validation Location and Document Archive Location fields.
  5. Enter the archive format in the Document Archive Format field.
  6. Enter the archive details in the Document Archive Detail field.
  7. Click Save.

Text Templates

Use this section to specify templates that will be used with this declaration line.

Adding a Text Template:

  1. Enter an ID in the Text Template ID field.
  2. Enter a type in the Document Type field.
  3. Enter any related text in the Text field.
  4. Click Save to record the information or click Override All if you want your information to overwrite the template information already in the system.

Additional Information

Use this section to add additional information to the declaration line.

Adding Additional Information:

Note: You may need to specify a value in this field for certain scenarios. For example, in the European Union, for certain additional information codes, you have to specify certain country codes as defined in the regulations. Â 

  1. Enter an ID in the Additional Information ID field.
  2. Enter the information code in the Code field.
  3. Enter details in the Text field.
  4. Enter relevant country in the Export From field.
  5. Click Save.

Reference Lines

If this declaration line is created through aggregation of several transaction or declaration lines, in this section you can see the list of those lines. For each line, Reference Line ID, Quantity Qualifier ID, Quantity, UOM (Unit of Measure), and Primary Line information is displayed.

Declaration History

This section displays any note associated with this declaration line.

Here you can view the date and time when the note was created, a summary and the details of the note, and how it got assigned to this declaration line.

Qualifications

If you are viewing/editing a declaration line, this section displays qualification information of the declaration line.

Adding a Qualification:

  1. Click Add Qualification. The Declaration Line Qualification page is displayed where you can enter the qualification details.
  2. Select an option from the Status field to indicate whether the declaration line is qualified or if it requires to be reviewed.
  3. Enter an ID in the Trade AgreementÂ field.
  4. Enter a producer in the Producer field. An example of a producer is a manufacturer.
  5. Select an option from the Minimum Operations Met drop-down list to specify whether the line satisfies the requirements for minimum operations.
  6. Select an option from the Wholly Originating drop-down list to specify whether the line wholly originates from a region/country.
  7. Enter an ID in the Regional Value Content Method field.
  8. Enter a percentage value in the Regional Value Content Percentage field.
  9. Enter an ID in the Preference Criteria field.
  10. In the Values section, do the following:
    1. Select a value qualifier from the Value Qualifier ID drop-down list.
    2. Enter a currency value in the Value field.
    3. Select a currency from the drop-down list.
    4. Select an exchange rate from Â the Exchange Rate ID drop-down list.
    5. Enter the date for which the exchange rate is to be applied in the Exchange Rate Date field.
    6. Click Save.
  11. The Conditions section displays the conditions applicable for the declaration line.
  12. Click Save.
  13. Click Finished when you have recorded all of the information you need on this page.
  14. Click View Qualification Report to see the qualification analysis details of the declaration line.
