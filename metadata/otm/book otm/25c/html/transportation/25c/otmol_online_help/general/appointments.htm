Set Appointments

This page is accessed via:

  • Order Management > Order Release > Order Release > Actions
  • Shipment Management > Shipment Management > Shipment Stops > Actions
  • Brokerage Forwarding > Booking > Freight Forwarding >Actions
  • Brokerage Forwarding > Job Management > Job Details > Order Releases > Actions

Pickup/delivery appointments represent the date/time that a service provider arranges for pickup and delivery as opposed to the Oracle Transportation Management-planned pickup and delivery dates and times. Appointments can be recorded for any business object that has pickup/delivery dates such as:

  • Order Release
  • Shipment Stop

Appointments can be manually recorded for each of these business objects, applied via XML integration, or supplied through shipment tracking events provided by the service provider.

Note: For appointment management, Oracle Transportation Management assumes that the activities for the various ship units are the same. For example, if you have two ship units, Oracle Transportation Management assumes they are both either picked up or dropped off.

Order Appointments

To manually apply appointments to order objects, edit any of the objects mentioned above and enter early or late pickup and delivery dates. Then mark the Pickup and/or Delivery Appointment checkboxes. If you enter both early and late dates, only the early date is used to propagate appointments to subsequent objects (for example, appointments from an order base to an order release or an order release to a shipment). For example, if you record an early and late pickup date as a pickup appointment, the early date is used (when copying appoints from an order release to a shipment). When creating order releases from an order base, the complete appointment time window is copied.

Also, if you apply appointments across multiple objects such as several ship units or line items on one order base, Oracle Transportation Management runs a time window analysis similar to what is performed in the order bundling logic to determine the appointment dates on the order release.

Shipment Stop Appointments

To manually apply appointments on a shipment stop, go to the Shipment Stop tab and edit a stop. Then, enter a date in the Appointment Pickup and Delivery Time fields. These fields are automatically populated if the order release that created the shipment has appointments defined.

Note: Â When you set this glog.appointment.shipmentstop.updateAppointmentTime property to true, the Appointment Pickup and Delivery Time fields are automatically populated.

Note: Oracle Transportation Management uses the stop appointments in planning functions. For example, if there is a delivery appointment specified and the driver arrives 3 hours early at a stop (as specified by entering an Actual Arrival Time), you will see a 3 hour wait time before activity starts at the stop. This calculation is only performed when a planning action is run on the shipment, not by merely editing and saving the shipment.

Set Appointments Action

The procedure for using this action is the same as if you manually entered appointments; however, in addition to setting appointments, this action also allows you to record appointment numbers as reference numbers on the object from which you are running the action. The reference number section of the page displays all existing reference numbers for the object that you can edit or delete. If you assign an appointment number as a reference number, you may also consider using the AO public qualifier which has a predefined description of Appointment Number.

If you run this action from an order base, you can set a pickup/delivery appointment for all ship units or all line items on the order at one time.

If you set appointments on stops using this action, the times are checked for feasibility. For example, if you make an appointment so late that there is not enough time to make an appointment for a subsequent stop, Oracle Transportation Management will flag it as infeasible.

View Appointments SmartLink

You can view appointments by editing the appropriate pages for the business objects mentioned above. You can also use the View Appointments SmartLink to view appointment data and other related business object information.

This SmartLink is available for the same business objects as the Set Appointment action mentioned above except Shipment Stops.
