Fleet Management

Driver Manager

This page is accessed via Fleet Management > Driver Management > Driver Manager.

When searching for drivers, besides the normal search fields there are also derived or calculated fields you can search by. See Calculated Fields-Driver Manager for details.

The driver manager contains the basic driver information.

Adding a Driver

  1. Enter a Driver ID. You can also have the system generate an ID using the Business Number Generator. If you leave this field blank, Oracle Transportation Management automatically assigns a default ID.
  2. Enter the driver's First Name.
  3. Enter the driver's Last Name.
  4. Enter the driver's Date of Birth.
  5. Enter the Adjusted Hire Date.
  6. If this is a current driver, select the Active checkbox.
  7. Enter the Default Home Location ID.Â Driver distance away from home is calculated using the following hierarchy: first, the Calendar Event Location on the Driver Calendar Event page. Next, the Home Location ID in the Work Profile section on the Driver manager, and if that is null, then the value entered in this field is used. Period based HOS Rules start at midnight to define the driver's day. The time zone used is that of the default home location. If the driver does not have a default home location and it is a fresh driver (no driver assignment records), then the driver is assumed to be at the shipment source location. If the driver does not have a default home location, the driver's time zone is considered to be UTC time so period based rules would start at midnight UTC time.
  8. Enter the Work Region ID.
  9. Enter a Contact ID.
  10. Enter a Driver Type ID.
  11. Select a Resource Type. This indicates the employment type of the driver. Values include: Employee; Independent Contractor; Third Party Driver.
  12. Select the Use HOS History checkbox to indicate that a driver's previous assignments, events, and HOS (hours of service) history should be factored in when making a given shipment assignment. The driver's previous assignments, events, and HOS history all contribute towards consuming time against the driver's individual HOS rules. The checkbox does not need to be checked for the NAT to be honored. If the checkbox is not selected, the driver is considered fresh for the assignment.
  13. Enter a Driver Configuration ID.
  14. Enter a Rate Offering ID.
  15. Enter a Rate Distance ID. This ID is used to determine the distance from one point to another.
  16. Enter an End Date. This indicates the last day the driver is available. The shipments' start date and end dates must both fall within the drivers' effective and end date range for an assignment to be feasible, in which case, the driver is assigned to the shipment.
    If the driver assignment itself pushes the end date of the shipment past the drivers' expiration date, it will be allowed.

Work Profile

The work profile links the activity calendar and the home location. This activity calendar typically defines the work shift profile (times) and the global non-working times (including holidays). This allows for switching between day shift and night shift or seasonality.

When no Work Profiles are found for the driver, the Driver Type Work Profile is checked.

  1. Enter a Home Location ID. This is where the driver's work shift typically starts. Driver distance away from home is calculated using the following hierarchy: first, the Calendar Event Location on the Driver Calendar Event page. Next, this field, and finally, the Default Home Location ID on the Driver header. This hierarchy is used when the Get The Driver Home functionality is applied when assigning drivers and shipments.
    If the Driver doesn't have a home location and it is a fresh driver (no Driver Assignment records), then the driver is assumed to be at the shipment source location.
  2. Enter an Activity Calendar.
  3. Select whether the work profile is Shift Based or not. If you set this to Yes, then the work profile is evaluated when you try and get the driver home. The driver will be sent home at the end of each shift, and they will start from their home location for their next assignment.
  4. Enter an Effective Date and Expiration Date.
  5. Click Save for each work profile you define.

Power Units

When you assign a driver, the associated power unit and the equipment associated with the power unit will be assigned. When you establish a relationship between a driver and a linked power unit and a piece of equipment, it may be referred to as a "kit".

  1. Enter a Power Unit ID.
  2. Enter a Calendar ID to indicate the day and times when the power unit can be used with the driver. For example, if you select a calendar for Monday through Friday, 8 a.m.-8 p.m., that power unit would be used with that driver Monday through Friday, 8 a.m.-8 p.m.. On a different driver you could then select a time of Monday through Friday, 8 p.m. -8 a.m., as well as on any time on Saturday and Sunday, and assign the same power unit to a different driver. Only calendars with an activity of Driver Work will be considered for overlap when other drivers share the same power unit. Validation checks against other calendars for duplicate assignments when you save. By establishing power unit availability to multiple drivers through the use of a calendar, you are defining "slip seating".
  3. Enter an Effective Date and Expiration Date. You can enter times as well as dates here. You may want to specify a time if the tractor is slip seated, meaning that more than one driver shares a tractor for a given day. Power units can be shared by multiple drivers on the same day if the hours on their calendars do not overlap.
  4. Click Save for each power unit you associate with this driver.
