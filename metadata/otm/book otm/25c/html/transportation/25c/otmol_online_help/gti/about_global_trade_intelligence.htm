Global Trade Intelligence

About Global Trade Intelligence

Global Trade Intelligence provides an environment for managing business metrics by monitoring, analyzing, and reporting metrics at various levels of the organization. To provide access to Global Trade Intelligence, you need to add a business intelligence application and a business intelligence role to each user as appropriate to access different functionality. Global Trade Intelligence is based on Oracle Analytics Server (OAS). Read the documentation for Oracle Analytics ServerÂ found on the Oracle Help Center.

Overview

OAS is the tool that was used to build Global Trade Intelligence. OAS enables Oracle Global Trade Management to query, format, and display the information you see in Global Trade Intelligence. Global Trade Intelligence is the set of rules that govern how operational data is moved from Oracle Global Trade Management to the historical database, defines metrics, and displays the results. The historical database is optimized for read-only queries along with pre-determined dimensions.

The process of moving data between databases is often referred to as ETL for Extract, Transform, and Load. The HD Star Schema refers to the way data is organized in business intelligence applications.

The data that is available to you is of two types: facts and dimensions.

  • Facts are typically numbers and can be added together. Examples of facts include shipment weight and number of orders.
  • Dimensions are often a non-number type of data and allow you to group the facts. Examples of dimensions include time periods, transport modes, and destination locations.

The final result is a metric, a measurement of which you want to keep track. You can visualize metrics in formatted reports. Or, you can display multiple metrics on a dashboard.

Global Trade Intelligence contains the following components.

Dashboard

A Dashboard is a collection of performance metrics reflecting common strategic goals. The metrics are displayed as various key performance indicators formatted as charts, graphs, tables, cross-tabs, and status indicators. There are multiple tabs organizing the metrics. Dashboard comes pre-defined with standard metrics. This page is accessed via Global Trade Intelligence > Dashboard. In user preferences, you can define a default dashboard. Dashboards are user-configurable. See My Dashboard below for details.

Note: A user can only have one default dashboard set at a time. As a result, a single user cannot have both an Transportation Intelligence (TI) default dashboard and a GTI default dashboard. To view both the TI and GTI dashboards, create one user with access to TI and set their dashboard preference to point to the TI dashboard. Then, create a second user with access to GTI and set their dashboard preference to point to the GTI dashboard.

My Dashboard

It is a configurable Dashboard so you can display the metrics you want in the format you want. This is in OAS. You should have an OAS license to work in this page. For details on My Dashboard, click the OAS help icon on the Dashboard Editor page. This page is accessed via Global Trade Intelligence > My Dashboard. Then click the available link.

Ad Hoc Query

This component allows creation of reports on the fly. For details on Ad Hoc Queries, click the OAS help icon. This page is accessed via Global Trade Intelligence > Ad Hoc Query. Then under Analysis and Interactive Reporting select Analysis > Global Trade Intelligence. The Global Trade Intelligence subject area contains data for the 6.3 version and later. You should have an OAS license to work in this page.

Catalog

Any queries that you have created and saved appear under Catalog > My Folders and are only be available to you.

Any queries that the GTI administrator created and saved appear under Catalog > Shared Folders. Any GTI user can access these queries.

Important! All user-defined reports and dashboards must be created inside the shared/Custom folder or a sub-folder. Defining reports and dashboards inside the shared/Custom folder will ensure your user-defined reports and dashboards are retained during upgrades. All out-of-the-box folders are read-only so you cannot save content in folders other than shared/Custom. If you try to save content outside of shared/Custom, you will see an error which may be one of the following: "Access denied for user to path..." or "Can't save in root folder.".

These folders permissions may be modified by a user with BiAdministrator privileges. This allows you to save content, both user-defined and out-of-the-box, outside of the shared/Custom folder; however, if you save content outside of the shared/Custom folder it will be lost during migration!

Data Visualization

This component uses GTI specific metadata to let you select required column combinations from the Global Trade Operational Intelligence subject area and display the results accordingly.

You can also view the data visualization workbook Restricted Party here.

Any queries that you have created and saved appear under Catalog > My Folders and are only be available to you.

Any queries that the GTI administrator created and saved appear under Catalog > Shared Folders. Any GTI user can access these queries.

Administration

Controls administrative functions such as managing sessions. This page is accessed via Â Global Trade Intelligence > Administration. This is in OAS. You should have an OAS license to work in this page. For help with Administration, click the OAS help icon.

Since this an OAS page, you see the same screen that is shown when administering OAS. Just because a link or function is show on this page, does not indicate that it is supported by Oracle Global Trade Intelligence. For example, Oracle Analytics Publisher is not supported as part of Oracle Global Trade Intelligence, but it will appear on this page since it is installed by OAS.

Note: This functionality is only available if you are logged in as a DBA.

Alerts

This component allows creation of notification. You can determine the priority, recipients, the schedule, and conditional requests among other criteria. This is accessed via Global Trade Intelligence > Alerts. This is in OAS. Since this an OAS page, you see the same screen that is shown when administering OAS. You should have an OAS license to work in this page. For help with Alerts which are called Agents in OAS, click the OAS help link.

Data Management

You can use Data Management to view information related to the data load including the logs and rejects.

Note: This functionality is only available if you are logged in as a DBA.

Process Management

Process Management allows you to manually push data to the HD (historic database). See the topic Data Flow to Global Trade Intelligence.

Note: This functionality is only available if you are logged in as a DBA.

Power Data

Power Data allows you to view and manage setup data for Global Trade Intelligence including Unload Object from HD and Upload Target CSV. Additionally, you can define dashboards.
