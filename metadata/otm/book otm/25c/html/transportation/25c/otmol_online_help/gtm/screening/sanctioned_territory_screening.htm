Sanctioned Territory Screening

This page is accessed via:

  • Compliance > Sanctioned Territory Screening
  • Ask Oracle > Sanctioned Territory Screening

Use this page to screen territories against your list of sanctioned territories.

Note: None of the fields on this page are required to be completed. The screening will be performed using whichever criteria you designate.

Note: A status will be assigned when the screening process is completed. For more information about internal status designations, see GTM Internal Status Designations.

Screening Criteria

Enter a Compliance Rule Set Group ID. This is considered as an optional input for sanctioned territory screening. If no rule group is provided, the service preference given in user preference is considered for getting rule group information. If a user preference has not been designated, screening will be done against all sanction rules using the Cumulative algorithm.

Territory Information

  1. Enter an ID in the Involved Country Qualifier ID field.
  2. Enter a code in the Country Code field.
  3. Click Save.

Note: You can add multiple lines of territory information. Remember to click Save after adding each line of information.

Reference Information

  1. Click beside the Transaction Date field to open a calendar on which you can select a transaction date.
  2. Click Match to perform the screening.
