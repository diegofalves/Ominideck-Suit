CM CONSOLIDATION ALGORITHM TYPE

CM Consolidation

6.3.3

Selection mechanism for the algorithm used to create continuous move tours.
Supported values are as follows:

  • Sequential savings - Chooses one shipment at a time and consolidates others with that shipment until no further consolidation is possible. Sequential savings sorts the shipments in decreasing order by weighted cost and then by weight. If you employ this algorithm, use logic configuration parameters only.
  • Greedy heuristic - This is the original continuous moves algorithm. When you employ this algorithm, configure your options using parameters.
  • Column generationÂ - Uses other seed algorithms to generate various combinations of continuous move tours. If you employ this algorithm, use logic configuration parameters only.
  • Concurrent SavingsÂ - Used to build continuous move tours and allows for multiple pairs of shipments to be considered and combined simultaneously. If you employ this algorithm, use logic configuration parameters only.
  • Complete enumerationÂ - Enumerates all possible combinations using the same business logic that is used in multi-stop algorithms to discard unnecessary combinations. The retained shipment combinations are passed into an algorithm to completely optimize and find the best set of shipments to form optimal tours. The solution is generally the best but with a significantly longer run time. This approach is not meant for large problems. If you employ this algorithm, use logic configuration parameters only.

See also Bulk Plan Tuning Continuous Moves Logic.

CM COST TOLERANCE IN MERGE ALGORITHM

General

6.3.4

This parameter is used to define the cost tolerance to be used by the savings algorithm.

When set to the default (0), continuous move tours are not formed if there is no cost benefit. The combined cost of all shipments in a tour should always be less than the un-combined costs for the tour.

If you set the cost tolerance value for this parameter to a positive value, it will encourage adding more shipments to the continuous tour even if there is no cost benefit. This only applies when using Distance baed savings option on the CM SAVINGS CALCULATION TYPE parameter.

CM DISCOUNT PERCENTAGE

CM Consolidation

6.3.3

The discount percentage is applied to the common distance value during the savings calculation for the continuous move merge algorithms when the SM SAVINGS CALCULATION TYPE parameter is defined as "Distance-based Savings." The higher the discount percentage value, the greater the distance savings. Also, the probability of placing more shipments into a continuous move tour is greater with a higher discount percentage.

Range of values = 0 to 100%.

Default value = 10%

CM ENUMERATION MERGE MAX SHIPMENTS

6.3.3

CM IGNORE RATE OFFERING RULE CHECKS

General

6.3.4

This parameter, when set to True, adds a check before executing the time consuming drive and rating algorithm so as to avoid the evaluation of unwanted continuous move formations, thus enhancing the overall performance.

The savings algorithm uses "find savings" to calculate the savings of each continuous moves formation as well as combining processes in order to string shipments that will form a continuous moves tour.

Default = False.

CM RETURN DISTANCE COST FACTOR

CM Sequencing

20A

As the setting is increased from zero (the default) the "cost" associated with returning from the end location of the continuous moves (CMs) last shipment back to the start location of the CM's first shipment is increased when considering the overall cost of the CM.

This applies to both cost based and distance based CM algorithms. This extra cost only applies to CMs, not other shipment strings created with CM logic (e.g. work assignments, driver assignments). The extra cost is only used in determining CM formation. Â The cost is not persisted or added to any existing cost in any of the shipments associated with the CM.

Using this cost has the effect of creating CM shipment strings that form a "loop" such that the last shipment in the string ends closer to the start of the first shipment in the string.

The default of zero (0) has the same functionality as prior to addition of this parameter.

CM SKIP DEPOTS WITHIN TOUR

CM Sequencing

6.3.3

Lets you select whether or not to keep depots within a continuous moves tour.

Setting this to TRUE will only keep the depot stops of the first stop of the first shipment, and/or the last stop of the last shipment in a tour as defined by the depot profile. Â Setting this to FALSE will include the depot stops for every shipment in the tour. Â For this parameter to be used, you must enable depot functionality.

Default - FALSE.

CM USE GENERIC SIMULATION DRIVE

General

6.3.4

When the parameter is set to true, the system uses the rate service defined in the parameter GENERIC ESTIMATION RATE SERVICE ID to calculate the rate service during continuous move pairing. However, at the end of the continuous move logic, for the final results, each continuous move tour is recalculated using the rate services defined on the shipments.

When this parameter is set to false, the system uses the rate service on the shipments during pairing and for the final continuous move result to calculate service time.

CM WAIT TIME PENALTY

CM Consolidation

6.3.3

Allows you to enter the wait time penalty (range: 0 to 100) to be considered for continuous moves. The default value is 0, which means no penalty.Â The purpose of this penalty is to avoid building continuous moves with large wait times in between shipments.
