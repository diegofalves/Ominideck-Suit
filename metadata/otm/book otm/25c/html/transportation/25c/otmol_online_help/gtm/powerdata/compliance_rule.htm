Compliance Rule: Identification

This page is accessed via Master Data > Power Data > Compliance Rules > Compliance Rules.

Use this page to view the existing rules available to you. You can configure screening to fit any combination of rules or policies that you deem to be necessary. Compliance rules can be added to a rule set, rule sets can be added to rule set groups, and rule set groups can be combined into a compliance policy which is then used in transaction screening.

Identification

  1. Enter an ID in the Compliance Rule ID field. You will use this ID when you refer to this rule.
  2. Select a control category from the Control Category drop-down list. Controls are organized into categories. The categories give you the ability to construct rules of different types. For example, if you wanted to write a control to screen for a particular license exception, you would select License Exception. More information about the categories is located at Compliance Rule: Controls.

    Note: Once you have selected a category and moved to another tab or saved the rule, you will not be able to edit this field.

  3. Enter a description of the rule in the Description text box.
  4. Select the domain where this rule will be active from the Domain Name drop-down list.
  5. If this rule will relate directly to an existing regulation, enter the regulation's ID in the Regulation Reference ID field.
  6. If needed, select the exact date the license exception is to occur (effective date) and the final day on which the license exception is effective (expiration date).
  7. If the new rule will be active immediately, select the Active checkbox.
  8. If this rule is to apply at the transaction line, select the For Transaction Line checkbox.

Note: If the For Transaction Line checkbox is selected, screening will take place against the date mentioned on the transaction line.

  1. Select an override control type from the Override Control Type drop-down list.

Transaction Line Values

This section enables you to set up the values needed for the reporting determination service.

  1. Select a value qualifier from the Value Qualifier ID drop-down list. This indicates the currency qualifier that should be used for the aggregation of currency on transaction lines.
  2. Select the AggregateÂ checkbox if this line should be aggregated to use with the Automatic Export System (AES). This option will determine whether the currency comparison should be done with or without aggregation of currencies of all the transaction lines.
  3. Enter the appropriate classification type in the Product Classification Type field. The transaction lines used for aggregation must be classified under the specified product classification type.

    Note: The Product Classification Type and the Aggregate fields are deprecated and is not supported. As an alternative, you can use the aggregate constraint set to aggregate the lines by classification type and product classification code. Values on a compliance rule should be used for direct value comparison with declaration line without aggregation.

  4. Select an operator from the Operator drop-down list. This is the operator that will be used for comparison.
  5. Enter a value in the Value field. This is the currency value to be used for comparison.
  6. Select a currency from the drop-down list.
  7. Click Save.

Transaction Line Saved Query

  1. Enter a saved query in the Saved Query field.
  2. Click Save.

Regions

  1. Select a qualifier ID from the Region Qualifier ID drop-down list.
  2. Enter an ID in the Region ID field.
  3. Select the Included checkbox if the region is to be included in the qualifier. This means that when the region qualifier is used in a compliance rule, the rule will apply to anything that is part of this qualifier. However, whenever the checkbox is NOT selected, the compliance rule will not apply to any transactions that apply to this region qualifier.

    Also, a region qualifier that is NOT included, does not automatically enforce all remaining data to be included. This means that multiple region qualifiers must be assigned for exclusive criteria to have the effect of true exclusivity. A compliance rule with only a NOT inclusive region qualifier will automatically exclude everything, unless a more encompassing qualifier is also assigned.
  4. Click Save.

Country

  1. Select a qualifier ID from the Country Qualifier ID drop-down list.
  2. Enter an ID in the Country Code field.
  3. Click Save.

Note:
Region criteria on compliance rule: When you define multiple region records on a compliance rule, the region criteria are prepared by taking a union (OR condition) of all the included regions minus the excluded regions defined for the same region qualifier.
For example, for a compliance rule with 3 region records - 1) SHIP_FROM,EU_REGION, 2) SHIP_FROM,CHILE_REGION, and 3) SHIP_TO, USA, GTM prepares 2 region criteria SHIP_FROM,(EU_REGION + CHILE_REGION) and SHIP_TO,USA.
Any trade transaction or its line with SHIP_FROM country as part of either EU_REGION or CHILE_REGION will be considered as a match to SHIP_FROM region criteria.

All other grids on compliance rule: Each record on any of the grids of compliance rule is considered as an independent criterion. All the criteria must match with the compliance rule to apply any controls.

Saved query criteria: Transaction Line GID is passed as input to saved query for any line compliance rule. Transaction GID is passed as input to saved query for any non-line compliance rule (like policy, transaction level document, etc.). Each saved query criteria is taken as independent criterion and must match.

Copy Compliance Rule

If you are editing an existing compliance rule, you have the option of copying the rule in order to build a new rule.
  1. Click Copy Compliance Rule. The page refreshes and contains the same information, but the Compliance Rule ID field is blank.
  2. Make any changes to the rule if needed.
  3. Enter the new ID in the Compliance Rule ID field.
  4. Click Finished.
