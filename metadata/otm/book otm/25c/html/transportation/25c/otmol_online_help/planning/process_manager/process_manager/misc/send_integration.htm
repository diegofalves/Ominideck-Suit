Send Integration

This page is accessed via Business Process Automation > Process Management > Send Integration.

Use the Send Integration page in the Process Manager to send interface transmissions to external systems. You can set the send integration process to run immediately or at regular intervals.

  1. From the Interface Type drop-down list, select the type of interface transmission you want to send such as Bulk Plan, Order Accrual, Rate, etc.
  2. Select the query you want to use to select transmissions in the Based on field.

    The drop-down list displays any query that you have saved for the type of transmission entered in Step 1.

    Note: You can also create a new query and save it for use by clicking Define. Once you create a new query, you must exit this process and restart it to have it appear on the drop-down list.

  3. Enter a saved condition in the Or on condition field. This field also allows you to create a new saved condition by defining a new query using SQL.
  4. Select the Notify if no data is found checkbox if you would still like to receive notification even if no data results from your query.

    This field restricts email to queries which return results, however, there may be a situation in which someone wants to be notified even in the event that there are no results (for example, where no results are a result).
  5. Select the Create New Process checkbox to launch the sending of the integration as a new process. Usually sending an integration is part of chain of processes. If you select this checkbox it will be a separate process. Spawning a new process helps with memory management. Since each Send Integration process is spawned as a separate process it allows memory to be reclaimed as each process finishes.
  6. Select how you want to send notifications in the Notify Method field.
  7. Enter the name of the system to which you are sending the transmission in the Receiving External System field.

    Note: If the system to which you want to send the transmission does not appear, you must define the system.

  8. Select Send Summary Only to send a transmission that contains an ID only. For example, if you are sending a shipment transmission, select the Send Summary checkbox and Oracle Transportation Management sends only the shipment ID to the foreign system.
  9. Select the Transaction Code.
  10. Select the Reference Object Transaction Code.
  11. Enter any text that you want to send with the transmission in the Remark field.
  12. The Guaranteed Delivery checkbox is reserved for future use.
  13. Enter a User Role. When you run, publish, or schedule a recurring process, you can specify the user role used to perform the process, just like when you run agents. The user role can be used to specify a separate VPD role or another domain to run the agent. When a recurring process is modified, the user role of the recurring process can be modified as well.
  14. Select a Log ProfileÂ to generate focused logging when reproducing an error for the specific action. You cannot select a log profile when you choose to schedule a process, as automatic generation of action logs is not supported.
  15. Select Execute, Publish, or Schedule to determine when the process will be executed.
  16. Click Submit.

When the Successfully Scheduled page displays, click View Execution Details to access the Process Manager page for more information about the process you just created.
