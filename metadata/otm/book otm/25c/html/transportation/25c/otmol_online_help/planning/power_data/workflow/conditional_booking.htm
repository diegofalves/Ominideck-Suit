Conditional Booking Profile

This page is accessed via Business Process Automation > Power Data > Event Management > Conditional Booking Profile.

When you accept a tender, you agree to transport a shipment. Oracle Transportation Management allows service providers to accept a tender with certain conditions. Conditional booking profiles can be defined so certain parameters are considered conditional.

Note: The Conditional booking page only appears if the service provider is capable of accepting conditional bookings.

Conditional booking is often used with ocean bookings. For ocean bookings, it is common practice for the service provider to send back modified information as part of the booking acceptance.

Note: Conditional booking is a process applicable to both the booking and tendering processes. It does not apply to broadcast tender and spot bid.

When a tender response is sent out, certain values on the shipment are compared to determine if an acceptance from a service provider is conditional. When tendering, the system uses the conditional booking profile specified on a service provider that is associated with the transport mode set on the shipment. If you do not specify the conditional booking profile on a service provider, the system uses the one on the transport mode specified on the shipment. If a transport mode is not on the shipment, then the system uses the conditional booking profile specified on the domain.

Adding a Conditional Booking Profile

  1. Enter a unique identifier in the Conditional Booking Profile ID field.
  2. Select a domain in which this booking profile will be active from the Domain Name drop-down list.
  3. Enter a Conditional Booking Profile Name.
  4. Specify if the profile will Support Equipment Modification. If this is selected, then the service provider is allowed to change the equipment as a condition of accepting the tender.
  5. Use the Conditional Booking Saved Condition field to specify specific conditions that must be true in order for the modified tender to be conditionally accepted.
  6. Select a Conditional Booking Accept Remark Qualifier ID. Conditional booking accept remark qualifiers are used to pass in additional conditions of acceptance by the service provider. The way the conditional booking remarks work, is if the service provider conditions of acceptance are not for one of the standard fields that can be picked in the conditional booking profile, conditional booking remarks can be used to pass this information back from the service provider to the planner.

Conditional Booking Fields

  1. Select a Conditional Booking Field ID. The conditional booking field lists all the values you can define as conditional booking parameters.
  2. Enter a Sequence number. The Sequence field enables you to sequence the fields so the conditions are checked in a specific order.
  3. Click Save for each conditional booking field ID you select.

Generated Stop Time Fields

The generated fields allow you to specify information for conditional booking that will appear for each stop.
  1. Select a Stop Time Field ID.
  2. Enter a Sequence number. The Sequence field enables you to sequence the fields so the conditions are checked in a specific order.
  3. Click Save for each Stop Time Field ID you select.
  4. Click Finished.
