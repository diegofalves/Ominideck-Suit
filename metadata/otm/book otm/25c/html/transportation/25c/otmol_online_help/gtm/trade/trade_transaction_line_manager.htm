Transactions

Trade Transaction Lines

This page is accessed via:

  • Transactions > Trade Transactions. Click a transaction. When the screen refreshes, edit the transaction line row or click New Transaction Line.
  • Transactions > Trade Transaction Lines.

Use this page to add, view, or edit existing transaction lines. The top section contains mostly read-only information about the transaction line, including status information for classification, transaction parties, compliance rule screening, and license availability.

Note: If you make changes to any quantity or value, you should re-run the license assignment process.

If you make changes to any of the following fields either via the user interface or by integration, you must re-screen the transaction once you have made the changes:

  • Control Type ID
  • Control Type Code
  • Product Classification Type
  • Product Classification Code ID
  • Item ID

Note: Flex fields are available on this page. See About Flex Fields for more information.

Transaction Line

This section contains basic information about the transaction line.

Adding a Transaction Line

  1. Enter a unique ID to the Transaction Line ID field.
  2. Enter a trade transaction ID in the Transaction ID field.
  3. Enter a description of the transaction line in the Line Description field. This description will be displayed when you generate a customs commercial invoice. If this field is blank, item description will be displayed in the customs commercial invoice.
  4. Select the Canceled checkbox to indicate that the transaction line is canceled. In this case, the transaction line will not be considered when running the action Estimate Duty and Taxes.
  5. Enter an item ID in the Item ID field.
  6. Enter a trade item structure in the Item Structure ID field.
  7. Enter the name of the Country of Origin and the Country of Manufacture.
  8. Enter a trade preference in the Trade Preference ID field.
  9. Enter a trade program ID in the Trade Program field. This will be sent to external system to get the duty and tax values related to the trade program for the selected transaction line, and used for landed cost calculation.
  10. Enter an Incoterm ID and an Incoterm Location.
  11. If this transaction line is a child of another transaction line, enter the ID of the parent line in the Parent Transaction Line ID field.

    Note: A transaction line cannot refer itself as a parent.

  12. You can enter country codes in the fields Country of Export and Country of Import when the property gtm.transaction.useCountryOfImportExportDirectField is true.

    Note: Since the fields Country of Export and Country of Import are not fully supported for any business logic, it is recommended that the property gtm.transaction.useCountryOfImportExportDirectField be set as false, and you do not use these fields for any business logic.

  13. If you are looking at an existing record, click Status to view the internal statuses assigned to the record. Please note the following:
    • Transaction status will be calculated based on the top level status only.
    • Any line status (which includes its own status and the status of its immediate child lines) will be considered when the status is calculated.
    • Whenever a line's status is calculated, the status of it's parent line along with the transaction's status will be recalculated.

    Note: When a structure is assigned via compliance rule, theÂ Component of Transaction Item StructureÂ field identifies the transaction item structure of the child transaction line.

Dates

Use this section to add date information to the declaration .

Adding a Date Qualifier

  1. Select a qualifier from the Date Qualifier ID drop-down list.
  2. Enter a date in the Date field.
  3. Click Save.

Required Classification

Note: This section will be visible when the property gtm.transaction.showRequiredClassificationGrid is set to true.

In this section you can view the classifications required on the declaration line. When you run the Look Up Product Classification and Verify Classification actions, required product classification types will be populated in this section depending on the service preference you configure.

These required classifications will be used to set product classification lookup statuses.

Product Classification

  1. Enter a classification type in the Product Classification Type ID field.
  2. Enter a code in the Product Classification Code field.

    Note: Global Trade Management will check to see if the product classification is valid and matches the selected classification type. If the classification is valid, you will see Y in the Current Data Version column. If N is displayed, the classification is not valid.

  3. EnterÂ a number in the Binding Ruling Number field. Binding rulings could be used to justify a decision to use a particular product classification code for a particular item.
  4. Enter the date from when the binding ruling is effective in the Binding Ruling Effective Date field.
  5. Enter the date till when the binding ruling is effective in the Binding Ruling Expiration Date field.
  6. Select Fixed if you do not want the product classification data to be overwritten by the Product Classification action of the Execute Compliance Rule action.
  7. Select the purpose of the classification code from the Purpose drop-down list.
  8. Click Save.
  9. Click Verify Classifications to view the classifications required on the transaction line. When you click this button, the required product classification types will be populated depending on the service preference you configure.

These required classifications will be used to set product classification lookup statuses.

Note: The Verify Classifications link will be visible when the property gtm.transaction.showRequiredClassificationGrid is set to false.

Quantity

  1. Select a qualifier in the Quantity Qualifier ID field.
  2. Enter a quantity in the Quantity field.
  3. Select a UOM (unit of measurement) from the UOM drop-down list.
  4. Click Save.

Values

Use this section to add values and value qualifiers that can apply to a transaction line. You can add as many value qualifiers as you need.

Note: Value qualifiers can also be added to a transaction line by running the Execute Compliance Rules action.

Adding a Value Qualifier

  1. Select a value qualifier from the Value Qualifier ID drop-down list.
  2. Enter a currency value in the Value field.
  3. Select a currency from the drop-down list.
  4. Enter a formula expression in the Formula Expression ID field. The value of the value qualifier is calculated based on this formula expression.
  5. Select an option from the Method of Calculation drop-down list. Your list of options include the following:
    • Entered Manually: Select this option if you want to specify the value of the value qualifier.
    • System Calculated: Select this option if the value of the value qualifier is calculated using a formula assigned.
    • Calculated Externally: Select this option if the value of the value qualifier is calculated using theÂ duty and tax values obtained from a third party data provider by executing the Estimate Duty and Taxes action.
  6. Select the Fixed checkbox ifÂ you DO NOT want to recalculate and override the value of the value qualifier using the modified formula on the Compliance Rule page, when the compliance rule is re-executed. Selecting this checkbox will also ensure that the method of calculation specified in the previous step will not be overridden. Clear this checkbox if you want to override the method of calculation and override the value of the value qualifier using the modified formula.
  7. Enter an additional note, if desired, in the Value Note field.
  8. Select an option from the Payment Method drop-down list.
  9. Click Save.

Note: If the optional feature "ENHANCED DUTY TAX CALCULATION" is enabled andÂ you perform the action Estimate Duty and TaxesÂ on a transaction line that has "DUTY" as a value qualifier, a Details link is displayed against this value qualifier.Â Click this link to view a breakup of the duty and taxes (base duty and details of duty types like penalty, exemption, etc.) for the product classification codes associated with the transaction line.

Percentage Values

  1. Select a percentage qualifier from the Percentage Value Qualifier ID drop-down list.
  2. Enter a percentage in the Value field.
  3. Click Save. You can enter multiple values. They should not total more than 100.

Involved Parties

Use this section to record involved party information.

Note: If a child transaction line is associated with the parent transaction line, you cannot view this section in the child Trade Transaction Line page.

Note: Address information for each selected involved party will display on both the View and Edit windows.

The Indicator column corresponding to each involved party displays the restricted party list screening (RPLS) status of the party. RPLS status is indicated as follows:

  • White: indicates status as RPLS_NOT STARTED
  • Green: indicates status as RPLS_PASSED
  • Red: indicates status as RPLS_FAILED
  • Yellow: indicates status as RPLS_REQUIRES REVIEW or RPLS_ESCALATED

Adding an Involved Party

  1. Select a qualifier from the Involved Party Qualifier ID drop-down list.

    Note: You can specify a Transmitter ID or Filer ID as reference numbers on the party.
    GTM will first check if a transmitter and/or filer is specified as an involved party. If so, GTM will use the reference number on the associated party to populate the Transmitter ID and Filer ID. If this data is not present on the party, then GTM will use the data in the properties.

  2. Select a contact in the Involved Party field.
  3. Click Save.

Control Screening Results

This section displays the following data associated with the selected transaction line:

  • Control Type
  • Control Code
  • Party Qualifier ID
  • Exception Control Type
  • Exception Control Code
  • Compliance Rule Set Group ID
  • Compliance Rule ID
  • Exception Compliance Rule ID

Additionally, if you have an authorization value, you can record the information in the Authorization Type and Authorization fields. This will be used to release a hold on the transaction line. Click Save after entering a value. Please note that if the transaction line has an exception code as a result of screening, you will not be able to record an authorization.

Note: The display of the Quantity and UOM fields is determined by the gtm.transaction.license.display.v62compatible property.

Assigned Licenses

This section displays license information that has already been associated with this transaction line.

Licenses are documents from a government entity that overrule any holds on a transaction line imposed by a control.

Other Control Screening Results

This section displays screening results based on the control category 'OTHER'. You can see the following data associated with the selected transaction line:

  • Control Type
  • Control Code
  • Compliance Rule Set Group ID
  • Compliance Rule ID

Additionally, if you have an authorization value, you can record the information in the Authorization field. Click Save after entering a value. This will release any hold on the transaction line resulting from the control code and control type provided in this section.

Sanction Control Results

This section displays the following data associated with the selected transaction line:
  • Control Type
  • Control Code
  • Compliance Rule Set Group ID
  • Compliance Rule ID

Additionally, if you have an authorization value, you can record the information in the Authorization, Quantity field. Click Save after entering a value. Please note that if the transaction line has an exception code as a result of screening, you will not be able to record an authorization.

Required Documents

This read-only section displays required documents that are associated with this transaction line.

Reporting Quantity

This section allows you to record reporting quantity details like product classification type, quantity qualifier, quantity, UOM, and attribute name for the transaction line.

Adding a Reporting Quantity

  1. Enter a product classification type in the Product Classification Type ID field.
  2. Enter a product classification code in the Classification Code field.
  3. Select a qualifier from the Quantity Qualifier ID drop-down list.
  4. Enter a value in the Quantity field.
  5. Enter a unit of measurement in the UOM field.
  6. Enter a name in the Attribute Name field. This attribute refers to which unit of measurement you select in the previous step. For example, the attribute name could be REPORTING UOM1 or REPORTING UOM2.
  7. Click Save.

User-defined Code

Adding a User-defined Code

  1. Select a user-defined code category from the Category drop-down list.
  2. Select a user-defined code type from the Type ID drop-down list. The available options are determined by your choice of category in the previous step.
  3. Select a user-defined code value from the Type Code drop-down list. The available options are determined by your choice of type in the previous step. For easy understanding of the user, this drop-down list displays description of the user-defined code values instead of only the codes.
  4. Click Save.

Remarks

Use this section to add as many remarks as you require.

Adding a Remark:

  1. Select a remark qualifier from the Remark Qualifier ID drop-down list.
  2. Enter your remarks in the Remark Text field.
  3. Click Save.

Reference Numbers

Use this section to record reference numbers that apply to this transaction. You can add as many reference numbers as you require.

Adding a Reference Number:

  1. Select a reference number qualifier from the Reference Number Qualifier ID drop-down list
  2. Enter the reference number in the Reference Number field.
  3. Click Save.

Value Formula Results

Note: This section is deprecated and is not supported.

This section displays value formula information associated with the item on transaction line.

Text Templates

Text templates are an output of the document compliance rule screening process and will be automatically associated to the trade transaction line. You can use this section to enter information that can override a text template.

Adding a Text Template:

  1. Select a template from the Text Template ID drop-down list.
  2. Enter a document type in the Document Type field.
  3. Enter any related text in the Text field.
  4. Click Save.
  5. You can use multiple templates.Â If one is to override the other templates, click Override All.

Transaction History

This section displays any note associated with this transaction line.

Here you can view the date and time when the note was created, a summary and the details of the note, and how it got assigned to this transaction line.

Qualifications

If you are viewing/editing a trade transaction line, this section displays qualification information of the transaction line.

Adding a Qualification:

  1. Click Add Qualification. The Transaction Line Qualification page is displayed where you can enter the qualification details.
  2. Select an option from the Status field to indicate whether the transaction line is qualified or if it requires to be reviewed.
  3. Enter an ID in the Trade AgreementÂ field.
  4. Enter a producer in the Producer field. An example of a producer is a manufacturer.
  5. Select an option from the Minimum Operations Met drop-down list to specify whether the line satisfies the requirements for minimum operations.
  6. Select an option from the Wholly Originating drop-down list to specify whether the line wholly originates from a region/country.
  7. Enter an ID in the Regional Value Content Method field.
  8. Enter a percentage value in the Regional Value Content Percentage field.
  9. Enter an ID in the Preference Criteria field.
  10. In the Values section, do the following:
    1. Select a value qualifier from the Value Qualifier ID drop-down list.
    2. Enter a currency value in the Value field.
    3. Select a currency from the drop-down list.
    4. Select an exchange rate from Â the Exchange Rate ID drop-down list.
    5. Enter the date for which the exchange rate is to be applied in the Exchange Rate Date field.
    6. Click Save.
  11. The Conditions section displays the conditions applicable for the transaction line.
  12. Click Save.
  13. Click Finished when you have recorded all of the information you need on this page.
  14. Click View Qualification Report to see the qualification analysis details of the transaction line.
