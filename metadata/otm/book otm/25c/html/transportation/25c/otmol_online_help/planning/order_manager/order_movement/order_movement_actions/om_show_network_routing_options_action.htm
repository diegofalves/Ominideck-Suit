Order Management

OM Show Network Routing Options Action

This page is accessed via Order Management > Order Movement > Order Movement > Actions > Operational Planning > Create Shipment > Show Network Routing Options.

This action focuses on shipments with or without a primary leg. You can use this option to find and filter routing options. OTM provides a list of possible routing options. After you decide on the option you want to use, you can create the shipment.

There are several stages and options associated with using this action, which are described below. The process begins by selecting an Order Movement and using the action Show Network Routing Options under Create Shipment.

The order movement action Show Network Routing Options - Ocean is similar to this action but a primary leg is required.

The Logic Configuration - Network Routing parameter ALLOW MIXED EQUIPMENT GROUP ROUTING OPTIONS affects this action.

The parameter USE ROUTING CONSTRAINT FOR PORT SELECTION Â should be off (false) when using Show Network Routing Options.

Points to Remember:

  • This works with multiple order movements Â All order movements must be network routable (the Network Routable checkbox on the order movement must be selected).
  • They must be able to be built in one order movement bundle.
  • If an order movement resides in a container, then all selected order movements must belong to the same container, and the container must contain only the selected order movements.
  • This will work with or without a primary leg.
  • Evaluate all options in a route in a single step.

Routing Constraints

The routing constraints do not populate automatically based on the order movement constraints. If a constraint field is not populated automatically and you do not enter an override in the input screen,Â the constraint in the order will be applied to the main leg shipment, if there is a main leg. The constraints defined on a parent order movement are currently not used, except for the Routing Constraint ID on the parent order movement which is used for routing the parent order movement.

  1. Select routing constraints. A page appears that allows you to limit the valid routes returned by Oracle Transportation Management to just those that have matching values to what you enter. Constraint options include:
    1. Parameter Set ID will be populated here if the Planning Criteria screen is turned on and you have selected Parameter Set ID from the Planning Criteria screen.
    2. The Max Options per Route field controls the number of route options returned per route.
    3. Select any additional constraints.
      • Transport Mode and Mode Profile
      • Service Provider and Service Provider Profile
      • Equipment Group and Equipment Group Profile
  2. If you want Oracle Transportation Management to return all valid routes except certain ones, then enter that value in the appropriate field and check the corresponding Exclude checkbox. If you do not enter any constraints, all valid routes are returned.
  3. The Apply Constraints to Leg field specifies whether the transport mode, service provider and equipment group constraints should be applied to the primary leg (if any) or the first leg in a route.
  4. When complete, click the Show Options button. This will calculate and display the options of valid routes.

Show Routing Options

All the valid routes from the previous page appear in the Show Routing Options page. Any routes that could not be built into feasible shipments are labeled accordingly.

All routes that can be built into feasible shipments are listed in order of cost and show shipment start and end times. You can use the Expand icon to drill down to see shipment details.

This shows the particular options for one selected path. It shows the cheapest service provider that has capacity. The 'Total Actual Cost' and 'Transit Time' values are accurate for the actual shipment to be built.

Select the route that you want to use to create the shipment, and click OK.
