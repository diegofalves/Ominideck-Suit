Access Control Lists

You use access control lists to access REST resources. You can add them to your own user roles, as required. To access data, you must also include the user role that you use to access REST services with a data role in Transportation and Global Trade Management Cloud.

Note:

To identify the privileges needed for accessing a resource, including power data, follow these steps:
  1. Go to Configuration and Administration > User Management > Access Control List.
  2. Enter the end point you want access to in the Access Control Entry Point field. For example, the end point for the Accessorial Codes power data resource is "accessorialCodes". You can find the entry points later in this guide, in the Tasks section. To find the Accessorial Codes example, in this guide go to Tasks > Power Data > Accessorial Codes. The ID appears as part of the Path listed in the records.
  3. Use the "Contains" option for the field.
  4. Click Search.

This will list all the associated Access Control List IDs for that resource. Note that a power data resource can be associated with more than one root resource. Any one of those privileges will be sufficient for access.
