Supported Media Types

Media types let you choose the format of the data you want to receive, defining the structure of the HTTP payloads exchanged between the server and the client. They allow an API to inform the client how to interpret the data in the payload. The following table lists the supported media types for REST APIs in Oracle Applications Cloud.

Note:

The following media types are the preferred values but for all except JSON PATCH, the basic "application/json" media type can also be specified.

Media Type Description
application/vnd.oracle.resource+json; type=collection

Represents a resource collection.
application/vnd.oracle.resource+json; type=singular

Represents a resource item.
application/vnd.oracle.resource+json; type=query-def

Represents an Advanced query definition.
application/vnd.oracle.resource+json; type=error

Represents an error response containing more detailed information on the cause of the error.
application/vnd.oracle.resource+json; type=status

Represents a status response to an asynchronous request.
application/json-patch+json

Represents a standard JSON PATCH message.
