Resource Types

Error Detail Resource

The error detail resource provides detailed information about a failed request. Typically, an error detail resource is returned when the request fails and, in addition to the HTTP error code and error message, the server provides more information about the failure.

The resource can contain the following properties:

Name Type Description Required
title String Summary error message Yes
status String HTTP status code No
detail String Detailed error message No
o:errorCode String Application specific error code No
o:errorDetails Array Drill down to the details. Multiple errors can be organized in a hierarchical structure. No

For example:

{
    "status": 409,
    "title": "Conflict",
    "detail": "The request could not be completed due to a conflict with the current state of the resource.",
    "o:errorDetails": [
        {
            "title": "BatchUpdateException"
        }
    ]
}

The above detail would suggest a SQLException, e.g. Primary Key or Foreign Key constraint violation. The exact nature of the exception needs to be observed in the application log.
