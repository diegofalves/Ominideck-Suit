Supported Data Types

No matter what client you use to make a REST API call, both the sent request and delivered response (when a status response) are in JavaScript Object Notation (JSON). Exported content will either be "text/plain", "text/csv" or "application/octet-stream". The latter is used for compressed content. The following table lists the JSON data types for resource attributes in our REST APIs.

JSON Data Type Description
string

Any sequence of characters added between double quotes. Similar to a C or Java string. Example: {"name":"Andrew"}
date

A string value conforming to the ISO 8601 format. For example, the data type for DateValidated is string (date). The value can be specified with or without double quote characters.

datetime

A string value conforming to the ISO 8601 format. For example, the data type for LastUpdateDate is string (date-time). The value can be specified with or without double quote characters. In addition to the standard ISO 8601 format, it's possible to specify a time zone property associated with a value not containing the time zone. For example, the following formats are equivalent:

ISO 8601:

{
   "value":"2022-01-02T16:00:00-05:00"
}

With time zone:

{
   "value":"2022-01-02T16:00:00",
   "timezone":"America/New York"
}
boolean

A value that must be either true or false.

number

A numeric value represented in base 10 with no extra leading zero. It includes digits between 0 and 9, can be negative, a fraction, or an exponent of 10 prefixed by e. The value doesn't include the octal and hexadecimal formats.

integer

A number without a fraction part or an exponent part.
null

No value to assign.

object

An unordered set of name and value pairs. A name is a string, and a value can be a string, a number, a Boolean, a null, an object, or an array data type.

array

An ordered collection of values, beginning with a [ (left bracket), ending with an ] (right bracket), and including values separated by commas. Example: {"contacts":[ "John", "Anna", "Peter" ]}
